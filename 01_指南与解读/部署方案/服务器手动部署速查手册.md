## 五、训练命令

### 5.1 冒烟测试（确认一切正常，每次修改代码后先跑）

```powershell
# 单患者 1 epoch（2-5 分钟）
python train_histogene_uni_tokens_augmix.py --patient HYZ15040 --epochs 1

# 跨患者 Fold 1 单 epoch（5-10 分钟）
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 1 --epochs 1
```

### 5.2 单患者完整训练（快速验证新想法，~2 小时）

```powershell
python train_histogene_uni_tokens_augmix.py --patient HYZ15040 --epochs 50 --tv_weight 0.01 --tv_mode l2
```

### 5.3 跨患者三折 CV（最终评估，每折 5-10 小时）

```powershell
# 三个独立的 PowerShell 窗口分别运行：

# 窗口 1 — Fold 1 (JFX+LMZ → HYZ)
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 1 --epochs 150 --tv_weight 0.01 --tv_mode l2

# 窗口 2 — Fold 2 (HYZ+LMZ → JFX)
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 2 --epochs 150 --tv_weight 0.01 --tv_mode l2

# 窗口 3 — Fold 3 (HYZ+JFX → LMZ)
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 3 --epochs 150 --tv_weight 0.01 --tv_mode l2
```

> ⚠️ RTX 4080 16GB：同时跑 2 个 Fold 一般可行，3 个可能 OOM。建议先跑 2 个，观察 `nvidia-smi` 显存占用再决定。

### 5.4 断点续训

```powershell
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 1 --epochs 150 --resume checkpoints/TV_Sweep_tv_l2_w0.01/best_histogene_uni_tokens_augmix.pth
```

### 5.5 加载已有权重仅评估

```powershell
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 1 --epochs 0 --resume checkpoints/xxx/best_xxx.pth
```

## 六、监控与调试

### 6.1 查看 GPU 状态

```powershell
nvidia-smi
# 关注：显存使用（Memory-Usage）、GPU 利用率（GPU-Util）、进程列表
```

### 6.2 查看训练日志

训练过程会实时打印到终端。关键指标解读：

```
Epoch 50/150
Train | loss=0.023456 mae=0.123 r2=0.456 pcc=0.389
Val   | loss=0.031234 mae=0.145 r2=0.401 pcc=0.417  ← 关注 val_pcc

*** New best val loss: 0.031234 ***   ← 最优 checkpoint 已保存
No improvement. Early stop patience: 3/10  ← 连续 3 epoch 无提升
```

- **正常训练**：train_loss 和 val_loss 逐渐下降
- **过拟合信号**：train_loss 持续下降但 val_loss 上升
- **收敛信号**：val_loss 多轮不变，触发 early stop

## 七、常见问题速查

### 🔴 启动/环境类

| 报错 | 原因 | 解决 |
|------|------|------|
| `can't open file 'config_utils.py'` | 当前目录不在项目根 | `cd D:\AIPatho\qzs\pfmval_deploy` |
| `ModuleNotFoundError: No module named 'uni2h'` | 同上 | 同上 |
| `'pip' is not recognized` | venv 创建不完整 | 用 `python -m pip` 代替 `pip` |
| `OfflineModeIsEnabled` | 之前设了离线变量 | 关掉当前 PowerShell，重新打开 |
| `UnicodeEncodeError` | 中文 Windows 编码问题 | 设 `$env:PYTHONIOENCODING = "utf-8"` |

### 🟡 CUDA / GPU 类

| 报错 | 原因 | 解决 |
|------|------|------|
| `CUDA out of memory` | 显存不足 | 减少 batch_size 或关闭其他训练窗口 |
| `torch.cuda.is_available() → False` | PyTorch 没装 CUDA 版 | 重新安装：`python -m pip install torch --index-url https://download.pytorch.org/whl/cu124` |
| `NVIDIA driver too old` | 驱动版本过低 | 当前服务器 Driver 576.02，足够新，一般不是这个问题 |

### 🟠 模型/数据类

| 报错 | 原因 | 解决 |
|------|------|------|
| `LocalEntryNotFoundError` (UNI2-h) | HF 缓存未就绪 | 需管理员复制旧用户缓存（见 §3.2） |
| `FileNotFoundError: patches_dir` | 数据路径配置错误 | 运行 `python config_utils.py` 检查路径 |
| `FileNotFoundError: *.pt cache` | 特征缓存未生成 | 运行 `python extract_uni_tokens.py` 先生成缓存 |
| `No PNG patches matched labels` | patches 和 CSV 文件名不匹配 | 检查 patches 目录和 labels CSV 的患者 ID 前缀 |

### 🟢 训练过程类

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| loss 一直不降 | 学习率太低/太高 | 检查 lr 默认值（当前 3e-5） |
| val_pcc 很低 (<0.1) | 特征缓存加载有问题 | 确认特征维度是 1536（不是 768） |
| 训练很慢（>2h/epoch） | batch_size 太小或磁盘 IO 瓶颈 | 当前默认 batch_size=64 正常 |
| 电脑被其他用户重启 | Windows Update / 手动重启 | 确认是否有自动更新策略 |

## 八、结果回收（RDP 方式）

训练完成后，打包结果并通过 RDP 拖回本地：

```powershell
# 在服务器上打包
cd D:\AIPatho\qzs\pfmval_deploy
Compress-Archive -Path @(
    "checkpoints\CrossPatient_Fold1_*",
    "checkpoints\CrossPatient_Fold2_*",
    "checkpoints\CrossPatient_Fold3_*",
    "predictions*.csv"
) -DestinationPath "D:\pfmval_results_$(Get-Date -Format 'yyyyMMdd').zip"
```

然后从 `D:\pfmval_results_YYYYMMDD.zip` 通过 RDP 复制回本地。
