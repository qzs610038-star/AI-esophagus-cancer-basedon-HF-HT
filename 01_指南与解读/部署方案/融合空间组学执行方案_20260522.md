## Phase 1: 多教师晚期融合（UNI2-h + Virchow2 预测加权平均）

### 目标

将 UNI2-h 和 Virchow2 两个教师模型的独立预测按 per-pathway 最优权重融合，零额外训练成本。

### 原理

- UNI2-h (ViT-h/14, 1536-dim) 和 Virchow2 (1280-dim) 是两个完全不同的病理基础模型
- 两者特征来源正交：UNI2-h 在大规模病理数据上自监督预训练，Virchow2 在另外的病理数据上训练
- 对 30 条基因通路，不同模型擅长的通路不同
- 晚期融合 = 在最终预测层面做加权平均，无需重新训练任何模型

### 关键问题：找到对齐的预测文件

当前 Fold1 的预测文件：

| 模型 | 预测文件 | PCC |
|------|---------|:---:|
| UNI2-h + AugMix | `CrossPatient_JFX_LMZ_to_HYZ_UNI_tokens_AugMix_20260516_164210/predictions.csv` | 0.4142 |
| Virchow2 Tokens | `CrossPatient_Fold1_Virchow2_Tokens_20260521_110949/predictions.csv` | 0.3537 |

**前提条件**：两个 CSV 的行（patch）必须一一对应（同一患者 HYZ15040 的同一组测试 patch）。

### 实施步骤

#### Step 1.1: 创建融合脚本 `ensemble_late_fusion.py`

路径：`d:\AI空间转录病理研究\PFMval_new\ensemble_late_fusion.py`

```
功能模块：
1. load_predictions(csv_path) → (true_df, pred_df)
   - 读取 predictions.csv
   - 解析 true_xxx / pred_xxx 列
   - 返回 (n_samples, 30) 的 true 和 pred DataFrame

2. align_predictions(preds_a, preds_b, trues_a, trues_b)
   - 按 patch_id 对齐两份预测（如果CSV包含patch_id列）
   - 或按行索引对齐（假设两份预测是同一数据集同一顺序）
   - 验证 true_a ≈ true_b（标签应一致）
   - 返回对齐后的 (true, pred_a, pred_b)

3. grid_search_weights(pred_a, pred_b, true, granularity=0.05)
   - 对每条通路独立搜索最优权重 w ∈ [0, 1]，步长 0.05
   - 融合预测 = w * pred_a + (1-w) * pred_b
   - 评估指标：per-pathway PCC
   - 返回 (best_weights[30], best_pcc[30])

4. evaluate_fusion(pred_a, pred_b, true, weights)
   - 计算融合后的全局 PCC、MAE、R²
   - 对比单模型 vs 融合的 per-pathway PCC
   - 生成融合报告

5. 额外策略（可选，快速验证）：
   - uniform_fusion: w=0.5 全通路统一
   - oracle_fusion: 每条通路选 PCC 更高的模型（理论上界）
   - pcc_weighted: w = pcc_a / (pcc_a + pcc_b) 按单模型 PCC 比例
```

## Phase 2: 空间 Total Variation (TV) 正则化损失

### 目标

在训练过程中加入空间平滑约束，惩罚相邻 patch 之间预测值的剧烈跳变。

### 原理

生物学先验：组织切片上物理相邻的区域，其基因表达谱应该是渐变的而非突变的。当前模型逐 patch 独立预测（虽使用坐标嵌入但未被显式约束），可能在空间上产生不连续的预测。TV Loss 直接惩罚相邻 patch 预测差异，起平滑正则化作用。

### 实现位置

- **新文件**: `spatial_tv_loss.py`
- **修改文件**: `train_histogene_uni_tokens.py`（加 `--tv_loss_weight` 参数）
- **修改文件**: `train_histogene_uni_tokens_augmix.py`（同上）

### 实施步骤

```python
        # 2. 对每条边 (i,j)，计算 |pred[i] - pred[j]|
        # 3. 返回 mean over edges
```

关键设计决策：
- **KNN on normalized coords**：与 GAT 中 `graph_utils_gat.py` 一致，`k=6`
- **batch 内建图**：每个 batch 独立构建邻居图（不需要跨 batch 图）
- **L1 vs L2**：默认 L1（对 outlier 邻居差异更鲁棒）
- **per-pathway 还是 global**：先做 global（30 条通路一起算），后续可改为 per-pathway

## Phase 3: Patch 自适应加权（软注意力）

### 目标

将 `LightweightTokenEncoder` 中的 `x.mean(dim=1)`（全局平均池化）替换为可学习的注意力加权，让模型自动学习"哪些 token（对应 patch 内不同区域）对预测每条通路更有信息量"。

### 原理

当前 token encoder 的池化逻辑（[model_uni_tokens.py:62](model_uni_tokens.py#L62)）：

```python
x = x.mean(dim=1)  # 所有 token 等权平均
```

这隐含假设所有 65 个 token 同等重要。但实际上：
- 不同组织形态学模式分布在不同 patch 区域
- 不同通路可能关注不同的组织区域（如免疫通路关注淋巴细胞浸润区域）
- 当前 AugMix 方案的 2 层 Transformer 已有 token 间交互，但池化仍是等权

改进方向：将 mean pooling 替换为 **learnable query-based attention**（类 Perceiver 风格）或 **token-level soft attention**。

### 两种可选方案

#### 方案 A：多头自注意力池化（轻量，推荐优先实现）

```
Token序列 [B, 65, hidden_dim]
    ↓
Self-Attention (已有, 来自 TransformerEncoder)
    ↓
Learnable attention pooling:
    attn_scores = softmax(Linear(x))  # [B, 65, 1]
    x = (x * attn_scores).sum(dim=1)  # [B, hidden_dim]
    ↓
output_proj → LayerNorm → [B, embed_dim]
```

参数量增加：`hidden_dim → 1` 的一个 Linear 层（~512 参数），几乎零成本。

#### 方案 B：Perceiver-style Learnable Queries（进阶）

```
Token序列 [B, 65, hidden_dim]
    ↓
Learnable queries [n_queries, hidden_dim] 交叉注意力 tokens
    ↓
query 聚合 → [B, n_queries * hidden_dim]
    ↓
output_proj → [B, embed_dim]
```

参数量增加：n_queries × hidden_dim 的可学习 query 向量 + cross-attention 层（~2-3M），收益可能更高但开发量更大。

### 实施步骤（方案 A — 优先）

#### Step 3.1: 修改 `LightweightTokenEncoder`

在 [model_uni_tokens.py:19-64](model_uni_tokens.py#L19-L64) 中：

```python
class LightweightTokenEncoder(nn.Module):
    def __init__(self, ..., use_attn_pool=False):
        ...
        if use_attn_pool:
            # 替换 mean pooling 为注意力池化
            self.attn_pool = nn.Sequential(
                nn.Linear(hidden_dim, hidden_dim // 4),
                nn.ReLU(),
                nn.Linear(hidden_dim // 4, 1)
            )
        self.use_attn_pool = use_attn_pool

    def forward(self, tokens):
        ...
        x = self.encoder(x)              # [B, num_tokens, hidden_dim]
        
        if self.use_attn_pool:
            attn = self.attn_pool(x)     # [B, num_tokens, 1]
            attn = attn.softmax(dim=1)
            x = (x * attn).sum(dim=1)    # 加权池化
        else:
            x = x.mean(dim=1)            # 原始等权池化
        
        x = self.output_proj(x)
        x = self.norm(x)
        return x, attn  # 返回注意力权重用于分析
```

注意：`forward` 返回值从 `[B, embed_dim]` 变为 `([B, embed_dim], attn_weights)`。需要同步修改 `HisToGeneUNITokens.forward`。

#### Step 3.2: 同步修改 `HisToGeneUNITokens`

在 [model_uni_tokens.py:116-129](model_uni_tokens.py#L116-L129) 中：

```python
def forward(self, tokens, pos_x, pos_y):
    encoded, attn_weights = self.token_encoder(tokens)
    x = self.proj(encoded)
    x = x + self.x_embed(pos_x) + self.y_embed(pos_y)
    x = self.head(x)
    return x
```

## 风险评估与预案

| 风险 | 概率 | 影响 | 预案 |
|------|:---:|------|------|
| UNI 和 Virchow2 预测行不对齐 | 中 | Phase 1 失败 | 用 patch_id 做精确对齐；若无 patch_id，需重新推理两份模型 |
| TV Loss 过强导致欠拟合 | 低 | Phase 2 效果不显著 | 降低 tv_weight，或只在 early stopping 后有改善时启用 |
| Attn Pool 与现有 AugMix 不兼容 | 低 | Phase 3 需要 debug | 先在不带 AugMix 的 base 版本验证，再迁移 |
| 3 患者交叉验证时 TV Loss 行为不一致 | 中 | 结论不可靠 | 优先在单患者验证，确认有效再扩展到跨患者 |
| 融合后 PCC 无提升 | 中 | Phase 1 收益为 0 | 检查 per-pathway PCC 互补性；若无互补则放弃融合方向 |
