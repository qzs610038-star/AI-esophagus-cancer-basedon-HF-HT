# PFMval 项目 Linux 服务器部署指南

> 本指南面向初学者，帮助你将 PFMval 项目从 Windows 迁移到 Linux GPU 服务器运行。

---

## 1. 环境准备

### 1.1 检查 GPU 和 CUDA 驱动

登录服务器后，首先确认 GPU 是否可用：

```bash
# 查看 GPU 信息和驱动版本
nvidia-smi
```

输出示例：
```
+---------------------------------------------------------------------------------------+
| NVIDIA-SMI 535.104.05             Driver Version: 535.104.05   CUDA Version: 12.2     |
|-----------------------------------------+----------------------+----------------------+
| GPU  Name                 Persistence-M | Bus-Id        Disp.A | Volatile Uncorr. ECC |
| Fan  Temp   Perf          Pwr:Usage/Cap |         Memory-Usage | GPU-Util  Compute M. |
|                                         |                      |               MIG M. |
|=========================================+======================+======================|
|   0  NVIDIA A100-SXM4-40GB          On  | 00000000:00:04.0 Off |                    0 |
| N/A   35C    P0              43W / 400W |      0MiB / 40960MiB |      0%      Default |
+-----------------------------------------+----------------------+----------------------+
```

> **关键信息解读**：
> - `Driver Version`: NVIDIA 驱动版本
> - `CUDA Version`: 系统支持的最高 CUDA 版本
> - 最下面的表格显示 GPU 内存使用情况

### 1.2 安装 Anaconda/Miniconda

```bash
# 下载 Miniconda（推荐，体积更小）
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh

# 运行安装脚本
bash Miniconda3-latest-Linux-x86_64.sh

# 按提示完成安装，安装完成后重启终端或执行：
source ~/.bashrc
```

### 1.3 创建 Python 虚拟环境

```bash
# 创建名为 pfmval 的环境，Python 3.10
conda create -n pfmval python=3.10 -y

# 激活环境
conda activate pfmval

# 验证 Python 版本
python --version
```

### 1.4 安装 PyTorch GPU 版本

根据你的 CUDA 版本选择安装命令（从 [pytorch.org](https://pytorch.org) 获取）：

```bash
# CUDA 11.8 版本（推荐，兼容性好）
pip install torch==2.1.0 torchvision==0.16.0 torchaudio==2.1.0 --index-url https://download.pytorch.org/whl/cu118

# CUDA 12.1 版本
pip install torch==2.1.0 torchvision==0.16.0 torchaudio==2.1.0 --index-url https://download.pytorch.org/whl/cu121
```

### 1.5 验证 PyTorch GPU 可用

```bash
python -c "import torch; print('CUDA available:', torch.cuda.is_available()); print('GPU count:', torch.cuda.device_count()); print('GPU name:', torch.cuda.get_device_name(0) if torch.cuda.is_available() else 'N/A')"
```

期望输出：
```
CUDA available: True
GPU count: 1
GPU name: NVIDIA A100-SXM4-40GB
```

## 2. 项目文件传输

### 2.1 使用 scp 上传（简单场景）

```bash
# 从 Windows PowerShell 或 Linux 终端执行
# 上传整个项目文件夹到服务器的 /home/username/ 目录
scp -r D:\AI空间转录病理研究\PFMval_new username@server_ip:/home/username/
```

### 2.2 使用 rsync 增量同步（推荐）

rsync 支持断点续传和增量同步，更适合大项目：

```bash
# 在 Windows 上安装 Git Bash 或 WSL 后使用 rsync
# 排除不需要的大文件
rsync -avz --progress \
  --exclude='*.pth' \
  --exclude='*.pt' \
  --exclude='uni2h_cache/' \
  --exclude='__pycache__/' \
  --exclude='.venv/' \
  --exclude='checkpoints/' \
  --exclude='HYZ15040/' \
  --exclude='.git/' \
  /d/AI空间转录病理研究/PFMval_new/ \
  username@server_ip:/home/username/PFMval_new/
```

> **参数说明**：
> - `-a`: 归档模式，保留文件属性
> - `-v`: 显示详细过程
> - `-z`: 传输时压缩
> - `--exclude`: 排除指定文件/目录

### 2.3 传输后检查

```bash
# 登录服务器后检查文件
ssh username@server_ip
cd /home/username/PFMval_new
ls -la
```

## 3. 配置修改（重点章节）

### 3.1 核心原则

> **只需修改 `config.yaml` 即可完成数据路径切换！**

PFMval 项目使用统一的配置文件管理所有路径，迁移时无需修改代码。

### 3.2 config.yaml 字段详解

```yaml
# config.yaml 示例（Linux 服务器版本）
data:
  # 训练数据 patches 目录
  train_patches_dir: "/data/HYZ15040/train_patches"
  
  # 验证数据 patches 目录
  val_patches_dir: "/data/HYZ15040/val_patches"
  
  # 原始标签 CSV 文件路径
  labels_csv_raw: "/data/HYZ15040_ssGSEA_scores.csv"
  
  # Z-score 标准化后的标签 CSV 文件路径
  labels_csv_zscore: "/data/HYZ15040_ssGSEA_scores_zscore.csv"

model:
  # 模型名称
  name: "histogene"
  
  # 是否仅使用本地模型（离线模式）
  local_only: false
  
  # 本地模型缓存路径
  cache_dir: "/data/hf_cache"

training:
  # 批次大小，根据 GPU 内存调整
  batch_size: 16
  
  # 训练轮数
  epochs: 100
  
  # 学习率
  learning_rate: 0.001
```

### 3.3 路径修改要点

| Windows 路径 | Linux 路径 |
|-------------|-----------|
| `D:\data\file.csv` | `/data/file.csv` |
| `HYZ15040\train_patches` | `/data/HYZ15040/train_patches` |
| `.\checkpoints\` | `./checkpoints/` 或绝对路径 |

> **重要提醒**：
> - Linux 使用正斜杠 `/` 作为路径分隔符
> - Linux 路径区分大小写
> - 建议使用绝对路径（以 `/` 开头），避免相对路径问题
> - 避免在路径中使用中文，防止编码问题

### 3.4 快速修改命令

```bash
cd /home/username/PFMval_new

# 使用 sed 批量替换路径（谨慎使用）
sed -i 's|\\\\|/|g' config.yaml

# 或使用 nano/vim 手动编辑
nano config.yaml
```

## 5. HuggingFace 模型配置

### 5.1 设置 HF_TOKEN

UNI2-h 模型需要从 HuggingFace 下载，需要设置访问令牌：

```bash
# 临时设置（仅当前终端有效）
export HF_TOKEN="hf_xxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# 验证设置成功
echo $HF_TOKEN
```

### 5.2 持久化到 .bashrc

```bash
# 编辑 .bashrc
echo 'export HF_TOKEN="hf_xxxxxxxxxxxxxxxxxxxxxxxxxxxx"' >> ~/.bashrc

# 使配置生效
source ~/.bashrc
```

### 5.3 设置 HuggingFace 缓存目录

```bash
# 将缓存目录设置到数据盘（避免占用系统盘空间）
export HF_HOME="/data/hf_cache"
export TRANSFORMERS_CACHE="/data/hf_cache"

# 持久化
echo 'export HF_HOME="/data/hf_cache"' >> ~/.bashrc
echo 'export TRANSFORMERS_CACHE="/data/hf_cache"' >> ~/.bashrc
```

### 5.4 离线模式（服务器无外网）

如果服务器无法访问外网，需要提前下载模型：

**步骤 1：在本地机器下载模型**

```bash
# 在能访问外网的机器上
python -c "
from huggingface_hub import snapshot_download
snapshot_download(
    repo_id='MahmoodLab/UNI2-h',
    cache_dir='./hf_cache',
    local_dir='./uni2h_model'
)
"
```

**步骤 2：上传模型到服务器**

```bash
rsync -avz ./uni2h_model/ username@server_ip:/data/hf_cache/uni2h_model/
```

**步骤 3：修改 config.yaml 启用离线模式**

```yaml
model:
  local_only: true
  cache_dir: "/data/hf_cache"
```

## 6. 训练启动

### 6.1 各模型启动命令

```bash
cd /home/username/PFMval_new

# HisToGene 模型训练
python histogene/train.py

# UNI2-h DenseNet MLP 训练
python uni2h_new/train_des.py

# UNI2-h 其他变体
python uni2h_new/train_mlp.py
```

### 6.2 使用 nohup 后台运行

```bash
# 后台运行 HisToGene 训练，输出保存到 train.log
nohup python histogene/train.py > train.log 2>&1 &

# 查看进程是否在运行
ps aux | grep train.py

# 查看实时日志
tail -f train.log
```

> **命令解析**：
> - `nohup`: 忽略挂起信号，终端关闭后继续运行
> - `> train.log`: 标准输出重定向到文件
> - `2>&1`: 标准错误也重定向到标准输出
> - `&`: 后台运行

### 6.3 使用 tmux 会话管理（强烈推荐）

tmux 可以创建持久会话，断开 SSH 后训练不会中断。

```bash
# 安装 tmux（如未安装）
sudo apt-get install tmux  # Ubuntu/Debian

# 创建新会话
tmux new -s pfmval_train

# 在 tmux 会话中启动训练
conda activate pfmval
cd /home/username/PFMval_new
python histogene/train.py

# 分离会话（训练继续后台运行）
按 Ctrl+B，然后按 D

# 重新连接会话
tmux attach -t pfmval_train

# 查看所有会话
tmux ls

# 终止会话
tmux kill-session -t pfmval_train
```

> **tmux 常用快捷键**：
> - `Ctrl+B D`: 分离会话（detach）
> - `Ctrl+B %`: 垂直分屏
> - `Ctrl+B "`: 水平分屏
> - `Ctrl+B 方向键`: 切换面板

### 6.4 完整的训练启动流程示例

```bash
# 1. 登录服务器
ssh username@server_ip

# 2. 创建 tmux 会话
tmux new -s histogene_train

# 3. 激活环境
conda activate pfmval

# 4. 进入项目目录
cd /home/username/PFMval_new

# 5. 检查配置
cat config.yaml

# 6. 启动训练
python histogene/train.py 2>&1 | tee train_$(date +%Y%m%d_%H%M%S).log

# 7. 分离会话（按 Ctrl+B，然后 D）
# 训练将在后台继续运行

# 8. 稍后查看进度
tmux attach -s histogene_train
```

### 6.5 查看训练进度

```bash
# 实时查看日志
tail -f train.log

# 查看最近 50 行
tail -n 50 train.log

# 查看包含特定关键词的行
grep "Epoch" train.log
grep "Loss" train.log

# 查看 GPU 使用情况（另一个终端）
watch -n 1 nvidia-smi
```

## 7. 常见问题与解决方案

### 7.1 CUDA Out of Memory

**症状**：
```
RuntimeError: CUDA out of memory. Tried to allocate X.XX GiB
```

**解决方案**：
```bash
# 修改 config.yaml，减小 batch_size
training:
  batch_size: 8  # 从 16 减小到 8

# 或使用梯度累积（如果代码支持）
# 清空 GPU 缓存
python -c "import torch; torch.cuda.empty_cache()"
```

### 7.2 权限问题

```bash
# 给脚本添加执行权限
chmod +x train.py

# 修改目录权限
chmod -R 755 /data/HYZ15040

# 修改文件所有者（如需要）
sudo chown -R username:username /home/username/PFMval_new
```

### 7.3 路径分隔符问题

> **警告**：Windows 使用 `\`，Linux 使用 `/`

```bash
# 错误示例（Windows 风格）
train_patches_dir: "\data\HYZ15040\train_patches"

# 正确示例（Linux 风格）
train_patches_dir: "/data/HYZ15040/train_patches"
```

### 7.4 中文路径问题

```bash
# 避免使用中文路径
# 错误
/data/病理数据/HYZ15040

# 正确
/data/pathology_data/HYZ15040
```

### 7.5 ModuleNotFoundError

```bash
# 检查虚拟环境是否激活
conda info --envs
conda activate pfmval

# 检查 Python 路径
which python
python -c "import sys; print('\n'.join(sys.path))"

# 重新安装缺失的模块
pip install xxx
```

### 7.6 HuggingFace 下载超时

```bash
# 使用镜像站（国内服务器）
export HF_ENDPOINT=https://hf-mirror.com

# 或设置代理
export HTTPS_PROXY=http://proxy_address:port

# 或使用离线模式（见第 5.4 节）
```

### 7.7 matplotlib 中文显示问题

```bash
# 安装中文字体
sudo apt-get install fonts-wqy-zenhei

# 清除 matplotlib 缓存
rm -rf ~/.cache/matplotlib

# 在代码中设置字体
import matplotlib.pyplot as plt
plt.rcParams['font.sans-serif'] = ['WenQuanYi Zen Hei']
plt.rcParams['axes.unicode_minus'] = False
```

### 7.8 端口转发（Jupyter Notebook）

```bash
# 在服务器上启动 Jupyter
jupyter notebook --no-browser --port=8889

# 在本地机器建立 SSH 隧道
ssh -L 8889:localhost:8889 username@server_ip

# 然后在本地浏览器访问 http://localhost:8889
```
