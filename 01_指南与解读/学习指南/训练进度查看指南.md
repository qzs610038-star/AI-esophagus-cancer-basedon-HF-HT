## 实时查看训练进度

### 方法 1：终端输出监控

当你在终端中运行训练脚本时，会实时输出每个 epoch 的训练信息：

#### HisToGene 模型
```bash
python histogene/train.py
```

输出示例：
```
Epoch [  1/150] Train Loss: 0.5234 MAE: 0.4123 R²: 0.3456 PCC: 0.6234 | Val Loss: 0.4987 MAE: 0.3987 R²: 0.3789 PCC: 0.6456 | LR: 1.00e-04 | 45.2s
  ✓ 最佳模型已保存 (val_loss=0.4987)
```

#### UNI2-h+MLP 模型
```bash
python uni2h_new/train_des.py
```

输出示例：
```
Epoch 1/100
Train | loss=0.523456 mae=0.412345 mape=12.345678 r2=0.345678 pcc=0.623456
Val   | loss=0.498765 mae=0.398765 mape=11.234567 r2=0.378901 pcc=0.645678
*** New best val loss: 0.498765 ***
```

### 关键指标说明

| 指标 | 含义 | 判断标准 |
|------|------|----------|
| **Loss** | 损失值（MSE/Huber） | 越低越好，验证集损失下降表示模型在改进 |
| **MAE** | 平均绝对误差 | 越低越好，表示预测值与真实值的平均偏差 |
| **MAPE** | 平均绝对百分比误差 | 越低越好，表示相对误差百分比 |
| **R²** | 决定系数 | 越高越好，范围通常 0~1，表示模型解释方差的比例 |
| **PCC** | 皮尔逊相关系数 | 越高越好，范围 -1~1，表示预测与真实的相关性 |
| **LR** | 学习率 | 会随着训练动态调整（ReduceLROnPlateau） |

### 方法 2：后台运行时查看日志

如果你在后台运行训练，可以将输出重定向到日志文件：

```bash
# Windows PowerShell
python histogene/train.py > training_log.txt 2>&1

# 实时查看日志
Get-Content training_log.txt -Wait
```

```bash
# Linux/Mac
python histogene/train.py > training_log.txt 2>&1 &

# 实时查看日志
tail -f training_log.txt
```

## 查看训练历史记录

### HisToGene 模型

训练历史保存在 CSV 文件中：

```bash
# 默认路径
histogene/training_history.csv
```

使用 Python 查看：
```python
import pandas as pd

# 读取训练历史
history = pd.read_csv('histogene/training_history.csv')

# 查看最新 10 个 epoch
print(history.tail(10))

# 查看最佳验证损失
best_idx = history['val_loss'].idxmin()
print(f"\n最佳 Epoch: {history.loc[best_idx, 'epoch']}")
print(f"最佳 Val Loss: {history.loc[best_idx, 'val_loss']:.4f}")
print(f"最佳 Val PCC: {history.loc[best_idx, 'val_pcc']:.4f}")
```

### UNI2-h+MLP 模型

训练历史保存在检查点同目录下：

```bash
# 默认路径（根据你的配置）
uni2h_new/checkpoints/HYZ15040/best_model_uni2h.history.csv
```

查看方法相同：
```python
import pandas as pd

history = pd.read_csv('uni2h_new/checkpoints/HYZ15040/best_model_uni2h.history.csv')
print(history.tail(10))
```

## 可视化训练结果

训练完成后，系统会自动生成可视化结果：

### 可视化文件位置

- **HisToGene**: `histogene/results_vis/`
- **UNI2-h+MLP**: `uni2h_new/results_vis/`

### 生成的图表

1. **训练曲线图** (`training_curves.png`)
   - 包含 Loss、MAE、R²、PCC 的训练/验证曲线
   - 直观展示模型学习过程

2. **逐通路指标表格** (`metrics_table.png`)
   - 展示每个预测目标的详细指标

3. **PCC 柱状图** (`pcc_barplot.png`)
   - 对比各通路的相关系数

4. **模型参数面板** (`params_panel.png`)
   - 记录训练超参数配置

5. **综合报告** (`full_report.png`)
   - 以上所有内容合成的一页综合报告图

### 手动生成可视化

如果训练中断或需要重新生成可视化：

```python
from visualize_results import generate_full_report

generate_full_report(
    model_name="HisToGene",
    history_csv="histogene/training_history.csv",
    predictions_csv=None,
    output_dir="histogene/results_vis",
    params={
        "batch_size": 64,
        "num_epochs": 150,
        "lr": 1e-4,
        # ... 其他参数
    }
)
```

## 常见问题排查

### Q1: 训练卡住了怎么办？

**检查点：**
1. 查看 GPU 使用情况（Windows 任务管理器或 `nvidia-smi`）
2. 检查数据加载是否阻塞（`num_workers` 设置为 0 可避免 Windows 多进程问题）
3. 查看最近的日志输出，确认当前 epoch

```bash
# Windows 查看 GPU
nvidia-smi
```

### Q2: 如何知道训练是否正常进行？

**判断标准：**
- 每个 epoch 应该有时间戳输出（HisToGene 显示如 `45.2s`）
- 验证集损失在前几个 epoch 应该逐渐下降
- 学习率应该保持合理范围（1e-4 ~ 1e-5）

### Q3: 训练意外中断后如何恢复？

**HisToGene 模型：**
当前版本不支持断点续训，需要从头开始训练。

**UNI2-h+MLP 模型：**
当前版本不支持断点续训，但已保存的最佳模型仍可用。

### Q4: 如何监控 GPU 内存使用？

```python
import torch

if torch.cuda.is_available():
    print(f"GPU 名称: {torch.cuda.get_device_name(0)}")
    print(f"已分配内存: {torch.cuda.memory_allocated(0) / 1024**2:.2f} MB")
    print(f"缓存内存: {torch.cuda.memory_reserved(0) / 1024**2:.2f} MB")
    print(f"总内存: {torch.cuda.get_device_properties(0).total_memory / 1024**2:.2f} MB")
```

### Q5: 训练进度在哪里显示？

训练进度通过以下方式显示：

1. **终端输出**: 每个 epoch 一行，格式为 `Epoch [当前/总数]`
2. **CSV 文件**: 每个 epoch 结束后追加一行记录
3. **早停计数器**: 显示为 `Early stop patience: 当前/最大值`

## 快速检查脚本

创建一个快速检查脚本 `check_training_progress.py`：

```python
"""快速检查训练进度"""
import pandas as pd
from pathlib import Path

def check_histogene_progress():
    """检查 HisToGene 训练进度"""
    history_path = Path("histogene/training_history.csv")
    
    if not history_path.exists():
        print("[INFO] HisToGene 训练历史文件不存在")
        return
    
    history = pd.read_csv(history_path)
    
    print("\n" + "="*60)
    print("HisToGene 训练进度")
    print("="*60)
    print(f"已完成 Epoch: {len(history)}")
    print(f"最新 Epoch: {history['epoch'].iloc[-1]}")
    print(f"\n最新指标:")
    print(f"  Train Loss: {history['train_loss'].iloc[-1]:.4f}")
    print(f"  Val Loss:   {history['val_loss'].iloc[-1]:.4f}")
    print(f"  Val PCC:    {history['val_pcc'].iloc[-1]:.4f}")
    print(f"  Val R²:     {history['val_r2'].iloc[-1]:.4f}")
    
    best_idx = history['val_loss'].idxmin()
    print(f"\n最佳 Epoch: {history.loc[best_idx, 'epoch']}")
    print(f"最佳 Val Loss: {history.loc[best_idx, 'val_loss']:.4f}")
    print(f"最佳 Val PCC:  {history.loc[best_idx, 'val_pcc']:.4f}")

def check_uni2h_progress():
    """检查 UNI2-h+MLP 训练进度"""
    history_path = Path("uni2h_new/checkpoints/HYZ15040/best_model_uni2h.history.csv")
    
    if not history_path.exists():
        print("[INFO] UNI2-h+MLP 训练历史文件不存在")
        return
    
    history = pd.read_csv(history_path)
    
    print("\n" + "="*60)
    print("UNI2-h+MLP 训练进度")
    print("="*60)
    print(f"已完成 Epoch: {len(history)}")
    print(f"最新 Epoch: {history['epoch'].iloc[-1]}")
    print(f"\n最新指标:")
    print(f"  Train Loss: {history['train_loss'].iloc[-1]:.6f}")
    print(f"  Val Loss:   {history['val_loss'].iloc[-1]:.6f}")
    print(f"  Val PCC:    {history['val_pcc'].iloc[-1]:.6f}")
    print(f"  Val R²:     {history['val_r2'].iloc[-1]:.6f}")
    
    best_idx = history['val_loss'].idxmin()
    print(f"\n最佳 Epoch: {history.loc[best_idx, 'epoch']}")
    print(f"最佳 Val Loss: {history.loc[best_idx, 'val_loss']:.6f}")
    print(f"最佳 Val PCC:  {history.loc[best_idx, 'val_pcc']:.6f}")

if __name__ == "__main__":
    check_histogene_progress()
    check_uni2h_progress()
```

使用方法：
```bash
python check_training_progress.py
```

## 总结

| 查看方式 | 适用场景 | 实时性 |
|---------|---------|--------|
| 终端输出 | 训练进行时 | ✅ 实时 |
| 日志文件 | 后台训练 | ✅ 近实时 |
| CSV 历史文件 | 训练后分析 | ✅ 每个 epoch 更新 |
| 可视化图表 | 训练完成后 | ❌ 训练结束生成 |
| 检查点文件 | 查看最佳模型 | ❌ 仅在改善时更新 |

**推荐工作流：**
1. 训练时：观察终端输出，关注验证集损失变化
2. 训练中：定期查看 CSV 文件了解整体趋势
3. 训练后：分析可视化图表，评估模型性能
