# infer.py 深度解读指南

> 本文档面向初学者，详细解读病理图像特征推理与评估脚本的原理与实现。

---

## 1. 文件概述

**一句话说明**：`infer.py` 是一个用于**病理图像多目标回归模型的推理与评估工具**，加载训练好的 UNI2-h 特征提取器 + 回归头模型，对验证集进行批量推理，并计算逐基因集及宏平均评估指标。

**在项目中的角色**：这是模型评估流程的核心环节，位于模型训练（`train.py`）之后，用于评估模型在验证集上的泛化性能，生成预测结果和评估指标报告。

---

## 2. 背景知识

### 2.1 什么是模型推理（Inference）？

模型推理是指使用**训练好的模型**对新数据进行预测的过程。与训练不同，推理阶段：

- **不更新模型参数**：模型权重保持固定
- **前向传播（Forward Pass）**：输入数据经过网络层计算得到输出
- **评估性能**：将预测结果与真实标签对比，计算评估指标

### 2.2 什么是 Checkpoint？

Checkpoint（检查点）是训练过程中保存的模型快照，包含：

| 组件 | 说明 |
|------|------|
| `model_state_dict` | 模型的权重参数（神经网络各层的权重和偏置） |
| `feature_dim` | 输入特征维度（如 1536，来自 UNI2-h） |
| `hidden_dim` | 隐藏层维度（如 512） |
| `dropout` | Dropout 比率（如 0.3） |
| `num_targets` | 预测目标数量（如 8 个基因集） |

**作用**：允许从任意训练阶段恢复模型，或用于推理部署。

### 2.3 什么是特征缓存？

UNI2-h 是大型预训练模型（约 300M 参数），提取特征计算量大。因此采用**缓存策略**：

```
第一次运行：图像 → UNI2-h → 特征向量 → 保存到磁盘（.npy文件）
后续运行：直接从磁盘加载特征向量（跳过UNI2-h前向传播）
```

**优势**：
- 推理速度提升 10 倍以上
- 节省 GPU 显存
- 支持快速实验迭代

### 2.4 评估指标简介

| 指标 | 全称 | 含义 | 理想值 |
|------|------|------|--------|
| MSE | Mean Squared Error | 均方误差 | 0 |
| MAE | Mean Absolute Error | 平均绝对误差 | 0 |
| R² | Coefficient of Determination | 决定系数 | 1 |
| PCC | Pearson Correlation Coefficient | 皮尔逊相关系数 | ±1（越接近1越好） |

### 4.6 批量推理（行 102-115）

**功能说明**：
遍历验证集，执行前向传播，收集所有预测结果和真实标签。

**执行流程**：

```
初始化 all_true = [], all_pred = []
for each batch in DataLoader:
    1. 将 features 移动到 device
    2. model(features) → predictions
    3. 将结果移回 CPU，转换为 numpy
    4. 追加到 all_true, all_pred
合并所有批次结果 → y_true_all, y_pred_all
```

**关键代码行**：

| 行号 | 代码 | 作用 |
|------|------|------|
| 107 | `with torch.no_grad():` | 禁用梯度计算，节省内存，加速推理 |
| 110 | `preds = model(features).detach().cpu().numpy()` | 前向传播并转换格式 |
| 114-115 | `np.concatenate(...)` | 合并所有批次的结果 |

**初学者注意**：
- `torch.no_grad()`：推理时必须使用，否则 PyTorch 会构建计算图，浪费内存
- `.detach()`：将张量从计算图中分离，允许转换为 numpy

### 4.8 宏平均计算（行 141-150）

**功能说明**：
计算所有基因集指标的宏平均（Macro-average）。

**关键代码行**：

| 行号 | 代码 | 作用 |
|------|------|------|
| 145-148 | `np.nanmean([row["mse"] for row in per_target_rows])` | 对每个指标取平均 |
| 143 | `scope="macro_avg"` | 标记这是宏平均行 |
| 144 | `target=f"ALL_{num_targets}_TARGETS"` | 标记包含的目标数 |

**宏平均 vs 微平均**：

| 类型 | 计算方式 | 适用场景 |
|------|----------|----------|
| 宏平均 | 每个类别指标的平均 | 类别平衡，关注每个类别表现 |
| 微平均 | 总体样本的指标 | 类别不平衡，关注总体表现 |

本项目使用**宏平均**，因为每个基因集的重要性相当。

## 6. 评估指标详解

### 6.1 MSE（Mean Squared Error，均方误差）

**数学公式**：
```
MSE = (1/n) × Σ(y_true - y_pred)²
```

**含义**：
- 预测值与真实值差值的平方的平均
- 对大误差惩罚更重（平方效应）

**判断标准**：
- MSE = 0：完美预测
- MSE 越小越好
- 与数据尺度相关，不同数据集间不可直接比较

### 6.2 MAE（Mean Absolute Error，平均绝对误差）

**数学公式**：
```
MAE = (1/n) × Σ|y_true - y_pred|
```

**含义**：
- 预测值与真实值绝对差值的平均
- 对大误差的惩罚比 MSE 轻

**判断标准**：
- MAE = 0：完美预测
- MAE 的单位与原始数据相同，更直观

### 6.3 R²（Coefficient of Determination，决定系数）

**数学公式**：
```
R² = 1 - (SS_res / SS_tot)
   = 1 - [Σ(y_true - y_pred)² / Σ(y_true - y_mean)²]
```

**含义**：
- 模型解释数据方差的比例
- R² = 1：模型完美预测所有方差
- R² = 0：模型预测等同于简单取均值
- R² < 0：模型比简单取均值还差

**判断标准**：
- R² > 0.7：模型解释力强
- 0.5 < R² < 0.7：中等解释力
- R² < 0.5：解释力较弱

### 6.4 PCC（Pearson Correlation Coefficient，皮尔逊相关系数）

**数学公式**：
```
PCC = cov(y_true, y_pred) / (σ_y_true × σ_y_pred)
```

**含义**：
- 衡量预测值与真实值的线性相关程度
- 取值范围 [-1, 1]

**判断标准**：
- PCC = 1：完全正相关（预测随真实值线性增长）
- PCC = -1：完全负相关
- PCC = 0：无线性相关
- |PCC| > 0.7：强相关
- 0.5 < |PCC| < 0.7：中等相关

### 6.5 指标组合解读

**场景 1：R² 高，PCC 高**
- 模型拟合效果好，预测准确

**场景 2：R² 低，PCC 高**
- 模型能捕捉趋势，但存在系统性偏差（可能需要校准）

**场景 3：R² 高，PCC 低**
- 罕见，通常表示计算错误

**场景 4：MSE >> MAE**
- 存在异常值或大误差样本

| `--batch_size` | 命令行 | 256 | 推理批次大小 | GPU 显存大时可增大 |
| `--num_workers` | 命令行 | 0 | 数据加载进程数 | 可设为 4-8 加速加载 |
| `--rebuild_cache` | 命令行 | False | 强制重建缓存 | 修改特征提取方式时启用 |
| `--target_start_col` | 命令行 | 1 | 标签起始列 | 与训练时保持一致 |
| `--num_targets` | 命令行 | 8 | 预测目标数 | 与训练时保持一致 |

## 9. 初学者注意事项

### 9.1 常见错误

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| `FileNotFoundError: checkpoint` | 模型路径错误 | 检查 `--checkpoint_path` 路径 |
| `RuntimeError: CUDA out of memory` | 批次过大 | 减小 `--batch_size` |
| `KeyError: 'feature_dim'` | checkpoint 损坏或格式不符 | 使用正确的 checkpoint 文件 |
| 指标异常（R² 为负） | 模型未训练好或数据不匹配 | 检查是否加载了正确的模型和标签 |
| 预测值全为 0 | 模型权重未正确加载 | 检查 `load_state_dict` 是否成功 |

### 9.2 调试建议

**1. 检查 checkpoint 内容**：
```python
checkpoint = torch.load("best_model.pth", map_location="cpu")
print(checkpoint.keys())  # 查看包含的键
print(f"feature_dim: {checkpoint['feature_dim']}")
print(f"num_targets: {checkpoint['num_targets']}")
```

**2. 验证数据加载**：
```python
# 在 main() 中添加调试代码
for features, targets in loader:
    print(f"Features shape: {features.shape}")  # 应为 (batch_size, 1536)
    print(f"Targets shape: {targets.shape}")    # 应为 (batch_size, 8)
    break
```

**3. 检查预测结果范围**：
```python
print(f"True range: [{y_true_all.min():.3f}, {y_true_all.max():.3f}]")
print(f"Pred range: [{y_pred_all.min():.3f}, {y_pred_all.max():.3f}]")
```

### 9.3 重要提醒

- ⚠️ **路径格式**：Windows 路径建议使用原始字符串 `r"C:\path\to\dir"`
- ⚠️ **设备一致性**：确保模型和数据在同一设备（CPU/GPU）上
- ⚠️ **标签一致性**：推理时必须使用与训练时相同的标签文件
- ⚠️ **缓存复用**：首次运行会较慢（提取特征），后续运行会快很多

## 11. 总结

`infer.py` 是一个完整的模型推理与评估脚本，其核心贡献是：

1. **模型恢复**：从 checkpoint 完整重建模型结构和权重
2. **特征缓存**：避免重复提取 UNI2-h 特征，大幅提升效率

- PyTorch 模型推理流程（eval + no_grad）

- 特征缓存的设计思想
- 回归任务的评估指标体系
- pandas 在结果处理中的应用
