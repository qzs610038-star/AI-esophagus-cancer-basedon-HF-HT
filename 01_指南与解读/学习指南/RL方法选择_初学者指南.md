# RL 方法选择：初学者指南

> 面向 PFMval 项目的 RL 方法选型参考。假设读者有 ML 基础但无 RL 背景。

---

## 1. RL 方法的两个分类维度

所有 RL 方法可以沿两个正交维度分类：

### 维度一：Agent 学什么？

| 类型 | 学的东西 | 直觉类比 |
|------|------|------|
| **Value-based** | 每个状态/动作"值多少钱"（Q 值） | 你不需要知道"该点什么菜"，只需要每道菜的评分，每次点评分最高的 |
| **Policy-based** | 在每个状态直接输出"该做什么"（策略） | 学打篮球——投进了这个位置下次多投，不分析"这个位置投篮价值 3.2 分" |

### 维度二：Agent 怎么学？

| 类型 | 数据来源 | 直觉类比 |
|------|------|------|
| **On-policy** | 只能用自己刚产生的经验 | 边开车边学——只能从自己刚才的驾驶中学习，别人的经验用不了 |
| **Off-policy** | 可以从别人的/历史的经验中学 | 看驾驶录像学——可以反复看别人的事故录像，不需要自己撞车 |

### 交叉产生四大流派

```
                  On-policy              Off-policy
           ┌─────────────────────┬─────────────────────┐
Value-based│   （较少使用）        │  DQN 家族           │
           │                     │  - 离散动作首选       │
           │                     │  - 可复用历史数据     │
           ├─────────────────────┼─────────────────────┤
Policy-based│  Policy Gradient    │  （较少使用）        │
           │  - REINFORCE        │                     │
           │  - PPO              │                     │
           │  - 连续动作 + 在线   │                     │
           └─────────────────────┴─────────────────────┘
```

**不基于梯度的黑箱优化方法**构成了第五个流派（见第 3 节），它们在很多场景下比 RL 更合适。

## 2. 四大方法家族详解

### 2.1 DQN 家族（Value-based + Off-policy）

**核心机制**：训练一个 Q 网络，输入状态 S 和动作 A，输出"在 S 做 A 能拿多少累积奖励"。

**关键创新——经验回放（Experience Replay）**：
- 把每次交互（S, A, R, S'）存入一个 replay buffer
- 训练时随机抽取一批经验更新 Q 网络
- **这打破了时间相关性**——同一个早上和晚上的经验可以混在一起学
- 也是 Q 系列能做 off-policy 的根本原因

**关键创新——目标网络（Target Network）**：
- 维护两个 Q 网络：一个用于选动作（在线网络），一个用于算目标值（目标网络）
- 目标网络每隔 N 步才从在线网络拷贝一次参数
- **防止"追自己尾巴"**——如果目标和预测用同一个网络，训练会震荡发散

| 方法 | 改进点 | 何时用 |
|------|------|------|
| **DQN** | 基础版：经验回放 + 目标网络 | 入门首选 |
| **Double DQN** | 修复 DQN 系统性地高估 Q 值的问题 | Q 值估计不稳定时 |
| **Dueling DQN** | 分别评估"状态本身好不好"和"这个动作好不好" | 动作多但很多动作对结果影响不大时 |
| **Rainbow** | 上述改进的全家桶 | 追求最优性能，不关心复杂度时 |

**适用条件**：
- 动作空间是离散的（有限的几个选项）
- 状态空间不太大（几维到几百维）
- 有历史数据可以复用，或评估成本低可以大量探索

### 2.2 Policy Gradient 家族（Policy-based + On-policy）

**核心机制**：直接优化策略网络 π(a|s)（输入状态，输出每个动作的概率）。好结果 → 提高该动作概率；坏结果 → 降低概率。

**REINFORCE（最基础的策略梯度）**：
```
对每个 episode 的每一步：
  如果最终结果好 → 把这步的动作概率向"更确定"方向更新
  如果最终结果差 → 把这步的动作概率向"更不确定"方向更新
```

**PPO（Proximal Policy Optimization）**：
- REINFORCE 的核心问题是：一次更新太大，策略突变，之前收集的经验全作废
- PPO 的解决方案：每次更新"悠着点"——裁剪更新幅度，不让新老策略差太远
- **这是 RLHF（ChatGPT 用的 RL）的核心方法**

| 方法 | 改进点 | 何时用 |
|------|------|------|
| **REINFORCE** | 最朴素版本，蒙特卡洛估计梯度 | 入门理解，极简场景 |
| **PPO** | 裁剪策略更新幅度，稳定训练 | 连续动作 + 在线交互 + 需要稳定训练 |
| **A2C/A3C** | 引入 Critic 评估"这一步比预期好多少"，降低方差 | 需要更快收敛，可用多核并行 |

**适用条件**：
- 动作空间是连续的（如调整力度 0~1），或离散但需要概率输出
- 必须在在线环境中交互——每次更新后旧数据作废
- 评估成本不能太高（否则 on-policy 的数据效率太低）

### 2.3 Actor-Critic 混合家族

**核心机制**：Actor（策略网络）负责"做决定"，Critic（价值网络）负责"评好坏"。Actor 从 Critic 的评分中学习，Critic 从实际结果中更新。

```
类比：演员 + 导演
  - Actor（演员）：看到舞台（状态），决定做什么动作
  - Critic（导演）：评价"这个动作在这个场景下好/差"
  - 演员根据导演反馈调整表演
  - 导演根据观众反应（实际 reward）调整评判标准
```

| 方法 | 一句话 | 何时用 |
|------|------|------|
| **SAC** | 最大化 reward + 最大化动作熵（鼓励探索） | 连续控制 + 需要鲁棒性 |
| **TD3** | DDPG 改良版，用双 Q 网络防高估 | 连续动作 + 确定性策略 |

**适用条件**：连续动作 + 在线交互。比纯 Policy Gradient 更样本高效，比纯 Value-based 更灵活。

### 2.4 黑箱优化方法（非 RL，但常是更优解）

**核心直觉**：如果问题不需要"序贯决策"（后一步不依赖前一步），那根本不需要 RL。直接用黑箱优化更高效。

```
问题判断题：需要序贯决策吗？
  ├── 是 → 用 RL (DQN/PPO/SAC)
  └── 否 → 用黑箱优化 (BO/遗传算法)
```

| 方法 | 一句话 | 何时用 |
|------|------|------|
| **Bayesian Optimization** | 用已知评估结果建一个"猜测函数"（高斯过程），选最有希望的点去试 | 评估有成本、reward 有噪声、想量化不确定性 |
| **遗传算法** | 好的组合"繁殖"（交叉），差的"淘汰"，随机"变异" | 不需要梯度、可高度并行、组合空间大 |
| **Simulated Annealing** | 早期大跨步探索（接受坏的），后期小步精调（只接受好的） | 有局部最优陷阱 |
| **Thompson Sampling** | 每次按"可能是最优的概率"随机选一个动作 | 多臂老虎机（单步决策，无状态转移） |
| **CMA-ES** | 维护搜索分布的均值和方差，逐步收缩到最优 | 连续参数优化（如调超参） |

## 4. 给初学者的 5 条核心记忆

### 4.1 DQN 学的是"评分"，Policy Gradient 学的是"习惯"

- DQN：问"在这个状态下做 A 能得多少分？"→ 选分最高的
- PG：不问分，直接养成"看到 S 就做 A"的习惯 → 结果好了就强化这个习惯

### 4.2 On-policy vs Off-policy 是工程上最重要的区别

- **On-policy（PPO/REINFORCE）**：每次更新策略后，之前收集的所有经验都作废。在 GPU 昂贵的场景（训练一次 3 小时），这是致命缺陷。
- **Off-policy（DQN）**：经验可以积攒在 replay buffer 里反复用。可以用历史日志预训练，也可以用一份数据训练很多轮。

**经验法则**：如果你的环境"跑一次很贵"，优先考虑 off-policy 方法。

### 4.3 不是所有优化问题都需要 RL

如果问题没有"时序依赖"（后一步的结果不取决于前一步），RL 是舍近求远。用 Bayesian Optimization 或遗传算法通常更简单、更高效。

### 4.4 离散动作 → DQN，连续动作 → PPO/SAC

这是一个快速判断的捷径：
- 动作是"选 A/B/C" → DQN
- 动作是"调到 0.37" → PPO 或 SAC

### 4.5 噪声大的 reward 需要"会建模不确定性"的方法

- DQN 用经验回放平滑噪声（多次采样求平均）
- BO 用高斯过程显式建模"我不确定这里好不好"
- PPO 对噪声 reward 比较敏感——因为它 on-policy，没有 replay buffer 来平滑

## 5. PFMval 项目中的应用对照

| 方案 | 问题类型 | 推荐方法 | 选择理由 |
|------|------|------|------|
| **B：RL 训练调度** | 序贯决策（epoch→epoch 状态转移） | DQN | 离散动作 + off-policy 可用历史日志 + 噪声 reward 需 replay buffer 平滑 |
| **D：RL 通路选择** | 单步决策（选子集→评 AUC） | Bayesian Optimization | 非序贯不需要 RL + 高斯过程建模噪声 + 可解释通路重要性 |
| **超参调优** | 单步决策（选超参→训完看结果） | BO 或 CMA-ES | 评估贵、需高效采样 |
| **Patch 筛选** | 序贯决策（一张张选 patch） | DQN | 离散动作 + 状态转移 |

## 6. 延伸阅读建议

- **DQN 原始论文**：Mnih et al., 2015, "Human-level control through deep reinforcement learning"（Nature）——讲清了经验回放和目标网络
- **PPO 原始论文**：Schulman et al., 2017, "Proximal Policy Optimization Algorithms"——讲清了为什么要裁剪更新幅度
- **BO 综述**：Shahriari et al., 2016, "Taking the Human Out of the Loop: A Review of Bayesian Optimization"——讲清了高斯过程和采集函数
- **RL 入门课程**：Spinning Up (OpenAI) ——实战导向的 RL 教程
