## 一、先理解：我们在解决一个什么问题？

### 1.1 一句话版本

**从一张食管癌 H&E 病理切片图片，预测 30 条基因通路的活性评分。**

- 输入：病理切片（H&E 染色的组织图像，看起来是粉紫色的一片）
- 输出：30 个数字，每个代表一条通路（如"细胞外基质""干扰素信号"）在该组织区域的活跃程度
- 训练数据：3 个患者的病理切片 + 对应位置的空间转录组测序数据（告诉模型每个位置的真实通路活性是多少）

### 1.2 关键概念速览

| 概念 | 通俗理解 |
|------|---------|
| **H&E 染色** | 病理科的常规染色方法，细胞核染成紫色，细胞质染成粉色 |
| **空间转录组 (ST)** | 在组织切片上测基因表达，保留空间位置信息——知道"哪里"表达了"什么基因" |
| **ssGSEA 通路评分** | 把几十到上百个相关基因的表达值综合成一个"通路活性"数字 |
| **UNI2-h** | 一个预训练的病理图像 AI 模型（病理基础模型/PFM），基于 ViT 架构，`img_size=224, patch_size=14` |
| **Token 序列** | UNI2-h 内部将输入图片切分为 16×16=256 个小块，每个小块变成一个 token 向量（详见下方图解） |
| **跨患者泛化** | 用患者 A、B 的数据训练，去预测患者 C——这是本项目最大的难点 |
| **PCC** | Pearson 相关系数，衡量预测值和真值的线性相关程度，范围 [-1, 1]，越大越好 |
| **过拟合 Gap** | 训练集 PCC - 验证集 PCC，越小说明模型泛化越好 |

#### Token 序列图解：两级分块机制

本项目的数据流存在**两级分块**，很容易混淆：

```
═══════════════════════════════════════════════════════════════════════════
                        整张 WSI 全切片图像
                   （例如 100,000 × 80,000 像素）
                                  │
        ┌─────────────────────────┼─────────────────────────┐
        │                         │                         │
        ▼                         ▼                         ▼
   小 patch A                 小 patch B              小 patch C
 (224×224 px, .png)        (224×224 px, .png)       (224×224 px, .png)
        │                         │                         │
        └─────────────────────────┼─────────────────────────┘
                                  │
            ★ 第一级分块 ★        │  数据预处理阶段完成
           （patch_noov_spilt/ 目录下的 .png 文件）
                                  │
        ┌─────────────────────────▼─────────────────────────┐
        │              每个小 patch 被送入 UNI2-h              │
        │        UNI2-h 内部 ViT 将其进一步切分为              │
        │        patch_size=14 → 224/14=16 → 16×16=256 个 patch│
        │        + 1 个 CLS token + 8 个 register token       │
        │        = 265 个 token，每个 token 是 1536 维向量     │
        │                                                    │
        │            ★ 第二级分块 ★  UNI2-h 内部完成           │
        └─────────────────────────┬──────────────────────────┘
                                  │
                                  ▼
                    每个小 patch → [265, 1536] token 序列
                    保存为 uni2h_cache_tokens/{patient}/{split}/xxx.pt
```

**关键结论**：

- 我们提取的 token 序列是**每个小 patch 图片被 UNI2-h 内部进一步切分后产生的**，不是整张 WSI 的 token
- 当前配置：UNI2-h `img_size=224, patch_size=14` → 256 个 patch token + 1 CLS + 8 reg = **265 tokens**
- 为什么用 lite 模式（65 tokens = 1 CLS + 前 64 个 patch token）？完整 265 tokens × 1536 维 × 4 bytes ≈ 1.6 MB/样本。3 个患者 ~2 万样本 → ~32 GB，lite 模式减到 ~8 GB，同时保留足够空间信息
- 这意味着模型可以看到**一个 patch 内部的精细空间结构**（如肿瘤细胞团 vs 间质区域在同一个 224×224 视野内的分布），而不是只看一个全局平均

### 1.3 关键架构决策：深化 intra-patch token 还是改用 inter-patch 空间图？

理解了上面的两级分块之后，自然会产生一个问题：**我们应该继续深化小 patch 内部的 token 编码（intra-patch），还是改为建模各 patch 之间的空间关系（inter-patch）？**

**答案：intra-patch 深化收益远大于 inter-patch 图建模。已有实验给出了非常强的一致性证据。**

#### 已有实验证据

| 实验 | 空间建模方式 | 跨患者 PCC | vs 纯坐标嵌入基线 |
|------|------------|-----------|:---:|
| **方案B** (Token + 坐标嵌入) | 仅 `x_embed + y_embed`（弱空间感知） | **0.3812** | — 基线 |
| **方案C** (Token + GAT) | 坐标嵌入 + KNN空间图 + **GAT注意力消息传递** | 0.4068 (单折) | **−0.27%** |
| **EGN-v2+UNI** | 坐标嵌入 + 半径空间图 + **GraphSAGE消息传递** | 0.3804 | ≈ 0 |

> 两种不同的图神经网络（GAT 和 GraphSAGE），在两个不同的下游架构上，都加上了显式的跨 patch 空间消息传递——结果都是零或负提升。这不是某一个实验的偶然，而是两次独立的交叉验证。

#### 为什么会这样？

**1. 生物学尺度不匹配**

每个 224×224 patch 对应 ~50μm 的真实组织区域，已经包含了数十到数百个细胞。通路活性（如 ECM 合成、细胞增殖、免疫浸润）主要由**这个 patch 内部的细胞类型组成和数量**决定。相邻 patch 的细胞组成可能完全不同（比如肿瘤细胞团紧挨着纤维间质），所以"邻居的通路活性"对预测当前 patch 的帮助很有限。

**2. 坐标嵌入已经提供了足够的大尺度空间上下文**

`x_embed + y_embed` 虽然不显式地在 patch 之间传递消息，但它告诉模型"你在组织的哪个宏观位置"。肿瘤中心和侵袭边缘天然有不同的基线通路活性分布，坐标嵌入已经捕获了这种百微米到毫米级别的空间趋势。再叠加精细的逐对 patch 消息传递，提供的是冗余信息。

**3. 跨患者泛化场景下图结构反而是负担**

空间图（KNN/半径图）的拓扑完全取决于 patch 的物理坐标——而不同患者的组织形状、大小、切面角度都不同。训练时在患者 A+B 上学到的图消息传递模式，到患者 C 上完全不适用。显式的图消息传递在跨患者场景下容易过拟合到训练患者的特定空间布局，反而损害泛化。

#### 这意味着什么

后续改进的优先级应该是：

```
✅ 继续深化的方向（intra-patch）：
   → P1-2 编码器加深（让 265 个 token 之间做更深的信息交互）
   → P2-1 UNI LoRA 微调（让 token 本身质量更高）
   → 未来可能的 token 级对比学习

❌ 搁置的方向（inter-patch 图建模）：
   → GAT 图注意力（已实验验证无效）
   → GraphSAGE 空间聚合（EGN-v2 已验证无效）
   → 多尺度图 / 混合图（理论可能有用，但前两个实验的负结果降低了优先级）
```

### 1.4 项目的核心矛盾

```
单患者内预测：PCC ≈ 0.55~0.61 —— 说明方向可行
跨患者预测：PCC ≈ 0.38~0.41 —— 骤降 0.15~0.20

问题不在"能不能预测"，而在"能不能跨患者泛化"。
```

**为什么跨患者这么难？** 三个患者的切片来自不同医院、不同扫描仪、不同染色条件，图像的颜色分布、组织形态都有差异。模型在患者 A 身上学到的"规律"，拿到患者 B 身上可能对不上——这就是**域偏移（Domain Shift）**。

## 二、逐项原因分析

### P0-b：MixUp α 超参扫描

**是什么**：MixUp 在训练时把两个样本的特征按 α 比例混合（如 0.3×样本A + 0.7×样本B），标签也按相同比例混合。α 控制混合强度——α 越小混合越"轻柔"（倾向于接近某一个样本），α 越大混合越"激进"（两个样本各占约一半）。

**已有实验事实**：
- 无 MixUp：Best PCC = 0.3994，Best Epoch = 4
- MixUp α=1.0：Best PCC = 0.4172，Best Epoch = 8

**为什么要扫 α？**

关键洞察来自一个有趣的实验现象：MixUp 并没有降低过拟合 Gap（两个实验的 Gap 都是 ~0.16），但它把最优 epoch 从 4 推到了 8。这说明 MixUp 的作用机制不是"抑制过拟合"，而是——

> **MixUp 创建了"中间患者"的虚拟样本，让模型在更长时间内持续学到有用信息，而不是在第 4 个 epoch 就学不动了。**

不同 α 值决定了"中间患者"离真实患者的距离：
- α 太小（0.1~0.2）：混合太轻柔，几乎等于没混合 → 接近无增强基线
- α 适中（0.4~0.8）：可能恰好填补了患者之间的特征空间空白
- α 太大（>2.0）：两个样本各占 ~50%，标签被过度平滑 → 可能丢失判别信息

扫 α 的花费几乎为零（零代码改动，纯改命令行参数），但有机会找到比 0.4172 更优的配置。

---

### P1-1：通路分组加权损失

**是什么**：让模型在训练时对"难预测的通路"给予更大的惩罚权重，强迫模型多关注它们。

**已有实验事实**：
- 容易的通路：ECM_Organization (PCC=0.775)、Coagulation (0.742)、Wound_Healing (0.715)
- 困难的通路：Interferon_Alpha (PCC=0.012)、Interferon_Gamma (0.087)、Inflammatory_Response (0.169)

**为什么这么"偏科"？**

这 30 条通路可以理解为 30 个独立的预测任务，它们的难度天然不同：

| 通路类型 | 为什么好/难预测 | 例子 |
|---------|---------------|------|
| 结构/基质类 | H&E 图像直接反映 ECM 密度、纤维化程度等形态特征 | ECM_Organization (0.775) |
| 细胞周期类 | 有丝分裂、细胞密度在 H&E 图像中有直接的形态对应 | G2M_Checkpoint (0.540) |
| **免疫/炎症类** | 由可溶性细胞因子驱动（干扰素、白介素），不直接改变细胞形态 | Interferon_Alpha (0.012) |

> **核心矛盾：模型的总损失 = 30 条通路的平均。当一个"简单"通路已经学得很好（loss 很小），即使"困难"通路还完全没学会（loss 很大），总损失也不会高——因为简单通路数量多，拉低了平均。模型没有动力去攻克困难通路。**

**解决方案的直觉**：给困难通路更大的"音量"。比如 Interferon_Alpha 的 loss 乘以 2.0（权重上限），让模型在优化时不得不关注它。同时设安全阀（权重 ∈ [0.5, 2.0]），防止某一个异常通路主导训练。

**科学依据**：Kendall et al. (CVPR 2018) 的多任务学习理论指出，当多个任务的难度不同时，等权损失会导致模型偏向简单任务。这与本项目观察到的现象完全吻合。

---

### P1-2：编码器深度增加（n_layers 1→2）

**是什么**：当前 Token 编码器只用了 1 层 Transformer，把它增加到 2 层。

**先明确作用域**：回顾上面的两级分块图解——Transformer 编码器处理的是**一个小 patch 内部的 265 个 token**（不是跨 patch 的全局 token）。每个 token 对应这个 224×224 小视野内的一个 14×14 像素区域。

**为什么 1 层不够？**

Transformer 的核心能力是**让 token 之间互相"交流"**。每个 token 代表小 patch 内部的一个 14×14 像素局部区域：

- 1 层 Transformer：每个 token 只能和其他 token 直接交互一次。信息流动范围 = 单跳。
- 2 层 Transformer：交互两次。token A 先和 token B 交流，B 再和 C 交流——A 间接"看到"了 C。信息流动范围 = 两跳。

**这对本项目意味着什么？**

在单个 224×224 小 patch 内部（比如 ~50μm × 50μm 的真实组织区域），可能同时包含：

1. 肿瘤细胞团的 token — 核大深染、排列紊乱
2. 免疫浸润的 token — 小淋巴细胞散在分布
3. 纤维间质的 token — 粉染的胶原束

1 层 Transformer 只能捕获 token 对之间的直接关系（"这两个相邻的 14×14 像素区域外观不同"）。2 层 Transformer 可以捕获更高阶的空间关系——通过中间 token 的中继，模型可以感知到"肿瘤区域被免疫细胞包围"或"肿瘤细胞团与纤维间质交错分布"这类**微环境空间组织模式**。免疫通路（如炎症反应、干扰素信号）的预测特别依赖这种微环境级别的空间上下文。

**为什么要同步降学习率？** 更多层 = 更多参数 = 更复杂的优化景观 = 更容易震荡。降 lr 到 3e-05（比基线的 5e-05 低）是预防性措施。

### P1-3：染色归一化（Stain Normalization）

**是什么**：用一种算法（Macenko/Vahadane）把三个患者不同颜色的 H&E 图像"翻译"成统一的颜色风格，再喂给 UNI 提取特征。

**为什么需要？**

三个患者的病理切片来自不同的医院和时间，染色效果不同：
- 患者 HYZ 的可能偏蓝（苏木精过染）
- 患者 JFX 的可能偏红（伊红过染）
- 患者 LMZ 的可能偏淡（染色不足）

UNI 虽然有一定的颜色不变性，但它是通用的病理模型，并非专门训练来对抗这种染色差异。当我们用 JFX+LMZ（一种颜色风格）训练、在 HYZ（另一种颜色风格）上测试时，模型可能被颜色差异误导。

**为什么排在 P1-3 而不是更靠前？**

因为 P0-b 的 MixUp α 优化和 P1-1 的通路加权都是零/低磁盘成本的改动。染色归一化需要处理 ~3 万张图片 + 重新提取 UNI 特征 + 新增 27GB 缓存，时间成本 4-6 小时。先跑完低成本实验，确认有继续优化的空间，再投入高成本任务——这是工程上的风险控制。

**与 HED 增强的区别**：HED 增强是"故意扰动颜色来增加训练数据的多样性"（已证明无效），染色归一化是"消除颜色差异来减少域偏移"（理论上互补）。前者增加方差，后者减少偏差。

### P2-1：UNI LoRA 微调（最大突破潜力）

**是什么**：不再把 UNI 当"冻住的黑箱"只提取特征，而是用 LoRA（Low-Rank Adaptation）技术在项目数据上微调 UNI 的最后几层。

**为什么这是"最大单项潜力"？**

三个不同架构的模型全部收敛到 PCC ≈ 0.37~0.41：

| 模型 | 架构差异 | 跨患者 PCC |
|------|---------|-----------|
| HisToGene-UNI Token | Transformer 编码器 | 0.381 |
| HisToGene-UNI Pooled | 简单池化 | 0.392 |
| EGN-v2+UNI | GNN | 0.380 |

> **改了三种完全不同的下游架构，PCC 都在 0.38 左右——这说明瓶颈不在下游架构，而在 UNI 特征的表达能力。UNI 被冻结，能提供的信息已经到顶了。**

LoRA 微调相当于"让 UNI 学习看食管癌病理的特定模式"——比如食管癌特有的免疫浸润形态、肿瘤-间质边界特征等。UNI 在预训练时见过很多器官，但它从未被专门训练去关注食管癌的病理特征。

**为什么排在 P2 而不是 P1？**

工程难度是最大的障碍：
1. 当前流程是离线提取特征（一次提完，存成 .pt 文件，训练时直接加载）→ 快且省显存
2. LoRA 微调必须在训练时加载整个 UNI2-h（307M 参数）→ 实时前向传播 → 8GB 显存几乎不够
3. 需要重构整个训练循环 + batch_size 降到 1 + gradient accumulation
4. 训练时间从 15 分钟暴涨到 1-2 小时/折

先做完 P1 的低成本优化，确保简单方法已经穷尽，再投入这个高风险高回报的方向。

---

### P2-2：基因通路多模态融合（GenePT 简化版）

**是什么**：利用每条通路的基因成员列表（如"Interferon_Alpha 通路包含 IFNA1, IFNA2, IFNB1..."），把基因名称编码成向量，和图像特征融合。

**为什么图像不够？**

回到那个核心观察：Interferon_Alpha 通路的 PCC 只有 0.012。干扰素是可溶性细胞因子——它的产生和释放不会在 H&E 图像上留下直接的形态痕迹。无论你怎么优化图像编码器，一个在细胞形态上没有反映的生物过程，纯靠图像就是预测不出来的。

但基因序列信息可以"提示"模型：这 30 条通路不是 30 个任意的数字，它们之间有已知的生物学关系——有些基因同时参与多条通路，有些通路在功能上协同作用。把这种先验知识注入模型，相当于告诉模型"你在预测的不是 30 个独立任务，而是有结构的通路网络"。

**为什么用简化版（阶段 A）？** 完整的多模态框架（交叉注意力 + 对比学习）需要大量工程工作。阶段 A 只是把基因嵌入和图像特征拼在一起，成本低，先验证方向对不对。

---

### P2-3：Virchow2 替代 UNI

**是什么**：把特征提取器从 UNI2-h 换成 Virchow2（更大、在更多数据上预训练的病理基础模型）。

**逻辑链**：

1. UNI 在 ~10 万张 WSIs 上预训练，Virchow2 在 310 万张 WSIs 上预训练（数据量 30 倍）
2. UNI 是 307M 参数，Virchow2 是 632M 参数（容量 2 倍）
3. OmiCLIP (Nature Methods 2025) 等工作的系统比较表明：更大规模的病理基础模型一致性地带来更好的空间转录组预测性能
4. 我们已有的证据：UNI 特征在三个不同下游架构上 PCC 都收敛到 ~0.38 → 特征是瓶颈 → 更强的特征提取器可能突破这个天花板

**为什么不确定性较高？** Virchow2 在食管癌数据上的表现没有直接证据。更大的模型不一定在所有任务上都更好。但鉴于替换成本较低（不需要改训练代码，只换特征文件），排在 P2 作为备选方向。

### 已排除方向的原因速查

| 方向 | 为什么排除 |
|------|-----------|
| H&E 颜色增强 | Phase 0 实验证明无效：UNI 已有足够颜色不变性，额外扰动没有帮助 |
| H&E+MixUp 联合 | 两者产生负向交互——MixUp 单独更好，联合反而变差 |
| 对比学习预训练 | ~3 万张 patch 不够。SimCLR/BYOL 通常需要 10 万+ 样本才能学到有用的表示 |
| CutMix on Token 序列 | 理论上有帮助但不确定性高，不如先做完确定性强的高优先级任务 |
| 模型集成 | 三个模型 PCC 都 ~0.38，集成低相关模型才有收益——但这三个高度相关（都用同一 UNI 特征） |

## 三、推荐阅读论文

以下 2 篇论文深度契合本项目的核心问题。与 5.13 文档推荐的 PEaRL（侧重通路对比学习）、FmH2ST（侧重基因序列融合）不同，这两篇侧重**方法基础**和**系统比较**——更适合初学者建立整体认知框架。

### 第 1 推荐：HisToGene（项目基线方法）

> **Pang M, et al. "HisToGene: Histology-Based Prediction of Spatial Transcriptomics Gene Expression Using Vision Transformer." *Nature Communications*, 2025.**

**为什么这是第一推荐**

你就是基于 HisToGene 在做——理解它的每一个设计决策，才能判断哪些值得改、哪些不能动。

**论文的核心设计（映射到本项目）：**

| 论文设计 | 本项目对应 | 为什么这样设计 |
|---------|-----------|--------------|
| ViT 提取 patch 特征 | 替换为 UNI2-h（更强的预训练特征） | 原版 ViT 在 ImageNet 上预训练，不懂病理 |
| 空间坐标编码 (x, y embedding) | 保留，`x_embed` + `y_embed` | 同一个基因在组织不同位置表达不同——坐标是重要的上下文 |
| MLP 回归头 | 保留，`head` | 每个 patch 独立预测通路评分，不需要空间平滑 |
| 全 slide 级别训练 | 改为 patch 级别（每个 patch 有对应 ST 点） | 提高分辨率：从"整张片子一个预测"到"几十个 patch 拼成空间热图" |

**阅读重点**：
1. **方法部分（Section 2）**：理解 ViT → 坐标嵌入 → 回归头的完整数据流
2. **空间坐标编码的设计动机**：为什么简单地把 x、y 作为 embedding 加进去有用？（因为相邻的 patch 倾向于有相似的通路活性——空间自相关性是生物学的基本规律）
3. **TCGA 预训练策略**：HisToGene 如何利用 TCGA 的大规模数据做预训练，再迁移到空间转录组数据——这个"预训练→微调"的思路与你项目中"跨患者泛化"的瓶颈直接相关

**阅读后你应该能回答**：
- HisToGene 的哪些设计是针对空间转录组任务定制的，哪些是通用的？
- 如果我们把 ViT 换成 UNI（已做），核心设计（坐标编码、回归头）是否还需要调整？
- TCGA 预训练用了多少数据？我们 3 个患者的数据量与之相比差了几个数量级？这解释了为什么跨患者泛化这么难吗？

### 第 2 推荐：OmiCLIP / Loki（系统比较 + 验证框架）

> **Chen W, Zhang P, Tran TN, et al. "A visual–omics foundation model to bridge histopathology with spatial transcriptomics." *Nature Methods*, Volume 22, pages 1568–1582 (July 2025).**
>
> DOI: [10.1038/s41592-025-02707-1](https://doi.org/10.1038/s41592-025-02707-1)
>
> ⚠️ *勘误说明：本文档初版错误地引用了 "Xu H et al., STORM, Nature Methods 2025"。实际 STORM 是 Jinxi Xiang 等人的 arXiv 预印本（arXiv:2604.03630, 2026），并非 Nature Methods 论文。此处已修正为正确的 OmiCLIP/Loki 论文。*

**为什么这是第二推荐**

OmiCLIP 是目前**与你的项目任务最接近的已发表顶刊论文**——它做的事情和你几乎一模一样：从 H&E 病理图像预测空间转录组。但它的意义不在于让你去复现它的方法，而在于**借用它的分析框架和实验设计来指导你自己的项目规划**。

**1. 任务高度重合：H&E 图像 → 空间转录组预测**

OmiCLIP 的核心模块 **Loki PredEx** 直接从 H&E 图像预测空间基因表达——这正是你项目做的事情（只不过你预测的是通路评分而非单个基因）。论文在 2.2M 对图像-转录组 patch 上训练，覆盖 32 个器官，这是目前最大规模的 H&E-ST 配对训练数据。

**2. 系统比较了病理基础模型的性能**

OmiCLIP 将 UNI 和 GigaPath 作为基线进行了对比，在统一实验条件下评估不同 PFM 对 ST 预测任务的影响。这是你 P2-3（Virchow2 替代 UNI）决策时可以直接引用的基准——不需要靠猜测，已有顶刊文献量化了不同 PFM 之间的性能差距。

**3. 五模块平台提供了完整的验证框架**

```
Loki 平台五模块              →  你项目的对应/可扩展方向
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Loki PredEx（H&E→基因预测）   →  你当前的核心任务（H&E→通路评分）
Loki Align（多切片对齐）       →  可用于对齐同一患者的多张连续切片
Loki Annotate（组织注释）      →  可用于验证你的通路预测在组织区域上是否合理
Loki Decompose（细胞分解）     →  未来可解释性方向：预测的免疫通路是否对应免疫细胞富集区？
Loki Retrieve（跨模态检索）    →  可用于查找"哪些形态模式对应高通路活性"
```

**阅读重点**：
1. **Figure 1（整体架构）**：理解 CLIP 风格的图像-转录组对比学习框架——这是当前 H&E-ST 多模态方法的主流范式
2. **PFM 对比部分**：UNI vs GigaPath 在 ST 预测任务上的性能对比——直接指导你 P2-3 的 PFM 选型
3. **跨数据集验证策略**：OmiCLIP 在 19 个公开数据集 + 4 个自建数据集上做了验证——这对你思考"3 个患者够不够、应该怎么扩展验证"有直接指导
4. **Loki PredEx 的评估指标**：论文用了哪些指标来衡量预测性能？你的 PCC/R²/MAE 三件套是否足够？

**阅读后你应该能回答**：
- OmiCLIP 的对比学习框架能否简化后用于你的项目（如 patch-通路 对比学习）？
- OmiCLIP 报告的 PFM 比较结论是什么？这如何影响你的 P2-3（Virchow2）决策？
- 从"单任务（H&E→通路）"扩展到"多任务平台（预测 + 注释 + 分解）"是不是一条合理的长期路线？

## 五、附：改进步骤的"为什么"速查表

| 步骤 | 一句话原因 | 如果不做会怎样 |
|------|----------|--------------|
| P0-b α扫参 | MixUp 让模型多学了 4 个 epoch，不同 α 的"混合强度"可能影响这个增益大小 | 错过 +1-2% 的零成本提升 |

| P2-3 Virchow2 | 更大更强的病理 AI 模型可能提取更丰富的信息，突破 UNI 的表达能力限制 | 错过确定性强的基础模型升级收益 |

## 六、附加分析：为什么 9 例数据会让一些"失败实验"重获机会？

> 操作建议文档第九~十三章详细列出了数据从 3 例→9 例后各项实验的重新评估。这里用初学者友好的方式解释背后的核心逻辑。

### 6.1 核心概念：统计可靠性 vs 样本量

**类比**：你要评估一个班级 30 个学生的平均身高。随机抽 3 个学生量 → 结果可能是 1.65m、1.72m、1.58m，均值 1.65m。再抽 3 个可能完全不同。但抽 9 个学生 → 结果会稳定很多，而且你可以算出"这次估计有多靠谱"（标准差）。

**映射到本项目**：
- 3 例跨患者：只有 3 个"留出患者测试"的结果。每个通路 PCC 的波动很大（同一配置跑两次，Coagulation PCC 0.291→0.382，差 31%）
- 9 例 LOPO：有 9 个独立测试结果。可以可靠地说"Interferon_Alpha 确实是最难的"（因为它在 9 折中都低），而不是"恰好在某一折表现差"

这就是为什么 P1-1（通路加权损失）在 3 例下失败的根因可能在 9 例下消失：**我们终于可以可靠地知道哪些通路真正难**。

### 6.2 核心概念：训练数据多样性与过拟合

**类比**：你只有 2 个人的笔迹样本学认字，再去认第 3 个人的笔迹 → 学到的可能不是"字本身"而是"那 2 个人的书写习惯"。但如果你有 8 个人的笔迹样本，就能学到"什么是这个字的本质特征"。

**映射到本项目**：
- 3 例跨患者（2 人训练→1 人测试）：模型只有 2 个患者的"世界经验"，容易学到患者特异性模式
- 9 例 LOPO（8 人训练→1 人测试）：模型从 8 个患者学到的模式更可能反映真正的"食管癌病理规律"

这就是为什么 P1-2（编码器加深）在 3 例下过拟合，但在 9 例下可能有效：**参数多了 27%，但训练数据多了 240%，每个参数对应的"学习素材"反而更充足**。

### 6.3 为什么对比学习从"不可能"变为"条件可能"

OmiCLIP 论文的关键洞察：**如果你想让 AI 学会"什么样的病理形态对应什么样的基因表达"，你需要足够多的配对例子，让 AI 自己能发现规律。**

- 3 例 ~18,000 对：相当于一本只有 18 页的教材，每页讲一个"形态-基因对应关系"。不够。
- 9 例 ~54,000 对：相当于 54 页的教材。仍然薄，但配合以下技巧可以用了：
  - **不从头学**：图像理解能力用 UNI2-h 的已有知识，只学"对齐"（哪些形态对应哪些通路）
  - **不做全基因**：只对齐 30 条通路（相当于只学 30 个"单词"），而不是 20,000 个基因
  - **利用患者内差异**：同一患者内肿瘤核心 vs 癌旁正常的形态差异天然构成有意义的对比

### 6.4 为什么染色归一化在 9 例下收益可能更大

染色归一化解决的核心问题是"不同医院染出来的片子颜色不一样"。

- 3 例 = 最多 3 种染色风格（可能只有 2-3 家医院）
- 9 例 = 可能来自 5-7 家不同医院 → 染色差异更显著 → 归一化的消除效果更明显
