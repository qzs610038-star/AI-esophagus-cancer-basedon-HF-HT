# H&E→基因通路预测：教师-学生架构改进方案

> 更新：2026-05-22 | 基于模型性能排名校验 + 师生架构分析

---

## 一、当前师生架构总览

本任务的本质是：**教师模型**提取病理图像特征 → **学生模型**学习特征到30条基因通路 ssGSEA 的回归映射。

```
┌─────────────────────────────────────────────────┐
│                  教师模型 (冻结)                  │
│  UNI2-h / Virchow2 / OmiCLIP / ResNet-50 / ViT  │
│  输入: H&E patch (224×224)                       │
│  输出: token序列 [B, N, D] 或 池化向量 [B, D]    │
└────────────────────┬────────────────────────────┘
                     │ token / pooled feature
┌────────────────────▼────────────────────────────┐
│                  学生模型 (可训练)                │
│  TokenEncoder / GraphSAGE / GAT / MLP            │
│  + CoordEmbed (空间坐标编码)                      │
│  输出: 30条通路 ssGSEA 预测值 [B, 30]             │
└─────────────────────────────────────────────────┘
```

## 二、已识别改进建议：师生归类

### A. 教师模型改进

| # | 建议 | 类别 | 预期收益 | 投入 | 优先级 |
|---|------|------|:---:|:---:|:---:|
| A1 | 多教师晚期融合（UNI+Virchow2 预测加权） | 输出端 | +3-5% PCC | 零训练 | **P0** |
| A2 | 多教师早期融合（token concat → 2816维） | 输出端 | +3-5% PCC | 1次训练 | P1 |
| A3 | 多分辨率特征金字塔（112/224/448 px） | 输入端 | +2-4% PCC | ~0.5d | P1 |
| A4 | Register token 利用（样式编码+条件归一化） | 输出端 | +1-3% PCC | ~1d | P2 |

| A7 | 换教师：DINOv2 病理版 | 整体替换 | 不确定 | ~2d | P2 |

## 三、优先级执行路线

```
第1周 (P0): B1 Ensemble + B4 空间平滑 + B3 通路解耦头 + A1 多教师融合
第2周 (P1): B2 Perceiver架构 + A8 染色归一化 + B5 Encoder加深
第3周 (P1-P2): B7 对比回归损失 + A2 多教师早期融合 + B6 Patch自适应加权
第4周 (P2): B11 空间对比学习预训练 + A5 新教师模型评估
长期 (P3): A6 教师域自适应 + B12 FiLM层 + 端到端微调
```

## 四、核心判断

1. **教师端**最大杠杆：多教师融合（UNI + Virchow2）——特征来源正交，互补性强，晚期融合零训练成本可立即验证。
2. **学生端**最大杠杆：空间感知增强——当前学生架构未利用"相邻组织区域基因表达连续"这一强生物学先验。
3. **架构 vs 数据**：3患者太少是泛化的根本瓶颈，但 P2-8（扩展到5患者）需要新数据，短期内应优先挖掘架构/训练层面的改进空间。

## 五、探索中的适配架构（2026-05-22 调研）

以下架构来自 2024-2025 年最新文献，按适配本项目的可行性和预期收益分三类。

---

### 类别 A：高度适配 — 可直接作为新教师或学生

#### A1. CONCH — 病理视觉语言基础模型（自然适配）

| 来源 | Nature Medicine, 2024 (Harvard/MIT) |
|------|------|
| 架构 | CoCa（对比+生成双塔），1.17M 病理图文对预训练，ViT-L/14 backbone |
| 输出 | [B, 257, 1024] token + CLS 768维 |
| 适配方式 | 直接替换 UNI2-h 作为教师模型，token 输入到现有 TokenEncoder 学生 |
| 预期收益 | +3-8% PCC（优于 OmiCLIP，因 CONCH 是病理域内预训练，非自然图像） |
| 适配难度 | 低（与 OmiCLIP 同架构，仅换权重文件） |
| 权重 | 公开 at HuggingFace (`MahmoodLab/conch`) |

**判断**：OmiCLIP 跨患者失败的根本原因是 CoCa 训练在自然图像上，域不匹配。CONCH 用相同 CoCa 架构但在 **1.17M 病理图文对**上预训练，理论上应大幅优于 OmiCLIP。是本项目最优先的教师替换候选。

#### A2. FmH2ST — 基础模型+双图融合（2025 最新，最直接对标）

| 来源 | Nucleic Acids Research, 2025 |
|------|------|
| 架构 | 双分支：①病理基础模型（spot+slice 多级特征）②Spot 特定分支（多尺度 CNN + Transformer + GAT 双图） |
| 核心创新 | **双图策略**：空间 KNN 图 + 可学习语义特征图，捕获局部和长程依赖 |
| 适配方式 | 保留双图 GAT 学生，换教师为 UNI2-h；或直接用 FmH2ST 完整管线 |
| 预期收益 | +3-5% PCC（双图已证明优于单空间图） |
| 适配难度 | 中（需实现双图构建 + 自适应注意力融合） |

**判断**：FmH2ST 的双图设计直接解决了当前 EGN-v2+UNI（单空间 KNN 图）的不足——语义相似的远距离 patch 也应有边连接。这是对 EGN-v2 的最强升级方向。

#### A3. SpaGT — 空间信息增强图 Transformer（结构感知自注意力）

| 来源 | Communications Biology, 2025 |
|------|------|
| 架构 | 纯图 Transformer，边嵌入 + 节点嵌入联合进化 |
| 核心创新 | **结构增强自注意力**——注意力权重同时取决于节点特征和空间边信息 |
| 适配方式 | 替代当前 TokenEncoder 的 standard self-attention 为 structure-reinforced attention |
| 预期收益 | +2-4% PCC |
| 适配难度 | 中（修改注意力机制，保留 token encoder 其余结构） |

#### A4. UNI2-h 本身的多层特征利用

| 来源 | UNI2-h ViT-h/14 内部 |
|------|------|
| 架构 | 24 层 ViT，各层 hidden states 含不同粒度信息 |
| 输出 | Layer 12/18/24 的 token → [B, 65, 1536] × 3 |
| 适配方式 | 提取 UNI2-h 多层的 token 输出，concat 或 cross-attn 融合后给学生 |
| 预期收益 | +2-4% PCC |
| 适配难度 | 低（仅需修改特征提取脚本 + 学生输入维度） |
| 特点 | 零新增模型依赖，仅改变特征提取策略 |

---

### 类别 B：学习范式 — 训练方法层面的架构级改进

#### B5. STFlow — 流匹配生成式建模（ICML 2025）

| 来源 | ICML 2025 |
|------|------|
| 核心思路 | 将基因表达预测从直接回归改为**条件生成**：学习从噪声→基因表达的 flow，以组织学特征为条件 |
| 创新点 | ZINB 先验匹配基因表达稀疏性，E(2)-不变去噪器保持空间一致性，全切片级别建模细胞间依赖 |
| 报告性能 | +18% PCC 相对提升 vs 病理基础模型 |
| 适配到本项目 | 替代 HuberLoss 回归头为 Flow Matching head，保留 UNI-Tokens encoder 作为条件编码器 |
| 预期收益 | +5-10% PCC |
| 适配难度 | 高（需重写训练逻辑+采样推理） |

**判断**：STFlow 是本次调研中最激进但最可能带来突破的方向。直接回归到 30 个连续值忽略了基因表达的随机性和多模态性。Flow Matching 建模的是 P(gene | histology) 的条件分布而非点估计，理论上更符合生物学本质。

#### B6. STPath — 大规模掩码生成预训练

| 来源 | npj Digital Medicine, Nov 2025（原 bioRxiv Apr 2025） |
|------|------|
| 核心思路 | 在 983 WSI-ST 对（38,984 基因, 17 器官）上做掩码基因表达预测的生成预训练 |
| 报告性能 | +6.9% PCC，零样本预测能力 |
| 适配到本项目 | 使用其预训练权重作为教师/学生初始化 |
| 预期收益 | +4-8% PCC |
| 适配难度 | 中（需评估权重可用性和兼容性） |

#### B7. HECLIP — 非对称对比学习（图像→基因单向对齐）

| 来源 | 2025 |
|------|------|
| 核心思路 | 用对比学习将图像 patch 和基因表达嵌入对齐，但只用图像编码器做检索式预测 |
| 适配方式 | 在现有 UNI token 特征上增加对比 loss 辅助训练，检索相似训练样本的表达作为额外预测 |
| 预期收益 | +2-4% PCC |
| 适配难度 | 低（改 loss + 检索模块） |
| 特点 | 特别适合小样本场景（本项目 3 患者），检索机制天然具备泛化性 |

#### B8. 空间对比学习预训练（患者内部自监督）

| 来源 | 综合 BLEEP + Magic (2025) 思路 |
|------|------|
| 核心思路 | 训练前先在同患者 WSI 内做自监督：相邻 patch 互为正样本，远距离为负样本 |
| 适配方式 | Stage1: 用对比 loss 训练学生 encoder → Stage2: 加回归 head finetune |
| 预期收益 | +3-6% PCC |
| 适配难度 | 中（需额外预训练阶段） |
| 备注 | 本质等价于 B11，但使用对比学习范式，已在上文 P2 中建议 |

### 类别 C：架构组件 — 可嵌入当前学生的技术模块

#### C9. Mamba/SSM — 线性复杂度序列建模

| 来源 | MambaMIL (2024), GAT-Mamba (2025), WSI-HGMamba (2025), SlideMamba (2025) |
|------|------|
| 核心思路 | 用状态空间模型替代 Transformer self-attention，O(n) 复杂度 |
| 适配方式 | 将学生 TokenEncoder 的 self-attention 层替换为 Mamba block |
| 优势 | 可处理更多 token（多分辨率/多教师 token 拼接），线性扩展 |
| 预期收益 | +1-3% PCC（算力不变时可用更多 token） |
| 适配难度 | 低-中（pip install mamba-ssm，替换 attention 层） |

**判断**：在 Uni Token (65 tokens) 规模下 Mamba 优势不明显，但做多教师/多尺度融合后 token 数增长到 200+，Mamba 的线性复杂度优势显著。

#### C10. Mixture of Experts — 多通路共享+专有混合

| 来源 | M4 (Medical Image Analysis, 2025) |
|------|------|
| 核心思路 | 多个专家 MLP + 门控网络，不同通路激活不同专家组合 |
| 适配方式 | 替代当前共享 30-head MLP 为 4-8 个专家 MLP + 通路级门控 |
| 预期收益 | +2-4% PCC |
| 适配难度 | 低-中（改 MLP 层，其余不变） |

**判断**：30 条通路涵盖代谢、免疫、信号转导等多种功能类别，共享一个 MLP 强制它们使用相同非线性变换。MoE 让模型自动学习"免疫通路用专家 A+C，代谢通路用专家 B+D"，提升表达力而不增加同等参数量。

#### C11. 基因先验注入 — 通路由已知生物网络约束

| 来源 | GeneQuery (2024), Prior Knowledge-Guided Multilevel GNN (2024) |
|------|------|
| 核心思路 | 将已知的基因共表达网络/通路层级关系编码为图或嵌入，约束预测输出的一致性 |
| 适配方式 | 在预测层前加一层 GraphConv(30→30)，邻接矩阵来自 MSigDB/KEGG 通路关系；或加先验正则化 loss |
| 预期收益 | +1-3% PCC（消除违反生物学的预测，间接提升精度） |
| 适配难度 | 低（构建 30×30 通路关系矩阵 + 加 GraphConv 层） |

### 适配架构优先级排序

```
P0 — 低投入、高置信度
├── A4  UNI2-h 多层特征（仅改特征提取，零新模型）
├── A1  CONCH 替换教师（仅换权重，与 OmiCLIP 同架构）
└── C11 基因先验注入（构建通路图 + GraphConv，开发量小）

P1 — 中投入、高收益潜力
├── A2  FmH2ST 双图策略（GAT 升级的最强方案）
├── A3  SpaGT 结构感知注意力（相对小的架构改动）
├── B7  HECLIP 检索增强（适合小样本场景）
├── C10 MoE 多专家 MLP
└── C9  Mamba 序列建模（为多教师/多尺度融合储备）

P2 — 高投入、突破性潜力
├── B5  STFlow 流匹配（需重写训练流程）
├── B6  STPath 大规模预训练（依赖权重可用性）
└── B8  空间对比学习预训练（已在原始方案 B11）
```

### 总结：最推荐的 3 个新增方向

1. **CONCH 教师**：OmiCLIP 架构 + 病理数据 = 直接修正 OmiCLIP 的域不匹配问题，替换成本最低。
2. **FmH2ST 双图**：当前 EGN-v2 仅用空间 KNN 图，增加语义特征图可捕获关键的远距离组织相似性。
3. **STFlow 流匹配**：从"点回归"升级为"分布建模"，是范式级突破，虽然实现成本最高但潜在收益最大。
