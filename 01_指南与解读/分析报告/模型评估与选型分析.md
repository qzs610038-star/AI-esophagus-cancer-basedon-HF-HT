## 1. 评估背景

### 1.1 项目特点概述

PFMval 项目是一个**病理图像到通路评分的多目标回归任务**，具有以下核心特点：

| 维度 | 项目现状 |
|------|----------|
| **任务类型** | 从病理图像 Patch 预测 8 个 ssGSEA 基因集通路评分（多目标回归） |
| **数据规模** | 约 10,578 个 Patch 样本（train ~9,806, val ~772） |
| **当前方案** | UNI2-h（冻结）提取 1536 维特征 → MLP 回归头（1536→256→8） |
| **数据分布** | 8 列均严重偏离正态，正偏态为主（偏度 0~3.2），ifng 异常值 19% |
| **空间约束** | 空间无重叠划分（距离 ≥ 350px），防止空间泄漏 |
| **预测目标** | 8 个通路/基因集评分（tls, tgfb, emt, hypoxia, mhc, icp, ifng, toxic） |
| **评估指标** | MSE, MAE, R², PCC |

### 1.2 评估目的

本次评估旨在：
1. 系统梳理空间转录组预测领域的最新进展
2. 结合项目数据特点，筛选最适合的模型架构
3. 提供可落地的实现路径和改造建议
4. 为后续实验路线提供决策依据

## 3. 推荐模型详解

### 3.1 PEaRL (首选推荐)

#### 3.1.1 模型简介与核心思想

**PEaRL** (Pathway-Enhanced Representation Learning) 是 2026 年发表于 WACV 的工作，是**首个专门针对通路表达预测设计的多模态框架**。

**核心创新点**：
- 使用 **ssGSEA** 将高维基因表达转换为通路评分（与项目目标完全一致）
- 设计 **Transformer-based Pathway Encoder** 编码通路间的生物学关系
- 采用 **对比学习** 对齐图像特征和通路特征
- 同时支持基因和通路表达预测

#### 3.1.2 架构示意

```
输入: 病理图像 Patch + 对应基因表达
                    ↓
    ┌───────────────┴───────────────┐
    ↓                               ↓
图像编码器 (UNI)              ssGSEA计算
    ↓                               ↓
图像嵌入 (1536-dim)           通路评分向量
    ↓                               ↓
    └───────────────┬───────────────┘
                    ↓
            对比学习对齐
                    ↓
    ┌───────────────┴───────────────┐
    ↓                               ↓
下游任务: 通路预测              下游任务: 基因预测
```

**Pathway Encoder 细节**：
- 输入：ssGSEA 通路评分矩阵 + 空间位置编码
- 架构：2层 Transformer Encoder + Multi-Head Self-Attention
- 输出：256-dim 通路嵌入

#### 3.1.3 数据特征与模型特征匹配分析

**a) "数据特征 → 模型特征" 匹配关系**

| 数据特征 | 具体表现 | 模型对应设计 | 为什么匹配 |
|---------|---------|-------------|-----------|
| **小样本量** | 仅 ~1万 patch (train 9806, val 772) | **对比学习预训练** | 对比学习能在有限样本下学习更有判别性的特征表示，比纯监督学习更适合小数据 |
| **严重非正态分布** | 8列 Shapiro-Wilk p ≈ 0，偏度 0~3.22 | **ssGSEA 通路编码器** | 通路编码器通过 Transformer 建模通路间关系，对单个通路的分布不敏感，关注相对关系 |
| **极端异常值** | ifng 异常值 19.01%，icp 峰度 15.48 | **对比学习损失 (InfoNCE)** | 对比损失关注相对相似度而非绝对值，对异常值比 MSE 更鲁棒 |
| **多目标同时预测** | 8个通路同时输出 | **Pathway Encoder 的 Self-Attention** | 8个通路作为序列输入，Self-Attention 自动学习通路间生物学关联（如 IFNG-Toxic 免疫相关性） |
| **空间无重叠划分** | 距离 ≥ 350px 的空间约束 | **空间位置编码** | Pathway Encoder 支持空间坐标输入，可建模空间上下文关系 |
| **标准差差异大** | mhc(2868) vs icp(521)，相差5.5倍 | **对比学习的归一化特性** | 对比学习将特征投影到单位超球面，天然处理不同尺度的目标 |

**b) 与当前方案的具体对比**

| 对比维度 | 当前 MLP 方案 | PEaRL 方案 | 预期改进 |
|---------|--------------|-----------|---------|
| **架构** | UNI2-h → MLP(1536→256→8) | UNI2-h → Pathway Encoder + 对比学习 | 引入通路间关系建模 |
| **损失函数** | MSELoss（对异常值敏感） | InfoNCE 对比损失 + 可选回归损失 | 对异常值更鲁棒，关注语义对齐 |
| **特征利用** | 仅使用图像特征 | 图像特征 + 通路特征对比对齐 | 学习更好的跨模态表示 |
| **通路关系** | 8个输出相互独立 | Pathway Encoder 建模通路间依赖 | 利用生物学先验提升预测一致性 |
| **小样本适配** | MLP 容易过拟合 | 对比学习提供更好的正则化 | 降低过拟合风险 |

**c) 评估维度匹配度**

| 评估维度 | 匹配度 | 具体理由 |
|----------|--------|----------|
| **任务匹配度** | ★★★★★ | **直接预测通路评分**，与项目目标完全一致；使用 ssGSEA 作为中间表示 |
| **数据规模适配** | ★★★★☆ | 对比学习对数据量要求适中；论文在3个数据集验证，规模类似 |
| **PFM兼容性** | ★★★★★ | 使用 UNI 作为图像编码器，**可直接复用现有 UNI2-h 特征** |
| **实现复杂度** | ★★★☆☆ | 中等；需实现 Pathway Encoder 和对比学习损失 |
| **预期性能** | ★★★★★ | 论文报告通路预测 PCC 提升 **20.4%**，基因预测提升 **58.9%** |
| **分布处理** | ★★★★☆ | ssGSEA 本身对原始分布不敏感；对比损失对异常值鲁棒 |

### 3.2 HistoPrism (次选推荐)

#### 3.2.1 模型简介与核心思想

**HistoPrism** 是 2026 年 ICLR 的工作，专注于**跨癌种泛化的通路级别预测**。

**核心创新点**：
- 提出 **Gene Pathway Coherence (GPC)** 评估基准，从基因级评估转向通路级评估
- 使用 **Cross-Attention** 注入癌种条件，实现 pan-cancer 建模
- 轻量级 Transformer 架构，计算效率高
- 直接使用 PFM 特征，无需端到端训练

#### 3.2.2 架构示意

```
输入: UNI特征 (N patches × 1024-dim) + 癌种条件 (one-hot)
                    ↓
        Cross-Attention (癌种条件注入)
                    ↓
        条件化特征 (N × 1024-dim)
                    ↓
        Transformer Encoder (上下文聚合)
                    ↓
        MLP回归头 → 基因表达预测
                    ↓
        聚合为通路评分 (GPC评估)
```

#### 3.2.3 数据特征与模型特征匹配分析

**a) "数据特征 → 模型特征" 匹配关系**

| 数据特征 | 具体表现 | 模型对应设计 | 为什么匹配 |
|---------|---------|-------------|-----------|
| **小样本量** | 仅 ~1万 patch (train 9806, val 772) | **轻量级 Transformer (4层)** | 浅层 Transformer 参数量适中，在小数据上训练稳定，不易过拟合 |
| **严重非正态分布** | 8列 Shapiro-Wilk p ≈ 0，偏度 0~3.22 | **GPC (Gene Pathway Coherence) 评估** | 从基因级评估转向通路级评估，聚合操作平滑了个别基因的极端分布 |
| **极端异常值** | ifng 异常值 19.01%，峰度高达 15.48 | **Transformer 的多头注意力机制** | 注意力机制可学习忽略异常 patch，关注代表性区域 |
| **多目标同时预测** | 8个通路同时输出 | **Cross-Attention 条件注入** | 癌种条件注入机制可学习不同通路间的共享表示和特异性模式 |
| **空间无重叠划分** | 距离 ≥ 350px 的空间约束 | **上下文聚合的 Transformer** | 可建模 patch 间的长距离依赖关系，利用空间上下文 |
| **标准差差异大** | mhc(2868) vs icp(521)，相差5.5倍 | **分层预测结构** | 先预测基因再聚合为通路，不同基因可学习不同尺度 |

**b) 与当前方案的具体对比**

| 对比维度 | 当前 MLP 方案 | HistoPrism 方案 | 预期改进 |
|---------|--------------|----------------|---------|
| **架构** | UNI2-h → MLP(1536→256→8) | UNI2-h → Cross-Attention → Transformer → MLP | 引入上下文建模和条件注入 |
| **特征聚合** | 单 patch 独立预测 | 多 patch Transformer 聚合 | 利用空间上下文信息 |
| **损失函数** | MSELoss | 基因级 MSE + GPC 评估 | 更细粒度的监督信号 |
| **通路关系** | 8个输出相互独立 | 基因→通路聚合，隐式建模关系 | 生物学上更合理的通路表示 |
| **实现难度** | 简单 | 中等（有开源代码参考） | 可快速验证 |

**c) 评估维度匹配度**

| 评估维度 | 匹配度 | 具体理由 |
|----------|--------|----------|
| **任务匹配度** | ★★★★☆ | 原生支持通路级评估；但主要预测基因再聚合 |
| **数据规模适配** | ★★★★★ | **轻量级架构**，适合小数据量；论文验证数据规模类似 |
| **PFM兼容性** | ★★★★★ | 基于 UNI 特征，**可直接复用现有 UNI2-h 特征** |

#### 3.2.6 实现路径

```python
# 基于现有代码的改造方案

# 1. 新增 Cross-Attention 癌种条件注入
class PanCancerConditioning(nn.Module):
    def __init__(self, d_model=1024, num_cancer_types=10):
        super().__init__()
        self.cancer_emb = nn.Linear(num_cancer_types, d_model)
        self.cross_attn = CrossAttention(d_model)
    
    def forward(self, patch_features, cancer_type):
        # patch_features: (N, 1024)
        # cancer_type: (num_cancer_types,) one-hot
        cancer_emb = self.cancer_emb(cancer_type)  # (1024,)
        conditioned = self.cross_attn(
            query=patch_features,
            key=cancer_emb.unsqueeze(0),
            value=cancer_emb.unsqueeze(0)
        )
        return conditioned

# 2. 新增 Transformer Encoder
class HistoPrismEncoder(nn.Module):
    def __init__(self, d_model=1024, nhead=8, num_layers=4):
        super().__init__()
        self.transformer = TransformerEncoder(
            d_model=d_model,
            nhead=nhead,
            num_layers=num_layers
        )
    
    def forward(self, x):
        return self.transformer(x)

# 3. 修改现有回归头
class HistoPrismHead(nn.Module):
    def __init__(self):
        self.conditioning = PanCancerConditioning()
        self.encoder = HistoPrismEncoder()
        self.regressor = nn.Sequential(
            nn.Linear(1024, 512),
            nn.ReLU(),
            nn.Linear(512, 8)  # 8个通路
        )
    
    def forward(self, uni_features, cancer_type):
        x = self.conditioning(uni_features, cancer_type)
        x = self.encoder(x)
        return self.regressor(x)
```

### 3.3 DeepPathway (第三推荐)

#### 3.3.1 模型简介与核心思想

**DeepPathway** (2025) 是**专门针对通路表达预测**的对比学习方法。

**核心创新点**：
- 使用 **H-optimus-0**（病理基础模型）提取图像特征
- 使用 **Transformer** 编码通路特征
- 双阶段训练：对比学习预训练 + 监督微调
- 直接预测通路表达，而非基因表达再聚合

#### 3.3.2 架构示意

```
阶段1: 对比学习预训练
    图像分支: H&E → ResNet18 + H-optimus-0 → 图像嵌入
    通路分支: 通路表达 → Transformer → 通路嵌入
    损失: 对比学习对齐

阶段2: 监督微调
    冻结图像编码器
    训练 DNN 预测头: 图像嵌入 → 通路表达
```

#### 3.3.3 数据特征与模型特征匹配分析

**a) "数据特征 → 模型特征" 匹配关系**

| 数据特征 | 具体表现 | 模型对应设计 | 为什么匹配 |
|---------|---------|-------------|-----------|
| **小样本量** | 仅 ~1万 patch (train 9806, val 772) | **双阶段训练策略** | 阶段1对比学习预训练学习通用表示，阶段2监督微调适应特定数据，提高小样本利用效率 |
| **严重非正态分布** | 8列 Shapiro-Wilk p ≈ 0，偏度 0~3.22 | **Transformer 通路编码器** | 通路编码器将8个通路视为序列，通过Self-Attention建模相对关系，弱化绝对分布影响 |
| **极端异常值** | ifng 异常值 19.01%，icp 峰度 15.48 | **对比学习预训练 + Huber Loss** | 对比学习关注相对排序，对异常值不敏感；可结合 Huber Loss 增强鲁棒性 |
| **多目标同时预测** | 8个通路同时输出 | **通路专用 Transformer** | 专门设计用于通路预测，而非基因预测的折中方案 |
| **空间无重叠划分** | 距离 ≥ 350px 的空间约束 | **ResNet18 局部特征 + H-optimus 全局特征** | 双编码器结构同时捕捉局部形态和全局模式 |
| **标准差差异大** | mhc(2868) vs icp(521)，相差5.5倍 | **特征投影层** | 独立的图像和通路投影层可学习适应不同尺度的特征 |

**b) 与当前方案的具体对比**

| 对比维度 | 当前 MLP 方案 | DeepPathway 方案 | 预期改进 |
|---------|--------------|-----------------|---------|
| **架构** | UNI2-h → MLP(1536→256→8) | ResNet18+H-optimus → Transformer → 对比学习 → DNN头 | 双阶段训练，更好的特征对齐 |

| **PFM兼容** | UNI2-h (1536-dim) | H-optimus-0 (768-dim) | 可替换为 UNI2-h，但需调整维度 |

class DeepPathwayContrastive(nn.Module):

### 3.4 EGN (第四推荐)

#### 3.4.1 模型简介与核心思想

**EGN** (Exemplar Guided Network, 2022) 是较早的**利用相似样本引导预测**的方法。

**核心创新点**：
- 在 ViT 中间层引入 **Exemplar Guidance (EG) 模块**
- 从训练集中检索相似 patch，融合其特征和表达信息
- 使用注意力机制进行特征融合

#### 3.4.2 架构示意

```
输入图像 → ViT Backbone → [EG模块] → 注意力回归头 → 基因表达
                ↑
        检索相似样本 (Exemplar Bank)
```

**EG模块细节**：
- 维护一个 Exemplar Bank 存储训练样本
- 根据中间特征检索 k 个最相似样本
- 使用注意力融合查询样本和相似样本特征

#### 3.4.3 数据特征与模型特征匹配分析

**a) "数据特征 → 模型特征" 匹配关系**

| 数据特征 | 具体表现 | 模型对应设计 | 为什么匹配 |
|---------|---------|-------------|-----------|
| **极小样本量** | 仅 ~1万 patch，val 仅 772 | **Exemplar Guidance (EG) 模块** | 从训练集检索相似样本提供额外监督信号，数据越少相对增益越大 |
| **严重非正态分布** | 8列 Shapiro-Wilk p ≈ 0 | **相似样本的加权融合** | 基于相似度的注意力融合，对目标分布形状不敏感 |
| **极端异常值** | ifng 异常值 19.01% | **k-NN 相似样本检索** | 异常值样本会检索到正常样本作为参考，起到平滑作用 |
| **多目标同时预测** | 8个通路同时输出 | **可扩展的回归头** | EG 模块输出融合特征，可接入任意回归头预测多个目标 |
| **空间无重叠划分** | 距离 ≥ 350px 的空间约束 | **Exemplar Bank 的空间感知** | 可扩展为考虑空间距离的相似度计算 |
| **标准差差异大** | mhc(2868) vs icp(521) | **注意力权重自适应** | 注意力机制可学习不同通路的重要性权重 |

**b) 与当前方案的具体对比**

| **推理方式** | 单样本独立预测 | 检索相似样本辅助预测 | 对训练分布内的样本更准确 |
| **可解释性** | 黑盒预测 | 可展示相似样本作为依据 | 提供预测的可解释支撑 |
| **实现难度** | 简单 | 中等（有开源代码） | 可快速验证 |

```python
# 基于现有代码的改造方案

class ExemplarGuidance(nn.Module):
    def __init__(self, k=5):
        super().__init__()
        self.k = k
        self.exemplar_bank = []  # 存储 (feature, pathway_score) 对
        
    def forward(self, query_feature):
        # 检索 k 个最相似样本
        similarities = cosine_similarity(query_feature, self.exemplar_bank)
        topk_indices = similarities.topk(self.k).indices
        
        # 获取相似样本特征和标签
        exemplar_features = self.exemplar_bank[topk_indices]
        exemplar_scores = self.exemplar_scores[topk_indices]
        
        # 注意力融合
        attn_weights = self.attention(query_feature, exemplar_features)
        fused = query_feature + attn_weights @ exemplar_features
        
        return fused, exemplar_scores

class EGNPredictor(nn.Module):
    def __init__(self):
        self.uni_encoder = UNI2H(frozen=True)
        self.exemplar_guidance = ExemplarGuidance(k=5)
        self.regressor = MLP(1536 + 8, 8)  # 融合特征+exemplar信息
    
    def forward(self, images):
        features = self.uni_encoder(images)
        guided_features, exemplar_scores = self.exemplar_guidance(features)
        return self.regressor(torch.cat([guided_features, exemplar_scores.mean(0)], dim=-1))
```

## 5. 不推荐模型说明

### 5.1 不推荐列表与数据不匹配分析

| 模型 | 不推荐原因 | 与本项目数据特征的不匹配点 |
|------|------------|------------------------|
| **STNet (2020)** | 过于陈旧，无PFM，性能已被后续工作全面超越 | **无预训练模型**：需从头训练DenseNet，在小样本(1万)上严重欠拟合；**无分布处理**：MSE损失对偏度3.22、异常值19%的数据极敏感 |
| **HisToGene (2021)** | 端到端训练，无法复用UNI2-h特征；需要大量数据 | **无法利用已有特征**：必须端到端训练ViT，与现有UNI2-h特征不兼容；**数据量不足**：ViT-MLP架构在1万样本上容易过拟合，论文使用32个样本的HER2+数据集 |
| **Hist2ST (2022)** | 架构复杂（ConvMixer+Transformer+GNN），训练困难；ZINB损失不适合通路预测 | **损失函数不匹配**：ZINB (Zero-Inflated Negative Binomial) 针对基因计数数据设计，不适合连续的通路评分；**训练不稳定**：ConvMixer+Transformer+GNN三层堆叠在小样本上难以收敛；**无PFM支持**：无法利用UNI2-h的强大特征 |
| **THItoGene (2023)** | 架构过于复杂（Dynamic Conv+Capsule+ViT+GAT），难以实现和调优 | **过度复杂**：Dynamic Conv + Capsule + ViT + GAT 四组件堆叠，参数量巨大；**小样本灾难**：1万样本不足以支撑如此复杂的架构训练；**调参困难**：多组件带来大量超参数，验证集(772)太小无法有效调参 |
| **BLEEP (2023)** | 基于检索的推理方式，泛化能力受限；不适合回归任务 | **推理方式冲突**：k-NN检索需要存储全部训练样本，而项目空间划分(≥350px)导致训练/测试分布差异大，检索失败率高；**不适合回归**：对比学习+检索针对分类/检索优化，对连续通路评分回归效果差；**异常值敏感**：检索到的相似样本可能包含异常值，导致预测偏差 |
| **mclSTExp (2024)** | 同样是检索-based方法，存在与BLEEP相同的泛化问题 | **检索泛化差**：与BLEEP相同，依赖检索的推理方式对空间划分后的数据泛化能力差；**DenseNet编码器**：无PFM支持，特征提取能力弱于UNI2-h；**对比学习不适用**：项目数据分布极端(偏度0~3.22)，对比学习难以建立稳定的相似度度量 |
| **OmiCLIP (2025)** | 基础模型预训练，需要海量数据；项目数据量不足以支撑 | **预训练数据需求巨大**：论文使用220万对图像-转录组数据预训练，项目仅1万样本；**CoCa架构复杂**：对比Caption架构需要大量数据学习视觉-文本对齐；**无法直接使用**：作为基础模型需要下游微调，但数据量不足以有效微调 |
| **FmH2ST (2025)** | 架构极其复杂（双分支+Dual Graph），实现难度极高 | **实现难度极高**：PathoDuet双分支 + Dual Graph + CNN + ViT-GAT 架构过于复杂；**无开源代码**：无法参考实现；**计算资源需求大**：双分支+图网络需要大量显存；**小样本不匹配**：复杂图网络在1万样本上容易过拟合 |
| **Digital Modeling (2025)** | 使用传统ML（RidgeCV/随机森林），与当前深度学习方案相比无优势 | **方法过于简单**：RidgeCV/随机森林无法捕捉图像特征的复杂非线性映射；**无表示学习**：直接使用UNI2-h特征，无任务特定的特征优化；**性能天花板低**：传统ML方法在当前深度学习方案基础上难以带来显著提升；**分布处理弱**：线性模型对偏态、异常值数据拟合能力差 |

### 5.2 关键排除理由与数据特征映射

1. **端到端方法（STNet/HisToGene/Hist2ST/THItoGene）**：
   - 无法复用已提取的 UNI2-h 特征
   - 需要从头训练，计算成本高
   - 在小数据量上容易过拟合

2. **检索-based方法（BLEEP/mclSTExp）**：
   - 依赖 k-NN 检索相似样本
   - 对未见过的表达模式泛化能力差
   - 不适合连续的通路评分回归

3. **基础模型预训练方法（OmiCLIP/FmH2ST）**：
   - 需要大规模预训练数据
   - 项目 1 万样本不足以支撑
   - 实现复杂度高
