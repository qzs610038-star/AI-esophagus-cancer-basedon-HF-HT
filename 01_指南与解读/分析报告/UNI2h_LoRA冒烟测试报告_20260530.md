## 2. 实验配置

```bash
PYTHONIOENCODING=utf-8 python -u train_online_cls.py \
  --mode lora --lora_rank 8 --lora_blocks all \
  --patient HYZ15040 --num_epochs 1 \
  --batch_size 1 --grad_accum_steps 2 --num_workers 0
```

| 参数 | 值 | 说明 |
|------|-----|------|
| mode | lora | Stage 1：全 24 层 Attention 注入 LoRA |
| lora_rank | 8 | LoRA 秩（rank=8，alpha=16，scale=2.0） |
| lora_blocks | all | 0-23 层全部注入 LoRA |
| batch_size | 1 | 最小 batch，防 OOM |
| grad_accum_steps | 2 | 有效 batch_size=2 |
| optimizer | AdamW | lr=1e-4（LoRA 组 + downstream 组） |
| criterion | HuberLoss | delta=1.0 |
| AMP | enabled | 混合精度训练 |

### 3.4 过拟合分析

```
Train PCC:  0.48 ──────────────────────────────▶ 0.94  (持续上升)
Val PCC:    0.54 ↗ 0.55 ↘ 0.52 ↘ 0.50 ↘ 0.49  (Epoch 2 触顶后持续下降)
Gap:        0.06 → 0.44  (过拟合 gap 急剧扩大)
```

| 关键指标 | 值 | 判断 |
|------|-----|------|
| 最佳 Epoch | **2** | ⚠️ 极早（与 OmiCLIP 的 epoch=1 类似） |
| 最佳 Val PCC | **0.5462** | vs frozen 0.5227 → +0.0235 |
| 最佳时过拟合 Gap | 0.047 | 轻微（Train 0.59 vs Val 0.55） |
| Epoch 10 过拟合 Gap | 0.438 | 严重（Train 0.94 vs Val 0.50） |
| 有效训练窗口 | **仅 1-2 epoch** | ⚠️ 超短窗口，无实用训练价值 |

> **关键判断**：单患者 LoRA 训练与 OmiCLIP 模式一致——极早过拟合、极窄有效窗口。这与跨患者泛化的核心挑战一致：模型快速学会训练分布的特征，但这些特征在跨患者场景下不可迁移。**单患者有提升（+0.0235）≠ 跨患者有提升——OmiCLIP 单患者 0.55 但跨患者 0.19（−64%）。真正的判断标准只有跨患者 Fold1。**

## 4. 关键验证确认

### 4.1 LoRA 注入正确性 ✅

```
[Model] 参数概览:
  总参数:        687.16M           ← 304M 进行中 300M UNI2-h + 其他
  可训练:        5.77M (0.8%)      ← 仅 LoRA + 下游头
    其中 LoRA:   1.77M             ← 24层 × 2(attn) × rank(8) × dim(1536) × 2(A+B)
    其中 非LoRA: 4.00M             ← proj + head
```

- 原始 backbone 300M 参数全部冻结（requires_grad=False）
- LoRA 96 个参数张量全部可训练
- 下游头 12 个参数张量全部可训练

### 4.2 显存安全性 ✅

- 峰值显存 4.5 GB，RTX 4060 8GB 余量 ~3.5 GB
- AMP 混合精度有效减少显存
- batch_size=2 或 grad_accum=4 仍有余量

### 4.3 训练稳定性 ✅

- Loss 正常下降（0.30→0.27→0.27）
- Train PCC 上升（0.48→0.59→0.66）
- Val PCC 有效（0.54），无 NaN/Inf
- 无 CUDA OOM、无梯度爆炸

## 5. 发现的坑与解决方案

| 问题 | 表现 | 解决 |
|------|------|------|
| **输出缓冲** | 后台训练 stdout 完全不可见 | 加 `python -u` 参数 |
| **HF 网络超时** | 每次启动 5 次重试（~30s） | 设 `HF_HUB_OFFLINE=1` 环境变量 |
| **参数名错误** | `--epochs` 不存在 | 正确名称是 `--num_epochs` |
