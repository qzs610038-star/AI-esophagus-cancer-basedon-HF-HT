## 三、分级改进建议

### P0：高收益高可行（建议立即启动）

#### P0-1 🔴 升级病理基础模型特征提取器：UNI → Virchow2 / UNI2-h

**文献支撑**：
- Virchow2：632M参数，3.1M WSIs预训练，混合放大倍率，覆盖40+组织类型；Virchow2G达1.9B参数
- STPath论文验证：使用GigaPath作为spot图像嵌入器，PCC比UNI高约4-6% [40]
- STORM论文：层级空间编码器使用多个PFM做消融实验 [38]

**当前状态**：UNI已使PCC从~0.25提升至~0.38，但三模型趋同（0.37-0.38）表明UNI特征已达天花板。

**实施方案**：

| 方案 | 特征提取器 | 特征维度 | 输入分辨率 | 难度 | 预期PCC增量 |
|------|-----------|:------:|-----------|:----:|:---------:|
| **3.1a** | Virchow2（推荐） | 2560 | 224×224 | 中 | +0.03~0.05 |
| **3.1b** | UNI2-h（备选） | 1536 | 224×224 | 低 | +0.01~0.03 |
| **3.1c** | 多特征拼接：UNI⊕Virchow2 | 3584 | 224×224 | 中 | +0.04~0.06 |

**实施步骤**：
1. 下载Virchow2权重（HuggingFace: `paige-ai/Virchow2`）
2. 编写 `extract_virchow2_features.py`：批量提取三患者patch特征为 `.pt` 文件
3. 修改EGN-v2+UNI的特征加载模块，替换为Virchow2特征
4. 对EGN-v2+UNI和方案B分别跑一轮三折交叉验证
5. 对比UNI vs Virchow2的PCC提升、逐通路差异

**风险与应对**：
- Virchow2显存消耗较大（2560维），方案B的Transformer可能需要减小batch size → 使用梯度累积
- 授权问题 → Virchow2模型权重已公开在HuggingFace，可自由下载

#### P0-2 🔴 引入基因序列多模态信息（核心创新）

**文献支撑**：
- **FmH2ST** [35][42]：基因序列+组织学图像双模态框架，交叉注意力+对比学习，在benchmark数据集上超越现有方法
- **OmiCLIP** [39]：CoCa架构对齐图像与转录组，对比学习统一表征空间，跨模态鲁棒
- **PEaRL** [43]：通路增强表征学习，Transformer通路编码器+对比学习，专为通路预测设计
- **Deep Association Multimodal**（MICCAI 2025）：病理图像+基因功能语义零样本预测

**当前状态**：项目仅使用病理图像→30通路评分，30条通路的基因组成（每条通路含数十至数百个基因）信息完全闲置。这是与2025-2026前沿工作的**最显著方法学差距**。

**核心思路**：

```
┌─────────────────────────────────────────────────────┐
│              多模态基因序列增强框架                     │
├─────────────────────────────────────────────────────┤
│                                                      │
│  病理图像 Patch ──→ PFM编码器 ──→ 图像特征 [d_img]    │
│                              │                       │
│                              ├──→ 交叉注意力融合 ←──┤ │
│                              │         │            │ │
│  30条通路基因列表 ──→ 基因LM ──→ 通路嵌入 [d_path]  │ │
│  (如 scGPT/GenePT)                                 │ │
│                              │                       │
│                              ▼                       │
│                     MLP回归头 → 30通路评分             │
│                                                      │
│  训练损失: HuberLoss + 对比损失(InfoNCE)               │
│                                                      │
└─────────────────────────────────────────────────────┘
```

**具体实施方案**：

| 子任务 | 内容 | 时间 |
|--------|------|:--:|
| **3.2a** 通路嵌入编码 | 使用预训练基因语言模型（scGPT [45]或GenePT）将每条通路的基因名称列表编码为通路嵌入向量 [30, d_path] | 1天 |
| **3.2b** 交叉注意力模块 | 设计 `CrossAttentionFusion`：图像特征作为Query，通路嵌入作为Key/Value，输出通路感知的图像特征 | 1天 |
| **3.2c** 对比学习损失 | 配对（图像, 对应通路）拉近 / 非配对推远的InfoNCE损失 + 原有HuberLoss联合训练 | 1天 |
| **3.2d** 训练验证 | 集成到EGN-v2+UNI训练脚本，三折交叉验证 | 2-3天 |

**预期收益**：
- 跨患者PCC：0.38 → 0.42~0.48（+10-25%相对提升，参考FmH2ST和PEaRL报告）
- 免疫/炎症通路提升最显著（基因序列语义最丰富的通路类别）
- **学术价值**：食管癌空间转录组的图像-基因多模态预测具备独立发表潜力

#### P0-2 技术规格详述

##### scGPT vs GenePT 选择分析

| 维度 | scGPT | GenePT |
|------|-------|--------|
| 预训练规模 | 3300万+单细胞数据 | 基于GPT-3.5文本嵌入 |
| 输出维度 | 512维 | 1536维（与UNI2-h一致） |
| 输入格式 | 基因表达值序列 | 基因名称文本描述 |
| 获取难度 | 需要申请权重（academic license） | 公开API调用（OpenAI） |
| 推理开销 | 较高（需GPU） | 低（API调用） |
| 与本项目适配度 | 需要额外的表达量输入 | 直接接受基因名列表 |
| **建议** | 备选 | **首选**（维度匹配、获取简便） |

##### 交叉注意力模块参数配置

```
模块名称: Pathway-Image Cross-Attention Block
├── Query来源: 通路嵌入 (GenePT输出, d=1536)
├── Key/Value来源: 图像patch token (UNI2-h输出, d=1536)
├── 注意力头数: 8
├── 每头维度: 192 (1536/8)
├── 层数: 2 (轻量化，避免过拟合)
├── Dropout: 0.1
├── Layer Norm: Pre-LN
├── 位置编码: 可学习位置编码 (仅图像token侧)
└── 输出: 通路感知增强的图像表示 (d=1536)
```

##### 对比学习损失函数定义

```
总损失 = α × L_regression + β × L_contrastive

L_regression = MSE(pred_scores, true_scores)  [主任务损失]

L_contrastive = InfoNCE对比损失
  - 正样本对: (patch_i图像特征, patch_i对应的通路嵌入)
  - 负样本对: batch内其他通路的嵌入
  - 温度参数τ = 0.07 (经验值，参考CLIP)

权重建议:
  α = 1.0 (主损失)
  β = 0.1 (辅助损失，初始较低避免干扰回归)
  动态调度: β在epoch 5后线性增至0.3
```

##### 通路基因集来源说明

- **30条通路对应的基因集来源**：MSigDB Hallmark Gene Sets（50条通路子集）
- **每条通路平均基因数**：30-200个
- **GenePT输入方式**：将通路内所有基因名拼接为逗号分隔文本，获取pooled embedding
- **维度对齐**：GenePT输出1536维，与UNI2-h的1536维patch token天然对齐，无需额外投影层

#### P0-3 🔴 EGN-v2+UNI 图神经网络深化（GAT + 多尺度图 + 边特征）

**文献支撑**：
- **GNN for NSCLC TME**（文献[32]）：GNN建模肿瘤-免疫空间互作，C-index=0.82，可解释性揭示CD8+/PD-L1+/FOXP3+空间依赖性
- **STORM** [38]：层级空间编码器设计，“spatially aware”设计对建模组织空间结构至关重要
- **STPath** [40]：空间感知Transformer，frame averaging偏置注意力图

**当前状态**：EGN-v2+UNI是最稳定模型（Gap仅0.05），但PCC 0.38尚有空间。GNN架构深化可在保持低过拟合的同时提升精度。

**实施方案**：

| 子任务 | 内容 | 预期PCC增量 | 时间 |
|--------|------|:---------:|:--:|
| **3.3a** GAT替换GraphSAGE | 多头注意力权重替代均匀邻居聚合，自动学习空间邻居重要性 | +0.01~0.02 | 0.5天 |
| **3.3b** 多尺度空间图 | 同时构建短程图(r=150)和长程图(r=500)，双分支聚合后融合 | +0.01~0.03 | 1天 |
| **3.3c** 边特征编码 | 空间边加入距离倒数、相对方向角等几何特征 | +0.005~0.015 | 0.5天 |
| **3.3d** 深层GNN+残差 | GNN层数增至3-4层，配合残差连接防过平滑 | +0.005~0.01 | 0.5天 |
| **3.3e** 联合实验 | 最优组合方案三折交叉验证 | — | 2天 |

**实施要点**：
- 使用PyG的`GATConv`替换`SAGEConv`，head数建议4-8
- 多尺度图在`build_spatial_graph`中分别构建两个`edge_index`
- 注意GAT的显存消耗（多头注意力），必要时减小hidden_dim

### P1：中高收益需一定投入（建议1-2周内启动）

#### P1-1 🟡 域适应：跨患者泛化增强

**文献支撑**：
- **HE2RNA**（文献[13]）：迁移学习将转录组表征转移到小型数据集
- **Pan-cancer computational histopathology**（文献[28]）：深度迁移学习跨28种癌症类型
- **Game-theoretic assessment**（文献[31]）：样本制备/成像质量显著影响ST推断，临床级方案（自动染色、40×成像）优于传统方法
- **STORM** [38]：跨23个独立队列（7,245患者）验证泛化能力

**当前状态**：JFX患者三模型均分最低（0.3613），跨患者泛化Gap 0.17-0.21。域偏移是当前性能的第二大瓶颈。

**实施方案**：

| 子任务 | 方法 | 说明 | 预期PCC增量 | 时间 |
|--------|------|------|:---------:|:--:|
| **3.4a** | 染色归一化 | Macenko/Vahadane方法标准化H&E染色矩阵，消除批次效应 | +0.01~0.02 | 1天 |
| **3.4b** | MMD特征对齐 | 训练时加入Maximum Mean Discrepancy损失，缩小不同患者特征分布距离 | +0.01~0.03 | 1天 |
| **3.4c** | 对抗域适应 | 添加患者域分类器+梯度反转层(GRL)，迫使编码器学习患者不变表征 | +0.02~0.04 | 1.5天 |
| **3.4d** | 混合训练策略 | 训练时随机混合不同患者batch，配合患者级BatchNorm | +0.005~0.01 | 0.5天 |

**推荐优先级**：3.4a（快速验证）→ 3.4c（最大收益）→ 3.4b（补充）

#### P1-2 🟡 对比学习预训练：增强特征判别力

**文献支撑**：
- **CTransPath**（文献[20]）：语义相关对比学习(SRCL)在组织病理学多项下游任务超越ImageNet预训练
- **PEaRL** [43]：通路感知对比学习，将通路激活作为监督信号对齐图像表征
- **OmiCLIP** [39]：对比学习对齐图像和转录组嵌入，CH score显著提升
- **mclSTExp**（2024）：多模态对比学习Transformer+DenseNet-121

**当前状态**：直接使用冻结UNI特征+监督回归，缺乏面向通路预测的判别性预训练。

**实施方案**：

| 子任务 | 内容 | 预期PCC增量 | 时间 |
|--------|------|:---------:|:--:|
| **3.5a** 通路感知SupCon | 同一通路高评分patch为正样本对，低评分为负；使用SupCon损失预训练MLP投影头 | +0.02~0.04 | 2天 |
| **3.5b** 空间邻近对比 | 空间相邻patch（距离<150px）互为增强视图，SimCLR风格对比学习 | +0.01~0.03 | 1.5天 |
| **3.5c** 两阶段训练 | 阶段1：对比学习预训练（仅用图像，无需标签）→ 阶段2：回归微调 | +0.02~0.05 | 2天 |

**训练流程**：
```
阶段1（对比预训练）:
  图像 → UNI → 投影头(1024→256) → SupCon/SimCLR损失
  目标: 学习通路感知的图像表征

阶段2（回归微调）:
  图像 → UNI(冻结) + 预训练投影头 → GNN/Transformer → MLP → HuberLoss
  目标: 精确回归30通路评分
```

#### P1-3 🟡 通路分组预测策略

**文献支撑**：
- **HistoPrism** [44]：UNI+Cross-Attention+Transformer，对泛癌通路进行功能分组分析
- **PEaRL** [43]：通路激活作为表征学习信号，不同通路使用不同注意力模式
- 本组实验数据：通路难度三层分化（结构组PCC>0.7，代谢组0.3-0.55，免疫组<0.3）

**当前状态**：30通路统一预测，未利用通路间的功能关联和难度差异。

**实施方案**：

| 通路组 | 包含通路 | 当前PCC范围 | 专用策略 |
|--------|---------|:---------:|------|

#### P1-4 🟡 HisToGene过拟合缓解

**问题定位**：当前HisToGene-UNI过拟合Gap高达0.26-0.39（单患者Val Gap: 0.2651，跨患者Test Gap: 0.3930），为性能瓶颈之首。

**改进策略**：

| 策略 | 具体措施 | 预期效果 | 实施难度 |
|------|---------|---------|----------|
| 数据增强 | 空间坐标扰动(±5%)、Patch随机翻转/旋转、Mixup(α=0.2) | Gap↓0.05-0.10 | 低 |
| 结构正则 | 增大Dropout(0.1→0.3)、增加LayerDrop(p=0.1) | Gap↓0.03-0.05 | 低 |
| 训练策略 | Label Smoothing(ε=0.1)、CosineAnnealing+Warmup | Gap↓0.02-0.03 | 低 |
| 模型剪枝 | 减少Transformer层数(4→3)或头数(8→6) | Gap↓0.02-0.04 | 中 |
| 知识蒸馏 | 用大模型(多患者联合)蒸馏小模型(单患者) | Gap↓0.05-0.08 | 高 |

**优先级**：数据增强 > 结构正则 > 训练策略 > 模型剪枝 > 知识蒸馏

**实施时机**：与P0改进并行，在Week 1-2即可快速验证前三项

## 六、风险管理与回退方案

### 决策树：当改进方案未达预期时

```
P0-1 Virchow2 PCC < 基线(0.38)?
├── Yes → 回退UNI2-h，节省的时间投入P0-2/P0-3
└── No  → 继续

P0-2 多模态 PCC < 0.42?
├── Yes → 分析原因:
│   ├── 基因嵌入质量差 → 替换为scGPT或简化为one-hot基因集编码
│   ├── 交叉注意力未收敛 → 增加层数/调整学习率/改为late fusion
│   └── 训练数据不足 → 引入3ST公开数据做预训练
└── No  → 继续集成

P1-1 域适应 PCC增量 < 0.01?
├── Yes → 域适应在冻结特征上效果有限，跳过
│         转向P1-2对比学习预训练
└── No  → 继续

全部P0+P1 最终PCC < 0.45?
├── Yes → 备选方案:
│   ├── 方案E1: 引入更多患者数据（3ST公开数据集）
│   ├── 方案E2: 切换到PEaRL/HistoPrism等新架构
│   └── 方案E3: 聚焦子集通路（放弃难通路，报告可预测通路）
└── No  → 目标达成，进入论文撰写
```

### 资源约束应对

| 风险 | 触发条件 | 应对方案 |
|------|---------|----------|
| Virchow2显存超限 | 单卡>24GB | 使用gradient checkpointing / 降低batch / 换用Virchow2-small |
| scGPT权重获取困难 | 申请被拒/延迟>1周 | 切换GenePT（首选）或简单mean pooling of gene embeddings |
| 训练时间超预期 | 单fold>8小时 | 减少epoch(18→10) + 更激进的early stopping(patience=2) |
| 3ST数据质量差 | 预处理后样本<5000 | 放弃3ST，回退到三患者方案 |
