# PEaRL 通路增强框架改进方案

> 版本：v2.0 | 日期：2026.05.12
> 状态：**已重新评估（基于数据限制）** | 基线PCC：0.3812（三折平均）/ 0.4095（Fold1）

---

## ⚠️ 数据限制声明 [v2.0新增]

> **关键约束**：本项目**无法获取原始空间转录组的全基因表达矩阵**。
> 可用标签仅为30条通路的ssGSEA得分（由上游从完整ST数据计算而来）。
> 这意味着所有依赖基因层面信息的方案组件均不可行，PEaRL原始设计需大幅调整。

---

## 一、方案概述

### 1.1 核心思路
在当前HisToGene-UNI Token方案基础上，**借鉴**PEaRL的通路关系建模理念，通过通路间关系感知机制提升30条通路的预测一致性。

### 1.2 预期收益 [v2.0修订]

| 目标 | 当前值 | ~~原预期~~ | **新预期** | 提升幅度 |
|------|--------|-----------|-----------|---------|
| 跨患者PCC（三折平均） | 0.3812 | ~~0.46-0.50~~ | **0.40-0.44** | +5-15% |
| Fold1 PCC | 0.4095 | ~~0.49-0.53~~ | **0.43-0.47** | +5-15% |

## 二、数据限制与质疑回应 [v2.0新增]

### 2.1 质疑原文

> "PEaRL模型的核心机制依赖于'通路表示'，即通过计算通路激活分数来替代原始基因表达作为输入。ssGSEA算法需要利用全基因组或大范围的基因表达谱来进行基因排序和富集分析。如果输入数据仅包含30个基因，将无法计算出有效的通路富集分数，PEaRL的基础架构也就无法成立。"

### 2.2 逐点分析

| # | 质疑观点 | 判定 | 分析 |
|---|---------|------|------|
| 1 | ssGSEA需要全基因组表达谱 | ✅ 正确 | ssGSEA需对全部基因排序后做富集打分 |
| 2 | 30个基因无法计算有效通路分数 | ⚠️ 前提偏差 | 项目有30条**通路得分**，非30个基因；这些是从完整ST数据计算的有效结果 |
| 3 | PEaRL基础架构无法成立 | ⚠️ 部分正确 | 原始PEaRL的完整流程不可行，但核心理念可适配 |

### 2.3 项目实际数据情况

| 数据层 | 是否可用 | 说明 |
|--------|---------|------|
| 全基因表达矩阵（~20000基因） | ❌ 不可获取 | 原始ST数据无法回溯 |
| 30条通路ssGSEA得分 | ✅ 可用 | 已由上游从完整ST数据计算，作为回归标签 |
| UNI2-h图像tokens | ✅ 可用 | 预提取的patch级特征 |
| 通路生物学先验（KEGG/Reactome） | ✅ 可获取 | 公开数据库中通路关系信息 |

### 2.4 对方案的影响总结

- **对比学习退化**：缺少基因层面信息，对比学习从"多层语义对齐"退化为"图像↔30维向量对齐"
- **Pathway Encoder输入贫乏**：仅30维标量输入（原论文处理数千维基因表达）
- **GenePT不可用**：无基因表达数据，基因语义嵌入失去锚定对象
- **预期收益大幅下调**：从+20-30%降至+5-15%

## 三、PEaRL组件可行性重新评估 [v2.0重写]

| 组件 | 原方案设计 | 数据限制下的可行性 | 调整方案 |
|------|-----------|-----------------|---------|
| Pathway Encoder | 编码30通路得分为关系感知嵌入 | ✅ 仍可用，但输入维度仅30 | 保留，作为标签空间关系建模 |
| InfoNCE对比学习 | 图像↔通路嵌入对齐 | ⚠️ 可用但效果打折 | 改用更适合的对比策略 |
| GenePT集成 | 引入基因语义嵌入 | ❌ 无基因表达无法有效利用 | 改为通路语义描述嵌入 |
| 多任务损失 | HuberLoss + InfoNCE | ⚠️ InfoNCE收益不确定 | 保留但降低权重 |

### 保留的PEaRL思想

- ✅ 通路间关系建模的理念（核心价值）
- ✅ 多任务学习框架（回归+正则）
- ✅ 推理时仅用图像分支的设计（无推理开销）
- ❌ 基因层面对比学习（数据不支持）
- ❌ GenePT基因语义嵌入（无基因表达数据）
- ❌ 论文级别的PCC提升预期（数据条件不同）

## 四、替代改进策略 [v2.0新增·重点]

### 策略A：通路关系感知预测头（🟢低难度·推荐）

**核心思想**：将30条通路视为有关系的整体，而非30个独立回归目标。

**实现方案**：
```
图像嵌入 [B, 1024]
    ↓
Transformer Decoder (2层)
  ├── Query: 30个可学习的通路嵌入 [30, 256]
  ├── Key/Value: 图像嵌入投影
  └── Self-Attention Mask: 通路邻接矩阵（来自KEGG/Reactome）
    ↓
通路预测 [B, 30]
```

| 要素 | 说明 |
|------|------|
| 生物学先验 | 利用KEGG/Reactome构建通路邻接矩阵作为attention mask |
| 关键优势 | 不需要基因表达数据，仅改造预测头 |
| 预期收益 | +5-10% PCC |
| 新增参数 | ~0.4M |
| 实现难度 | 🟢 低（改造现有MLP预测头即可） |

### 策略B：通路嵌入对比正则化（🟡中难度）

**核心思想**：简化版PEaRL，仅利用通路得分的分布模式做对比对齐。

**实现方案**：
```
图像分支: 图像嵌入 → 投影头 → [B, 256]
通路分支: 30通路得分 → MLP编码器 → [B, 256]
    ↓
InfoNCE对比对齐
  正样本: 同patch的 (图像, 通路) 对
  负样本: 不同患者的 (图像, 通路) 对（跨患者差异大，自然形成hard negatives）
```

| 要素 | 说明 |
|------|------|
| 关键洞察 | 跨患者差异提供天然hard negatives，无需复杂采样策略 |
| 预期收益 | +3-8% PCC |
| 风险 | 30维空间过小可能导致对比信号弱 |
| 实现难度 | 🟡 中（需新增对比损失模块） |

### 策略C：多尺度通路预测（🟡中难度）

**核心思想**：利用通路层次结构，先预测通路大类再精细预测。

**实现方案**：
```
图像嵌入 [B, 1024]
    ├── 分支1: MLP → 通路大类预测 [B, 5] (免疫/代谢/增殖/信号/修复)
    └── 分支2: MLP → 精细通路预测 [B, 30]
        ↓
分层损失 = MSE_粗 + MSE_细 + λ·一致性约束
```

| 要素 | 说明 |
|------|------|
| 层次关系 | 30条通路按功能分为5大类，建立coarse-fine映射 |
| 一致性约束 | 精细预测聚合应与粗预测一致 |
| 预期收益 | +5-12% PCC |
| 优势 | 与策略A/B正交，可组合验证 |
| 实现难度 | 🟡 中（需定义通路层次映射） |

### 6.2 关键风险

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 30维通路空间太小，关系建模过拟合 | 中 | 高 | 限制Decoder层数、增大dropout |
| 对比学习信号在30维空间过弱 | 中 | 中 | 降低β权重、增大batch size |

### 6.3 止损标准

- **Phase-A**：Fold1 PCC < 0.41 → 策略A无效，终止
- **Phase-B**：叠加后PCC未提升 → 对比学习无增益，仅保留A
- **全局止损**：所有策略均无正向效果 → 转向GigaPath backbone替换
