# MPP2 后续方案与 LoRA 新数据实验建议

日期：2026-07-09

用途：记录团队最新决策，统一后续 agent / 会话引用口径，避免继续把旧三患者实验或已暂停的 MPP 分支当作正式证据。

## 一、当前生效结论

1. **后续 MPP 主线只使用 MPP2**：MPP2 作为后续数据处理、模型训练、汇报解释和新实验设计的默认 MPP 方案。
2. **MPP1 / MPP3 / MPP4 / MPP5 暂时弃用**：这些方案的统一重跑结果只保留为背景分析和方法学对照，不再作为后续主线实验入口。
3. **旧三患者实验结论不再作为证据**：旧 3 患者 LoRA、Token+LoRA、频域和空间修复实验仅可作为调参启发，例如低秩 LoRA、早停窗口、过拟合风险和通路可预测性排序参考；不得作为“已验证有效”的正式性能证据。
4. **当前 MPP1-5 统一重跑用于解释取舍，不用于推翻团队决策**：MPP1 的 PCC 上限、MPP4 的 raw-scale 稳定性仍可写入背景，但后续行动以 MPP2 的队列/外部测试口径一致性为准。

## 三、下一步推荐实验

推荐 P0 实验：**MPP2 新数据 LoRA 优化效果验证**。

实验目标：在 MPP2 口径下验证 LoRA 是否能在新数据上带来稳定增益，而不是复用旧三患者结论。

建议分两步：

1. **先锁定 MPP2 冻结基线**
   如果新数据、标签变换或缓存有任何变化，先按当前统一协议重跑 `UNI2-h frozen features + 2-layer MLP`，作为同批次 baseline。若输入数据完全未变，可暂用 `mpp2_std10val_xzy_ext_uni2h_mlp_20260706` 作基线。

2. **再跑 MPP2 + LoRA r=8 主实验**
   只使用 MPP2 的 train/internal val manifest，固定 external test 为 MPP-2/XZY；选优只看 internal `val_loss`，外部 XZY 只在 best checkpoint 固定后评估一次。

建议判定门：

| 指标 | 通过条件 |
|------|------|
| External XZY PCC | 相比同批次 MPP2 frozen baseline 提升至少 +0.02 |
| Raw-scale MAE/R2 | 不得明显恶化；若 PCC 提升但 raw MAE/R2 变差，标记为排序增益而非部署增益 |
| Val-External gap | 不应显著扩大；若 internal val 很高但 XZY 不升，优先判定为过拟合 |
| 稳定性 | 主实验通过后再追加 seed 43/44，不先展开大矩阵 |

调参边界：LoRA rank=8、低学习率、短早停窗口可以参考旧三患者经验，但这些只是起始超参，不是有效性证据。rank=4/16、dropout、weight decay 等消融放在 MPP2 LoRA r=8 主实验之后。

## 四、后续 agent 引用规则

- 引用 MPP 主线时写：**MPP2 是 2026-07-09 后续唯一主线 MPP 方案**。
- 引用旧三患者结果时必须写：**仅作调参参考，不作正式证据**。
- 引用 MPP1/3/4/5 时必须写：**暂时弃用，仅作背景或方法学对照**。
- 若发现其它文档仍称 CLS LoRA、Token+LoRA 或 MPP1 为“当前主力/最强证据”，应按本文件更新或至少追加过时警示。
