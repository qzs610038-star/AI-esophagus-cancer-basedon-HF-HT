## 2. 第1步：上传项目代码到服务器

### 方式A：通过 Git（推荐）

```bash
# 在服务器终端执行：
cd ~
git clone <你的仓库地址> PFMval
cd PFMval
```

### 方式B：通过远程桌面复制

1. 在本地将 `PFMval_new` 文件夹打包为 ZIP（**排除** `data_new_3ST/`、`*.pth`、缓存目录）
2. 通过远程桌面复制到服务器
3. 在服务器解压：`unzip PFMval.zip -d ~/PFMval`

### 方式C：通过 scp 或 rsync（推荐有经验的用户）

```bash
# 在本地 PowerShell 执行：
rsync -avz --exclude='data_new_3ST' --exclude='*.pth' --exclude='*cache*' `
    "d:\AI空间转录病理研究\PFMval_new\" user@server_ip:~/PFMval/
```

## 3. 第2步：配置服务器环境

### 3.1 安装 Miniconda（如服务器尚无）

```bash
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh
# 安装完成后重新登录或 source ~/.bashrc
```

### 3.2 创建项目虚拟环境

项目已为你准备了环境配置文件，直接使用：

```bash
cd ~/PFMval

# 创建 HisToGene 系列训练环境（主力）
conda env create -f env_histogene.yml

# 创建 EGN-v2 / GAT 训练环境（如需）
```

## 4. 第3步：配置数据路径（关键步骤）

### 4.1 理解路径配置系统

项目中有两类脚本，使用不同的路径配置方式：

| 脚本类型 | 位置 | 路径来源 |
|---------|------|---------|
| **受保护脚本**（不可修改） | `histogene/`、`egnv1/`、`egnv2/` | `config.yaml` 的 `data:` 节 |
| **新训练脚本**（可修改） | 根目录 `train_*.py` | `config.yaml` 的 `paths:` + `patients:` 节 |

### 4.2 编辑 config.yaml

用编辑器打开 `~/PFMval/config.yaml`，找到以下三个部分：

#### 第一步：更新 `data:` 节（给受保护脚本用）

```yaml
data:
  # 将下面的路径改为服务器上的实际位置
  train_patches_dir: "/data/HE_images/HYZ15040/train_patches"
  val_patches_dir:   "/data/HE_images/HYZ15040/val_patches"
  labels_csv_raw:    "/data/ssGSEA/HYZ15040_ssGSEA_zscore.csv"
  labels_csv_zscore: "/data/ssGSEA/HYZ15040_ssGSEA_zscore.csv"
```

> **字段说明**：`train_patches_dir` = 训练集 PNG 所在目录；`val_patches_dir` = 验证集 PNG 所在目录；`labels_csv_raw` = 原始标签文件；`labels_csv_zscore` = Z-score 标准化后的标签文件

#### 第二步：配置 `paths:` 节（给新训练脚本用）

```yaml
paths:
  # 补丁图像根目录（所有患者共用的上层目录）
  patch_base: "/data/HE_images"
  # ssGSEA 标签文件根目录
  ssgsea_base: "/data/ssGSEA"
  # 特征缓存目录（如缓存放在项目目录下，可保持注释）
  caches:
    uni_tokens: "/data/caches/uni_tokens"
    # ... 其他缓存目录按需配置
```

> **提示**：`patch_base` 和 `ssgsea_base` 是上层目录，脚本会自动拼接患者子目录名。

#### 第三步：配置 `patients:` 节（如每位患者路径不同）

```yaml
patients:
  HYZ15040:
    patches_subdir: "HYZ15040_noov_split"    # 子目录名
    labels_csv: "HYZ15040_ssGSEA_zscore.csv"  # CSV 文件名
    # 如果该患者的 HE 图像和标签不在 paths.patch_base 下，直接指定绝对路径：
    patches_dir: "/data/HE_images/HYZ15040"    # 直接指定（可选）
    labels_path: "/data/ssGSEA/HYZ15040_scores.csv"  # 直接指定（可选）
```

#### 路径配置优先级

对于每位患者，脚本按以下顺序查找路径：

```
1. patients.{name}.patches_dir / labels_path（直接指定）
2. paths.patch_base + patients.{name}.patches_subdir（根目录 + 子目录拼接）
3. {PROJECT_ROOT}/data_new_3ST/...（项目本地默认，仅在未配置时使用）
```

### 4.3 验证路径配置

```bash
cd ~/PFMval
conda activate pfmval_histogene
python config_utils.py
```

检查每个路径后是否显示 **[存在]**。如果显示 **[不存在]**，请核对路径是否正确。

完整输出应类似：
```
=======================================================
PFMval 配置系统自检
=======================================================
[OK] 配置文件加载成功

--- 数据路径 (data: 节) ---
  train_patches_dir: /data/HE_images/HYZ15040/train_patches  [存在]
  val_patches_dir: /data/HE_images/HYZ15040/val_patches  [存在]
  ...

--- 患者路径 ---
  HYZ15040: patches[存在]  labels[存在]  cache[存在]
  JFX0729: patches[存在]  labels[存在]  cache[存在]
  LMZ12939: patches[存在]  labels[存在]  cache[存在]
```

## 5. 第4步：拷贝特征缓存数据

特征缓存（`.pt` 文件）是预提取的深度学习特征，直接拷贝比重新生成快得多。

### 5.1 通过远程桌面复制（推荐）

1. 在本地将 `uni2h_cache_tokens/` 文件夹打包为 `.tar.gz`：
   ```powershell
   # 在本地 PowerShell 中执行（需安装 7-Zip）
   tar -czf uni2h_cache_tokens.tar.gz uni2h_cache_tokens/
   ```
2. 通过远程桌面复制到服务器
3. 在服务器解压：`tar -xzf uni2h_cache_tokens.tar.gz -C ~/PFMval/`

### 5.2 通过 scp 传输（推荐有经验用户）

```bash
# 在本地 PowerShell 执行：
scp -r "d:\AI空间转录病理研究\PFMval_new\uni2h_cache_tokens\" user@server_ip:~/PFMval/
scp -r "d:\AI空间转录病理研究\PFMval_new\uni2h_cache\" user@server_ip:~/PFMval/
```

### 5.3 服务器上重新生成（如无法传输）

如果传输不可行，在服务器上运行提取脚本：

```bash
cd ~/PFMval
conda activate pfmval_histogene

# 生成 UNI token 缓存（约需 2-4 小时）
python extract_uni_tokens.py --patient HYZ15040 JFX0729 LMZ12939 --mode lite

# 生成 AugMix 增强缓存（约需 8-12 小时）
python extract_uni_tokens_augmented.py --patient HYZ15040 JFX0729 LMZ12939 --n_augments 4
```

### 5.4 下载 OmiCLIP 权重（如需要）

```bash
python scripts/download_omiclip.py
# 该脚本会自动从 HuggingFace 下载约 7.14 GB 的 checkpoint 文件
```

## 6. 第5步：创建符号链接（给受保护脚本用）

`histogene/`、`egnv1/`、`egnv2/` 目录下的脚本是受保护的（不可修改），它们内部通过 `项目根目录/data_new_3ST/` 拼接路径。如果服务器的数据不在项目根目录下，需要创建符号链接（类似于 Windows 的快捷方式）。

### 6.1 确认数据位置

假设：
- HE 图像在 `/data/HE_images/`（包含 `patch_noov_spilt/` 子目录）
- ssGSEA 在 `/data/ssGSEA/`
- 缓存目录已经拷贝到项目下

### 6.2 创建符号链接

```bash
cd ~/PFMval

# 链接 data_new_3ST 到服务器的实际数据位置
ln -s /data/HE_images data_new_3ST
# 如果 ssGSEA CSV 不在 HE_images 下，需要单独链接
# 先创建 data_new_3ST 目录结构：
mkdir -p data_new_3ST
ln -s /data/ssGSEA data_new_3ST/ssGSEA_zscore

# 如果缓存不在项目目录下，同样链接
# ln -s /data/caches/uni2h_cache_tokens uni2h_cache_tokens
```

### 6.3 验证符号链接

```bash
ls -la data_new_3ST/
# 应看到类似输出：
# lrwxrwxrwx ... data_new_3ST -> /data/HE_images
```

## 8. 常见问题排查

### Q1: `config_utils.py` 显示路径 [不存在]

**原因**：config.yaml 中配置的路径在服务器上不存在。

**解决**：
1. 检查路径拼写是否正确（区分大小写！Linux 大小写敏感）
2. 用 `ls /data/HE_images/` 确认目录确实存在
3. 确认路径是绝对路径（以 `/` 开头）

### Q2: `ImportError: No module named 'torch'`

**原因**：conda 环境未激活或未创建。

**解决**：
```bash
conda activate pfmval_histogene
python -c "import torch; print(torch.__version__)"
```

### Q3: CUDA out of memory

**原因**：GPU 显存不足。

**解决**：减小 batch size：
```bash
python train_histogene_uni_tokens.py --patient HYZ15040 --batch_size 16
```

### Q4: `FileNotFoundError: config.yaml`

**原因**：未在项目根目录执行命令。

**解决**：
```bash
cd ~/PFMval  # 确保在项目根目录
python config_utils.py
```

### Q5: 受保护脚本找不到数据

**原因**：符号链接未创建或创建错误。

**解决**：
```bash
# 检查链接是否正确
ls -la ~/PFMval/data_new_3ST
# 如果链接断了，删除并重建
rm ~/PFMval/data_new_3ST
ln -s /正确的/数据/路径 ~/PFMval/data_new_3ST
```

### Q6: 编码错误（中文乱码）

**原因**：Linux 默认 UTF-8，一般不会出现此问题。Windows 上需要 `PYTHONIOENCODING=utf-8`，**Linux 上不需要**。

## 9. 日常训练命令速查

```bash
# 激活环境
conda activate pfmval_histogene
cd ~/PFMval

# === 单患者训练 ===
# UNI Token（主力）
python train_histogene_uni_tokens.py --patient HYZ15040 --epochs 150
# UNI Token + AugMix
python train_histogene_uni_tokens_augmix.py --patient HYZ15040 --epochs 150
# OmiCLIP
python train_histogene_omiclip.py --patient HYZ15040 --epochs 150
# Virchow2
python train_histogene_virchow2_tokens.py --patient HYZ15040 --epochs 150

# === 跨患者训练（三折交叉验证）===
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 1 --epochs 150
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 2 --epochs 150
python train_histogene_uni_tokens_augmix.py --cross_patient --fold 3 --epochs 150

# === 特征提取（更换 backbone 时运行一次）===
python extract_uni_tokens.py --patient HYZ15040 JFX0729 LMZ12939 --mode lite
python extract_omiclip_features.py --patient HYZ15040 JFX0729 LMZ12939

# === 可视化 ===
python config_utils.py  # 检查配置
```

> **Linux 上不需要 `PYTHONIOENCODING=utf-8` 前缀！** 这与 Windows 不同。

## 附录：路径配置速查表

| 配置位置 | 用途 | 影响哪些脚本 |
|---------|------|-------------|
| `config.yaml` → `data:` 节 | 受保护脚本的数据路径 | `histogene/train.py`、`egnv2/train.py` 等 |
| `config.yaml` → `paths:` 节 | 新脚本的数据根目录 | `train_histogene_*.py` 等 |
| `config.yaml` → `patients:` 节 | 每位患者的子路径或覆盖 | `train_histogene_*.py` 等 |
| `config.yaml` → `patients.{name}.patches_dir` | 该患者 HE 图像绝对路径（最高优先级） | 所有新脚本 |
| `config.yaml` → `patients.{name}.labels_path` | 该患者标签文件绝对路径（最高优先级） | 所有新脚本 |
| 符号链接 `data_new_3ST → /data/...` | 让受保护脚本找到数据 | `histogene/`、`egnv1/`、`egnv2/` |
