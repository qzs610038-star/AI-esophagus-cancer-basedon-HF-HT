# Meeting Report Guide — 组会汇报撰写指南

帮助撰写面向"医学背景但无 AI 背景"听众的组会汇报。

> 核心目标：让医生也能快速理解研究进展。详见 `.qoder/skills/组会汇报撰写指南.md`（完整版）。

## 调用格式

```
/meeting-report              # 完整指南
/meeting-report structure    # 文档结构模板
/meeting-report analogies    # 技术概念类比库
/meeting-report qa           # 专业听众 Q&A 预判
/meeting-report checklist    # 完成后检查清单
```

## 一、核心写作原则

1. **结论前置**：每节第一句话即结论
2. **医生可读**：技术原理用类比/比喻解释，首次出现的术语附括号注释
3. **术语克制**：一段话不超过 3 个专业术语
4. **数据可视化优先**：表格 > 文字，图表 > 表格
5. **数据真实性**：所有数据来自实际训练结果 CSV，标注来源路径
6. **学术准确性**：技术描述对齐原论文，不确定的先查再写

## 二、文档结构模板

```markdown
# 组会汇报_[主题]_[日期]

## 摘要
> 2-3 句概括核心发现

## 一、研究背景（1 页内）
## 二、方法概述（架构图 + 类比解释）
## 三、实验结果
### 3.1 主要指标对比（表格）
### 3.2 逐通路分析（Top 5 / Bottom 5）
### 3.3 跨患者泛化（如适用）
### 3.4 消融实验（如适用）
## 四、讨论与结论
## 附录：详细参数表 + 完整通路 PCC + 预判 Q&A
```

## 三、技术概念类比库

撰写汇报时直接复用以下类比：

| 技术概念 | 医生友好解释 |
|---------|------------|
| PCC | 预测值与真实值的吻合程度，1.0=完美预测 |
| 过拟合 | 学生只背答案没学方法，换卷子就考砸 |
| 跨患者泛化 | 在 A 患者身上学到的经验能否用于 B 患者 |
| UNI 特征提取器 | AI 的"眼睛"，从病理图像看出有意义信息 |
| Dropout | 训练时随机"遮住一只眼"，迫使不依赖单一特征 |
| ViT | 把图像切成小块逐块分析，类似医生用放大镜看切片 |
| 消融实验 | 逐个去掉模型组件看效果变化，排查哪根线路出问题 |
| 预训练模型 | 已在海量数据上"预习"过的模型，迁移到新任务起步更高 |
| 收敛 | 模型性能趋于稳定不再提升，类似学习到了瓶颈 |

> 完整类比库（25 个术语）见 `.qoder/skills/组会汇报撰写指南.md` §三。

## 四、数据呈现规范

- 指标表标注方向：↑ 越高越好，↓ 越低越好
- 保留 3 位小数
- 最佳 epoch 基于 **val_loss 最小**（非 val_pcc 最大）
- 通路名使用全名+中文，禁止索引号（如 `Pathway_0`）
- 模型命名统一格式：`模型名-特征类型`（如 `HisToGene-UNI`）
- **EGN-v1 不参与正式对比**

## 五、预测困难归因类型

| 类型 | 说明 |
|------|------|
| 微环境依赖型 | 通路活性由细胞间信号调控，单 patch 无法捕获 |
| 分子层面型 | 由蛋白修饰/表观遗传决定，H&E 无法反映 |
| 异质性型 | 同一形态学下通路活性差异大 |
| 低动态范围型 | 所有样本活性接近，预测与随机无异 |

## 六、检查清单

- [ ] 摘要（2-3 句核心发现）
- [ ] 每节开头有结论句
- [ ] 所有术语首次出现有通俗解释
- [ ] 数据来自实际 CSV，来源路径已标注
- [ ] 最佳 epoch 标准统一（val_loss 最小）
- [ ] 通路名使用全名+中文
- [ ] EGN-v1 不参与正式对比
- [ ] 模型命名统一
- [ ] 预判 ≥ 3 个专业听众提问并准备回答

## 七、专业听众 Q&A 预判

在附录中准备高频问题：

| 类型 | 典型问题 | 准备方向 |
|------|---------|---------|
| 实验设计 | "为什么只跑 Fold 1？" | 快速验证策略：单折 15min vs 三折 45min |
| 结果可靠性 | "差异显著吗？" | 同参数两次运行波动 ~0.0095 |
| 方法选择 | "为什么用 HuberLoss？" | 对异常值鲁棒，MSE 实验验证 |
| 负结果归因 | "是不是数据太小？" | 承认局限性 + 15K 样本是当前可获得的最大数据集 |
| 文献对比 | "跟 SOTA 比？" | 准备 1-2 篇顶会论文的核心结论和差距 |

> 完整 Q&A 指南（含 5 种提问类型 + 回答模板）见 `.qoder/skills/组会汇报撰写指南.md` §九。
