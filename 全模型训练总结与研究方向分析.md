## 第一部分：项目背景与实验概述

### 1.1 项目目标简介

本项目属于**空间转录组学（Spatial Transcriptomics）**与**计算病理学（Computational Pathology）**的交叉领域，核心目标是：

> **从病理切片图像（H&E 染色）直接预测基因通路表达活性评分。**

通俗地说，就是教会计算机"看"病理切片——就像病理科医生在显微镜下观察组织形态一样——然后让计算机告诉我们：**这个区域的细胞正在活跃地进行哪些生物学过程？** 例如，某个组织区域的炎症反应有多强？细胞增殖速度快不快？伤口愈合过程是否在进行？

**为什么这很有价值？**
- 传统的基因表达检测需要将组织切片、提取 RNA，成本高且破坏样本
- H&E 染色是临床常规操作，几乎每个病理样本都有
- 如果能从图像推断分子层面的活动，就能在不增加成本的情况下获得大量分子信息

### 1.2 三个数据集介绍

本项目使用了三例癌症患者的病理切片数据，每位患者的数据包含数千个 224x224 像素的图像小块（patch）及其对应的 30 条基因通路活性评分。

| 数据集 | 患者代号 | 训练样本 | 验证样本 | 总计 | 数据划分 |
|--------|---------|---------|---------|------|---------|
| **HYZ15040** | HYZ | 2,390 | 265 | **2,655** | 90% / 10% |
| **JFX0729** | JFX | 7,055 | 783 | **7,838** | 90% / 10% |
| **LMZ12939** | LMZ | 6,762 | 751 | **7,513** | 90% / 10% |
| **联合训练（MultiPatient_3ST）** | 3 患者合并 | 16,207 | 1,799 | **18,006** | 90% / 10% |

**数据规模解读（给初学者）：**
- HYZ15040 是最小的数据集（约 2,600 个 patch），对于深度学习来说属于"小数据"
- JFX0729 和 LMZ12939 样本量较大（约 7,500-7,800 个 patch），属于中等规模
- 联合训练将三者合并，达到约 18,000 个样本，是单数据集的 2-3 倍

**关于数据划分**：我们采用 90% 训练 / 10% 验证的划分方式。验证集用于评估模型在未见数据上的泛化能力——就像学生做完作业后，用一套老师没讲过的练习题来检验自己是不是真的学会了。

### 1.4 评估指标解释

我们用四个指标来评估模型的预测质量：

#### PCC（Pearson Correlation Coefficient，皮尔逊相关系数）
- **含义**：衡量预测值与真实值之间的线性相关程度
- **取值范围**：[-1, 1]，越接近 1 表示预测越准确
- **通俗解释**：想象你在画散点图，横轴是真实值，纵轴是预测值。PCC = 1 意味着所有点都完美地落在一条从左下到右上的直线上；PCC = 0 意味着点完全随机分布，预测和真实值毫无关系
- **本项目的意义**：PCC = 0.5 意味着预测和真实值有中等强度的正相关，模型捕捉到了一部分规律，但还有很多误差

#### R2（R-squared，决定系数）
- **含义**：衡量模型解释了目标变量多少比例的方差
- **取值范围**：(-inf, 1]，越接近 1 越好。R2 = 0 意味着模型预测效果等同于"总是预测平均值"
- **通俗解释**：如果 R2 = 0.3，说明模型解释了目标变量 30% 的变化，剩下 70% 的变化是模型无法解释的（可能是噪声，也可能是模型没学到的复杂规律）
- **注意**：在回归任务中，R2 可能为负数，说明模型比"瞎猜平均值"还差

#### MSE（Mean Squared Error，均方误差）
- **含义**：预测值与真实值差的平方的平均值
- **通俗解释**：放大对大误差的惩罚。如果预测偏差 2 分，MSE 记为 4；偏差 3 分，记为 9。所以 MSE 对大误差特别敏感
- **本项目的意义**：由于我们的标签经过 Z-score 标准化（均值为 0，标准差为 1），MSE 的绝对值大小可以直接比较。MSE = 0.3 意味着平均误差约为 sqrt(0.3) = 0.55 个标准差

#### MAE（Mean Absolute Error，平均绝对误差）
- **含义**：预测值与真实值差的绝对值的平均
- **通俗解释**：直观反映"平均偏差有多大"。MAE = 0.5 意味着预测值平均偏离真实值 0.5 个标准差
- **与 MSE 的区别**：MAE 对大误差不那么敏感，更能反映典型误差水平

**为什么 PCC 是主要指标？**
- 在空间转录组领域，PCC 是最常用的评估指标，因为它衡量的是**相关性**而非绝对误差
- 我们关心的是"模型能否正确排序通路活性高低"，而不仅仅是"数值有多接近"
- 例如，如果真实值是 [1, 2, 3]，预测值是 [2, 3, 4]，PCC = 1（完美相关），但 MAE = 1（有偏差）——在这种情况下，PCC 更能反映模型的有用性

## 第二部分：模型架构详细对比

### 2.1 三种模型概览

本项目训练和对比了三个深度学习模型。它们代表了从病理图像预测基因表达的三种不同技术路线：

| 维度 | **HisToGene** | **EGNv2** | **EGN-v1** |
|------|--------------|-----------|------------|
| **特征提取器** | ViT-Large（自实现，patch_size=16） | ResNet-50（ImageNet 预训练） | ViT-Large（自实现，patch_size=32） |
| **GNN 类型** | 无（不使用图神经网络） | GraphSAGE（2 层） | GCN（2 层） |
| **图构建方式** | 坐标嵌入（x_embed + y_embed） | 空间半径图（radius=300） | KNN 图（基于特征相似性） |
| **核心创新** | 坐标编码 + Transformer | Exemplar Learning + 图卷积 | Exemplar Learning + GCN |
| **参数量** | **~70.6M** | **~3.0M** | **~6.8M** |
| **空间建模** | 隐式（通过坐标嵌入） | 显式（图卷积聚合邻居） | 半显式（KNN 图基于特征） |

**初学者提示**：参数量（Parameter Count）是模型中可学习参数的总数。一般来说，参数量越大，模型的"容量"越大，能学习更复杂的模式；但也更容易在数据不足时"死记硬背"（过拟合）。

### 2.2 HisToGene：ViT Transformer + 位置编码 + MLP

**论文出处**：Pang et al., "Predicting gene expression from histology images through adaptive learning of spatial transcriptomics", *Nature Communications* 2025

**架构工作流程（通俗类比）：**

想象一位病理科医生在看一整张病理切片。HisToGene 的工作方式就像这位医生在做以下事情：

1. **分块观察（Patch Embedding）**：将 224x224 的图像切成 16x16 的小块（共 14x14=196 个小块），就像医生将切片分成若干视野逐一观察
2. **给每块贴标签（Patch Embedding 线性映射）**：将每个小块转换为一个 1024 维的数字描述
3. **记住位置（位置编码）**：除了图像内容，还记录每个小块在整张切片中的坐标位置（x, y）。这就像医生说"这个视野在切片的左上角"——位置信息很重要，因为肿瘤中心与边缘的组织特征不同
4. **综合分析（Transformer）**：通过 8 层 Transformer 编码器，让模型"看完全部视野后做综合判断"。自注意力机制允许模型在处理某一块时，参考其他所有块的信息——就像经验丰富的医生看到一个可疑区域时，会自动联想切片其他区域的特征
5. **输出预测（MLP 回归头）**：最后输出 30 个通路的活性评分

**关键组件详解：**

```
输入图像 (224x224)
    |
    v
分块 + 嵌入 (196 patches x 1024 dim)
    |
    v
+ CLS token (用于聚合全局信息)
+ 坐标嵌入 (x, y 位置信息)
+ ViT 位置嵌入 (patch 在图像中的相对位置)
    |
    v
Transformer Encoder x 8 层
    |---- 每层: 多头自注意力(16头) + 前馈网络 + 残差连接
    |
    v
取 CLS token 输出 --> MLP 回归头 --> 30 通路预测
```

**HisToGene 的核心优势：**
- **全局感受野**：自注意力机制让每个 patch 都能看到其他所有 patch，能捕捉长距离依赖
- **坐标信息融入**：显式编码空间位置，模型知道"在哪里"
- **参数量大（70.6M）**：表达能力强大，在大数据集上表现优异

**HisToGene 的潜在弱点：**
- **过拟合风险高**：70.6M 参数对 2,000-7,000 样本的数据集来说非常庞大
- **没有显式图建模**：空间关系仅通过坐标嵌入引入，不如图卷积直接

### 2.3 EGNv2：ResNet-50 + GraphSAGE + Exemplar Learning

**论文出处**：Yan Yang et al., "Spatial transcriptomics analysis of gene expression prediction using exemplar guided graph neural network", *Pattern Recognition* 2024

**架构工作流程（通俗类比）：**

EGNv2 的工作方式更像一个"社区投票系统"：

1. **提取身份特征（ResNet-50）**：每个 patch 先通过 ResNet-50 提取一个 2048 维的特征向量，就像给每个社区居民发一张"身份卡"
2. **找邻居（空间图构建）**：以每个 patch 为中心，在半径 300 像素范围内找邻居，建立"邻里关系网"——这就是**图结构**
3. **邻里交流（GraphSAGE）**：每个 patch 与邻居交换信息，更新自己的表示。就像社区居民开会讨论，了解邻居的情况后调整自己的认知
4. **请教专家（Exemplar Learning）**：从训练集中构建一个"专家库"，预测时找出与当前 patch 最相似的 10 个训练样本（代表），参考它们的已知通路评分
5. **综合决策（MLP 回归头）**：将图卷积后的特征与代表信息融合，输出最终预测

**关键组件详解：**

```
输入图像 (224x224)
    |
    v
ResNet-50 特征提取 (输出 2048 维)
    |---- 前 3 个 layer 冻结（使用 ImageNet 预训练知识）
    |
    v
特征投影层 (2048 -> 512 维)
    |
    v
GraphSAGE 图卷积 x 2 层
    |---- 输入: 节点特征 + 空间邻接图 (edge_index)
    |---- 每层: SAGEConv + LayerNorm + 残差连接 + GELU
    |
    v
Exemplar 融合 (可选)
    |---- 从代表库检索 K 近邻
    |---- 将图特征与代表特征拼接融合
    |
    v
MLP 回归头 --> 30 通路预测
```

**EGNv2 的核心创新——Exemplar Learning（代表学习）：**

> 代表学习是一种"记忆-检索"机制。模型在训练时记住所有训练样本的特征和标签；预测时，对于新样本，找出训练集中最相似的几个"代表"，参考它们的标签来做预测。
>
> 类比：就像一位经验丰富的老中医，看过成千上万病人后，遇到新病人时，会在脑海中搜索"这个病人的症状和最像哪个以前的病人"，然后参考那个病人的治疗方案。

**EGNv2 的核心优势：**
- **轻量级**：仅 3.0M 可训练参数，过拟合风险低
- **显式空间建模**：GraphSAGE 直接聚合空间邻居信息
- **代表学习提供正则化**：KNN 检索引入隐式数据增强

**EGNv2 的潜在弱点：**
- **特征提取器较弱**：ResNet-50 在 ImageNet（自然图像）预训练，缺乏病理学语义
- **感受野受限**：GraphSAGE 的感受野取决于图半径（300 像素），无法捕捉远距离依赖

### 2.4 EGN-v1：ViT-Large + GCN + Exemplar Bridging

**论文出处**：Yan Yang et al., "Exemplar Guided Deep Neural Network for Spatial Transcriptomics Analysis of Gene Expression Prediction", *WACV* 2023

**架构工作流程：**

EGN-v1 是 EGNv2 的前身，设计理念类似，但在几个关键组件上有所不同：

1. **特征提取**：使用自实现的 ViT-Large（patch_size=32, depth=8, dim=1024），而非 ResNet-50
2. **图卷积**：使用 GCN（Graph Convolutional Network）而非 GraphSAGE
3. **图构建**：使用 KNN 图（基于特征相似性）而非空间半径图
4. **代表机制**：Exemplar 融合方式略有不同

**EGN-v1 与 EGNv2 的关键差异：**

| 差异点 | EGN-v1 | EGNv2 | 影响 |
|--------|--------|-------|------|
| 特征提取器 | 自实现 ViT-Large | ResNet-50 (ImageNet 预训练) | v1 无预训练权重，特征质量依赖随机初始化 |
| GNN | GCN | GraphSAGE | GCN 是转导学习（需全图），GraphSAGE 是归纳学习（支持小批量） |
| 图类型 | KNN 图（特征相似性） | 空间半径图（坐标距离） | KNN 图可能忽略空间连续性 |
| Dropout | 0.5 | 0.3 | v1 正则化更强 |
| Batch size | 16 | 64 | v1 ViT 显存占用大，只能用小 batch |
| Learning rate | 1e-5 | 1e-4 | v1 更保守的优化 |

**EGN-v1 的劣势（从实验结果可见）：**
- ViT-Large 自实现版本没有预训练权重，特征提取能力远弱于 ImageNet 预训练的 ResNet-50
- GCN 的转导学习特性不适合多患者联合训练
- 训练不稳定的迹象：训练 PCC 低于验证 PCC（负 Gap），说明模型可能欠拟合或优化困难

### 2.5 架构对比总结表

| 对比维度 | HisToGene | EGNv2 | EGN-v1 |
|---------|-----------|-------|--------|
| **特征提取器** | ViT-Large (自实现, 16x16 patch) | ResNet-50 (ImageNet 预训练) | ViT-Large (自实现, 32x32 patch) |
| **GNN 类型** | 无 | GraphSAGE (归纳式) | GCN (转导式) |
| **图构建方式** | 坐标嵌入 | 空间半径图 (r=300) | KNN 特征图 |
| **空间建模强度** | 弱（隐式） | 强（显式图卷积） | 中（基于特征相似性） |

| **训练 batch size** | 64 | 64 | 16 |
| **正则化强度** | 中 (dropout=0.3) | 中 (dropout=0.3) | 强 (dropout=0.5) |

## 第三部分：训练结果全面对比

3. EGN-v1 的 GCN 架构不适合多患者联合训练，验证了 EGNv2 使用 GraphSAGE 的改进必要性

| EGNv2 | 1.0x | ~72 | **72x** | ResNet-50 + 图卷积，中等速度 |
| EGN-v1 | 2.0x | ~150 | **300x** | 自实现 ViT 效率低，batch size 仅 16 |

**结论**：HisToGene 虽然单次 epoch 慢，但因收敛快，总训练时间反而最短。EGN-v1 总训练时间最长，但效果最差。

### 3.6 过拟合分析：训练集 vs 验证集 PCC 差距

过拟合（Overfitting）是机器学习中描述"模型在训练数据上表现很好，但在新数据上表现差"的现象。Gap = Train PCC - Val PCC 是衡量过拟合的直观指标。

#### 过拟合程度分级

| Gap 范围 | 过拟合程度 | 说明 |
|---------|-----------|------|
| < 0 | **欠拟合** | 训练 PCC 反而低于验证 PCC，模型没学好 |
| 0 - 0.05 | **轻微/正常** | 健康的泛化差距 |
| 0.05 - 0.10 | **中等** | 有一定过拟合，但在可接受范围 |
| 0.10 - 0.20 | **严重** | 明显过拟合，需要正则化或更多数据 |
| > 0.20 | **非常严重** | 模型在训练集上严重"死记硬背" |

#### 各模型过拟合情况

| 模型 | HYZ15040 | JFX0729 | LMZ12939 | Multi | 平均 Gap | 过拟合评级 |
|------|---------|---------|---------|-------|---------|-----------|
| **HisToGene** | 0.2074 | 0.1914 | 0.1848 | 0.2051 | **0.1972** | **非常严重** |
| **EGNv2** | 0.0375 | 0.0540 | 0.0431 | 0.0528 | **0.0469** | **轻微/正常** |
| **EGN-v1** | -0.0861 | -0.0447 | -0.0399 | -0.0381 | **-0.0522** | **欠拟合** |

**关键发现**：

1. **HisToGene 过拟合极其严重**：平均 Gap 0.20，意味着训练 PCC 0.75，但验证 PCC 仅 0.55。这相当于一个学生在"模拟题"上考 95 分，但"正式考试"只考 75 分——说明他记住了模拟题答案，而非真正掌握了知识。

2. **EGNv2 过拟合控制极佳**：平均 Gap 仅 0.05，训练 PCC 和验证 PCC 非常接近。这说明 EGNv2 的 3.0M 参数在当前数据量下没有过拟合风险——但这同时也暗示：**模型容量不足**，无法充分学习数据中的复杂模式。

3. **EGN-v1 欠拟合**：负 Gap 意味着模型在训练集上的表现还不如验证集。这通常发生在：学习率过低、正则化过强、模型容量不足或训练不充分时。对于 EGN-v1，最可能的原因是学习率 1e-5 过于保守，加上 dropout=0.5 的强烈正则化，导致模型"不敢"充分拟合训练数据。

**过拟合与性能的关系**：

```
HisToGene: 高容量 (70.6M) + 中等数据 (~7K) --> 严重过拟合 (Gap 0.20) --> Val PCC 0.55
EGNv2:     低容量 (3.0M)  + 中等数据 (~7K) --> 轻微过拟合 (Gap 0.05) --> Val PCC 0.41
EGN-v1:    低容量 (6.8M)  + 强正则化       --> 欠拟合 (Gap -0.05)    --> Val PCC 0.25
```

这个三角关系揭示了一个核心矛盾：**过拟合最严重的 HisToGene 反而取得了最好的验证性能**。这说明：
- 当前任务需要较大的模型容量才能充分学习病理图像到基因通路的复杂映射
- HisToGene 的过拟合问题可以通过更强的正则化来缓解，而不应简单地减小模型
- EGNv2 和 EGN-v1 的轻量级设计在当前任务上"矫枉过正"，牺牲了太多表达能力

## 第四部分：关键发现与结论

基于以上 12 次训练、30 条通路的全面数据分析，我们总结出以下 8 条核心发现：

### 发现 1：HisToGene 是当前最优模型，但过拟合是最大瓶颈

**数据支撑**：
- HisToGene 在 12 次训练中的 11 次取得最高 Val PCC（唯一例外是 DNA_Damage_Response 单通路）
- 平均 Val PCC 0.55，领先第二名 EGNv2 约 0.14（相对领先 34%）
- 但过拟合 Gap 高达 0.20，训练 PCC 0.75 vs 验证 PCC 0.55

**解读**：HisToGene 的 70.6M 参数使其具有强大的特征学习能力，但当前数据量（最大 7,800 样本）不足以"喂饱"这个庞然大物。模型记住了训练数据的细节，包括噪声，导致在新数据上表现打折。

### 发现 2：EGNv2 全面落后 HisToGene，但过拟合控制优异

**数据支撑**：
- EGNv2 平均 Val PCC 0.41，落后 HisToGene 0.14
- 但过拟合 Gap 仅 0.05，是 HisToGene 的 1/4
- 在 30 条通路中，28 条落后于 HisToGene

**解读**：EGNv2 的轻量级设计（3.0M 参数）成功避免了过拟合，但也导致**模型容量不足**。ResNet-50 提取的 ImageNet 特征缺乏病理学语义，尤其在免疫/炎症通路上表现极差（Inflammatory_Response 差距高达 0.38）。

### 发现 3：EGN-v1 在当前数据上表现最差，验证了"大模型+无预训练"的风险

**数据支撑**：
- EGN-v1 平均 Val PCC 仅 0.25，不到 HisToGene 的一半
- 所有训练均出现负 Gap（欠拟合），训练 PCC 低于验证 PCC
- 即使联合训练（16K 样本）也仅达到 PCC 0.25

**解读**：EGN-v1 的自实现 ViT-Large 没有预训练权重支持，在 2K-7K 样本的数据集上完全无法发挥 ViT 的潜力。这验证了一个重要原则：**大模型架构必须配合大规模预训练才能生效**。没有预训练的 ViT-Large 甚至不如 ImageNet 预训练的 ResNet-50。

### 发现 4：联合训练提升有限，数据异质性是主要障碍

**数据支撑**：
- HisToGene：单训均值 0.5497 -> 联合 0.5569（+0.0072，+1.3%）
- EGNv2：单训均值 0.4110 -> 联合 0.4245（+0.0135，+3.3%）
- EGN-v1：单训均值 0.2532 -> 联合 0.2460（-0.0072，-2.8%）

**解读**：三个患者的数据合并后，模型并未从中获得预期的显著提升。原因包括：
1. **域偏移（Domain Shift）**：不同患者的组织类型、染色批次、扫描参数存在差异
2. **坐标系统冲突**：HisToGene 的坐标嵌入假设统一坐标空间，合并后产生混淆
3. **图结构割裂**：EGNv2/EGN-v1 的空间图在患者边界处断开，无法跨患者传播信息

### 发现 5：免疫/炎症通路是所有模型的共同短板，但 HisToGene 相对最强

**数据支撑**：
- 在 Inflammatory_Response 上，HisToGene PCC=0.62，EGNv2=0.24，EGN-v1=0.13
- 在 ifng 上，HisToGene PCC=0.55，EGNv2=0.26，EGN-v1=0.20
- 在 icp 上，所有模型 PCC 均 < 0.31（HisToGene=0.30，EGNv2=0.23，EGN-v1=0.07）

**解读**：免疫/炎症相关的通路需要从 H&E 图像中推断细胞类型和功能状态，这是一个间接的、多步骤的推理过程。ViT 的全局自注意力机制能捕捉长距离依赖和细胞排列模式，因此在免疫通路上相对更强；ResNet 的局部卷积则难以完成这种全局推理。

### 发现 6：基质/纤维化通路最容易预测，且各模型差距较小

**数据支撑**：
- ECM_Organization：HisToGene=0.80，EGNv2=0.65，EGN-v1=0.44
- Fibrosis：HisToGene=0.77，EGNv2=0.60，EGN-v1=0.38
- Wound_Healing：HisToGene=0.75，EGNv2=0.57，EGN-v1=0.34

**解读**：基质/纤维化通路在 H&E 图像中有明确的形态学对应（胶原沉积、纤维组织增生），且空间连续性强。即使特征提取能力较弱的 ResNet-50 也能捕捉这些低层次视觉特征。

### 发现 7：数据量与模型性能呈正相关，但存在天花板效应

**数据支撑**：
- HisToGene：HYZ(0.52) < LMZ(0.53) < JFX(0.60)，与样本量排序一致
- EGNv2：HYZ(0.40) < LMZ(0.38) < JFX(0.44)，基本符合（LMZ 略反常）
- EGN-v1：HYZ(0.23) < LMZ(0.22) < JFX(0.31)，JFX 优势明显

**解读**：对于 HisToGene 和 EGN-v1，更大的数据集（JFX 7,800 vs HYZ 2,600）确实带来了更好的性能。但提升幅度并非线性——从 HYZ 到 JFX，样本量增加了 3 倍，但 HisToGene PCC 仅从 0.52 提升到 0.60（+15%），存在明显的**收益递减**。

### 发现 8：模型排名在不同数据集上保持稳定

**数据支撑**：
- 在所有 4 个数据集配置（HYZ、JFX、LMZ、Multi）上，排名均为：HisToGene > EGNv2 > EGN-v1
- HisToGene 与 EGNv2 的差距范围：0.11（HYZ）- 0.16（JFX）

**解读**：三种模型的相对排名具有高度一致性，说明性能差异主要来自**架构本身的能力差异**，而非数据集特性或训练随机性。HisToGene 的 Transformer 架构在当前任务上具有固有优势。

### 模型排名与推荐场景

| 排名 | 模型 | 平均 Val PCC | 适用场景 | 不推荐场景 |
|------|------|------------|---------|-----------|
| **1** | **HisToGene** | **0.5515** | 追求最高预测精度的场景；有充足计算资源 | 需要严格避免过拟合的临床部署；超小数据集（<1K） |
| **2** | **EGNv2** | **0.4144** | 资源受限环境；需要快速训练；作为集成模型的一员 | 对免疫/炎症通路预测精度要求高的场景 |
| **3** | **EGN-v1** | **0.2512** | 学术研究（理解 Exemplar Learning 演进）；作为基线对照 | 任何需要实际预测精度的应用场景 |

### 性能瓶颈分析

```
当前性能瓶颈（从大到小排序）：

1. 【最大瓶颈】特征提取器缺乏病理学预训练
   - ResNet-50 在 ImageNet（自然图像）训练，不懂病理学语义
   - 自实现 ViT 无预训练权重，特征质量差
   - 证据：免疫/炎症通路全面落后，而这些通路需要病理学知识

2. 【次大瓶颈】HisToGene 过拟合严重
   - 70.6M 参数 vs 7K 样本，容量-数据比严重失衡
   - 证据：Gap 0.20，训练 PCC 0.75 vs 验证 PCC 0.55

3. 【第三瓶颈】多患者数据异质性未处理
   - 联合训练提升仅 +1.3%~3.3%，远低于预期
   - 证据：三患者合并后性能没有质的飞跃

4. 【第四瓶颈】困难通路缺乏形态学对应
   - icp、Apoptosis、tgfb 等通路 PCC < 0.30
   - 证据：所有模型在这些通路上都表现差，说明是任务本身的难度
```

## 第五部分：各模型后续改进方向

### 5.1 HisToGene 改进方向

HisToGene 是当前最优模型，主要问题是过拟合。改进目标是：在保持预测精度的同时降低过拟合 Gap。

#### 方向 1：数据增强减轻过拟合

**原理解释**：数据增强通过对训练图像进行随机变换（旋转、翻转、颜色扰动等），人为扩充训练数据多样性，让模型无法"死记硬背"原始样本。

**具体措施**：
- **随机旋转/翻转**：H&E 切片无方向性，90/180/270 度旋转和水平/垂直翻转均合理
- **颜色扰动**：模拟不同批次的染色差异（H&E 染色存在显著批次间差异）
- **Macenko 归一化**：标准化染色矩阵，消除染色差异
- **MixUp/CutMix**：将两个训练样本混合，强制模型学习插值表示

**预期效果**：
- 过拟合 Gap 从 0.20 降至 0.10-0.15
- Val PCC 可能提升 +0.01~0.03（因为模型能学到更鲁棒的特征）

**实施难度**：低
**所需时间**：0.5-1 天

#### 方向 2：更强的正则化

**原理解释**：正则化是在训练过程中对模型复杂度施加惩罚，防止模型过度拟合训练数据的噪声。

**具体措施**：
- **增大 dropout**：从 0.3 提高到 0.4-0.5
- **增大 weight decay**：从 1e-4 提高到 1e-3
- **标签平滑（Label Smoothing）**：对 Z-score 标签添加微小噪声
- **随机深度（Stochastic Depth）**：随机跳过某些 Transformer 层

**预期效果**：
- 过拟合 Gap 从 0.20 降至 0.12-0.16
- Val PCC 可能轻微下降（-0.01）或持平，但泛化稳定性大幅提升

**实施难度**：低
**所需时间**：0.5 天

#### 方向 3：学习率调度优化

**原理解释**：学习率决定了模型参数每次更新的步长。合适的学习率调度可以帮助模型在训练初期快速收敛，在后期精细调整。

**具体措施**：
- **Warmup + Cosine Annealing**：训练初期学习率从小逐渐增大（warmup），然后按余弦曲线衰减
- **分层学习率**：ViT 特征提取器使用较小学习率（1e-5），回归头使用较大学习率（1e-4）

**预期效果**：
- 收敛更稳定，最终 Val PCC 提升 +0.01~0.02

**实施难度**：低
**所需时间**：0.5 天

#### 方向 4：预训练特征替换（UNI / CONCH）

**原理解释**：病理基础模型（如 UNI、CONCH）在大规模病理切片上预训练，提取的特征包含丰富的病理学语义（细胞类型、组织结构、病变模式等），远优于 ImageNet 预训练或从头训练的 ViT。

**具体措施**：
- 将 HisToGene 的 Patch Embedding + Transformer 替换为 UNI 的特征提取器
- 保留 HisToGene 的坐标嵌入和回归头设计
- UNI 输出 1024 维特征，与当前维度兼容

**预期效果**：
- Val PCC 提升 +0.05~0.10
- 免疫/炎症通路（ifng, Inflammatory_Response）提升最显著（+0.10~0.20）

**实施难度**：中
**所需时间**：2-3 天

#### 方向 5：模型集成与交叉验证

**原理解释**：通过训练多个略有差异的模型（如不同随机种子、不同数据划分），然后取平均预测，可以降低方差、提升稳定性。

**具体措施**：
- 5-fold 交叉验证：将数据分为 5 份，轮流用 4 份训练、1 份验证
- 多模型集成：训练 3-5 个 HisToGene（不同初始化），取平均预测

**预期效果**：
- Val PCC 提升 +0.01~0.03
- 预测稳定性大幅提高

**实施难度**：中
**所需时间**：2-3 天

### 5.2 EGNv2 改进方向

EGNv2 的核心问题是**特征提取能力不足**。改进应聚焦于提升上游特征质量，其次优化 GNN 架构。

#### 方向 1：特征提取器升级（UNI / CONCH / DINO-v2）

**原理解释**：当前 ResNet-50 在 ImageNet（自然图像：猫、狗、汽车等）上预训练，提取的特征偏向颜色、纹理、边缘等低级视觉特征。病理图像需要识别细胞核形态、组织结构、有丝分裂相等**病理学语义**，这些知识只能通过在病理数据上预训练获得。

| 方案 | 来源 | 预训练数据 | 特征维度 | 预期效果 | 实施难度 | 所需时间 |
|------|------|-----------|---------|---------|---------|---------|
| **UNI** | Mahmood Lab, Nature Medicine 2024 | 100K+ WSI, 100M+ tiles | 1024 | PCC +0.05~0.10 | 中 | 2-3 天 |
| **CONCH** | Mahmood Lab, Nature Medicine 2024 | 1.17M WSI-caption 对 | 768 | PCC +0.04~0.08 | 中 | 2-3 天 |
| **DINO-v2** | Meta AI, TMLR 2024 | 142M 自然图像 | 768/1024 | PCC +0.02~0.05 | 低 | 1-2 天 |
| **PLIP** | Stanford, Nature Comm 2024 | 13M 病理图像-文本对 | 768 | PCC +0.03~0.06 | 中 | 2-3 天 |

**推荐优先级**：**UNI > CONCH > PLIP > DINO-v2**

**实施要点**：
- 替换 `ResNetFeatureExtractor` 为对应模型的特征提取器
- UNI 输出 1024 维，需调整 `feature_proj` 层输入
- 大多数病理基础模型以 256x256 输入训练，可考虑调整输入分辨率

**预期效果**：
- 联合训练 PCC 从 0.42 提升至 0.47~0.52
- 免疫/炎症通路提升最大（ifng +0.10~0.20, Inflammatory_Response +0.15~0.25）

**实施难度**：中
**所需时间**：2-3 天

#### 方向 2：GNN 架构增强（GAT / 更多层 / 更大隐维度）

**原理解释**：当前 2 层 GraphSAGE 使用均匀聚合（所有邻居同等重要），无法区分不同邻居的重要性。增强 GNN 可以更好地建模 patch 间的复杂空间关系。

| 改进项 | 说明 | 预期效果 | 实施难度 | 所需时间 |
|--------|------|---------|---------|---------|
| **GAT（图注意力网络）** | 用注意力权重替代均匀聚合，自动学习邻居重要性 | PCC +0.01~0.03 | 低 | 0.5 天 |
| **增加层数（3-4 层）** | 扩大感受野，捕获更远距离的空间依赖 | PCC +0.005~0.02 | 低 | 0.5 天 |
| **更大隐维度（768/1024）** | 增加模型容量，更丰富的节点表示 | PCC +0.01~0.02 | 低 | 0.5 天 |
| **跳跃连接/DeepGCN** | 解决深层 GNN 的过平滑问题 | 允许使用更多层 | 中 | 1-2 天 |
| **边特征编码** | 在边中加入距离/方向信息 | PCC +0.005~0.015 | 中 | 1 天 |

**推荐优先级**：GAT 替换 GraphSAGE > 增加隐维度 > 增加层数 > 边特征

**注意**：单独增强 GNN 的收益有限（预估 +0.01~0.03 PCC），因为瓶颈主要在特征提取器。**GNN 增强应与特征提取器升级结合使用**。

**实施难度**：低~中
**所需时间**：0.5-2 天

#### 方向 3：混合图构建

**原理解释**：当前 EGNv2 仅使用空间半径图（基于坐标距离），忽略了 patch 间的功能相似性。混合图同时考虑空间邻接和功能相似性，可能捕获更丰富的关系。

**具体措施**：
- 同时构建空间半径图（坐标距离 < 300）和 KNN 特征图（特征空间中最近的 K 个邻居）
- 将两类边的并集作为最终图结构
- 可选：为不同类型的边赋予不同的初始权重

**预期效果**：
- PCC +0.01~0.03
- 特别有利于免疫通路（功能相似但空间距离远的 patch 可能属于同一免疫微环境）

**实施难度**：中
**所需时间**：1 天

#### 方向 4：超参数调优

**原理解释**：当前超参数为论文默认值，未针对 PFMval 数据做系统性搜索。关键超参数可能存在较大优化空间。

| 超参数 | 当前值 | 搜索范围 | 预期收益 | 实施难度 | 所需时间 |
|--------|--------|---------|---------|---------|---------|
| **learning_rate** | 1e-4 | [1e-5, 5e-4] | PCC +0.01~0.03 | 低 | 1-2 天 |
| **radius** | 300 | [150, 500] | PCC +0.01~0.03 | 低 | 1 天 |
| **k_neighbors** | 10 | [5, 20] | PCC +0.005~0.02 | 低 | 1 天 |
| **hidden_dim** | 512 | [256, 1024] | PCC +0.005~0.02 | 低 | 1 天 |
| **dropout** | 0.3 | [0.1, 0.5] | PCC +0.005~0.01 | 低 | 0.5 天 |
| **freeze_layers** | 3 | [0, 5] | PCC +0.01~0.03 | 低 | 0.5 天 |

**推荐搜索策略**：
1. **第一阶段**（1 天）：在 JFX0729 上用粗网格搜索 learning_rate x radius
2. **第二阶段**（1 天）：固定最优 lr 和 radius，搜索 k_neighbors x hidden_dim
3. **第三阶段**（0.5 天）：微调 dropout 和 freeze_layers

**预期总体收益**：PCC +0.02~0.05

**实施难度**：低
**所需时间**：2-3 天

#### 方向 5：数据增强

**具体措施**：
- 随机旋转/翻转（H&E 无方向性）
- 颜色扰动（模拟染色差异）
- Macenko 归一化

**预期效果**：PCC +0.01~0.02

**实施难度**：低
**所需时间**：0.5-1 天

### 5.3 EGN-v1 改进方向

EGN-v1 在所有实验中都表现最差，改进优先级较低。以下建议仅供学术参考。

#### 方向 1：加载预训练 ViT 权重

**原理解释**：EGN-v1 的自实现 ViT-Large 没有预训练权重，这是其性能差的最根本原因。如果能加载 ImageNet 或病理数据上的预训练 ViT 权重，特征质量将大幅提升。

**具体措施**：
- 安装 `timm` 库，使用 `timm.create_model('vit_large_patch32_224', pretrained=True)`
- 将预训练权重加载到 `ViTFeatureExtractor`
- 注意：标准 ViT-Large 是 patch16 而非 patch32，需调整或改用 patch16

**预期效果**：
- Val PCC 从 0.25 提升至 0.35~0.45
- 这是 EGN-v1 最有潜力的单一改进

**实施难度**：中
**所需时间**：1-2 天

#### 方向 2：正则化增强与训练策略调整

**原理解释**：EGN-v1 的负 Gap 表明训练不充分。需要降低正则化强度，让模型更充分地拟合训练数据。

**具体措施**：
- 降低 dropout：从 0.5 降至 0.3
- 提高学习率：从 1e-5 提高到 1e-4
- 增大 batch size：如果显存允许，从 16 提高到 32-64
- 延长训练：增加最大 epoch 到 200-300

**预期效果**：
- 消除负 Gap，训练 PCC 超过验证 PCC
- Val PCC 可能提升至 0.30~0.40

**实施难度**：低
**所需时间**：0.5-1 天

#### 方向 3：架构精简（如果保留 EGN-v1）

**原理解释**：如果必须使用 EGN-v1 架构，可考虑将其核心思想（Exemplar + GCN）迁移到更高效的实现中。

**具体措施**：
- 将 GCN 替换为 GraphSAGE（如 EGNv2 所做）
- 将自实现 ViT 替换为 timm 预训练 ViT
- 实际上，这就是"将 EGN-v1 升级为 EGNv2"

**预期效果**：本质上变成 EGNv2，预期 PCC ~0.40

**实施难度**：中
**所需时间**：2-3 天

## 第六部分：新数据补充后的研究规划

本项目计划后续补充两例新患者的病理数据，届时总数据集将从 3 例扩展到 5 例。以下是针对 5 患者数据的实验设计建议。

### 6.1 数据扩展后的预期规模

| 配置 | 当前（3 患者） | 扩展后（5 患者） | 增长比例 |
|------|--------------|----------------|---------|
| 总样本 | ~18,000 | ~30,000 | +67% |
| 训练样本 | 16,207 | ~27,000 | +67% |
| 验证样本 | 1,799 | ~3,000 | +67% |

### 6.2 5 患者联合训练方案

**实验设计**：

| 实验编号 | 训练数据 | 验证数据 | 目的 |
|---------|---------|---------|------|
| Exp-1 | 5 患者联合 | 5 患者联合（随机划分） | 验证大数据集下的性能上限 |
| Exp-2 | 4 患者 | 第 5 患者（留一法） | 评估跨患者泛化能力 |
| Exp-3 | 3 患者（原数据） | 2 新患者 | 验证模型是否能泛化到完全未见患者 |

**关键技术调整**：
1. **坐标归一化**：不同患者的坐标系统必须统一归一化到 [0, 1] 或相同物理尺度
2. **批次效应处理**：使用 Macenko 归一化或对抗域适应消除染色差异
3. **HisToGene 坐标嵌入调整**：当前 n_pos=128 可能不足以表示 5 患者的坐标范围，需增大

### 6.3 跨患者泛化能力评估方案

**核心问题**：模型在患者 A 上训练，能否在患者 B 上准确预测？

**评估方法**：

| 评估类型 | 训练集 | 测试集 | 指标 | 目标 |
|---------|--------|--------|------|------|
| **同患者验证** | 患者 A 的 90% | 患者 A 的 10% | PCC | 基准性能 |
| **跨患者测试** | 患者 A 全部 | 患者 B 全部 | PCC | 域泛化能力 |
| **混合验证** | 患者 A+B+C | 患者 D | PCC | 多患者训练的泛化能力 |

**关键指标**：
- **域泛化 Gap**：同患者 PCC - 跨患者 PCC
- 如果 Gap < 0.10，说明模型具有良好的跨患者泛化能力
- 如果 Gap > 0.15，说明存在严重域偏移，需要域适应技术

### 6.4 留一法交叉验证（LOOCV）方案

**留一法（Leave-One-Patient-Out Cross Validation）**：轮流将一位患者作为测试集，其余四位作为训练集，重复 5 次。

**实施步骤**：

```
第 1 轮：训练=[P2,P3,P4,P5], 测试=[P1]
第 2 轮：训练=[P1,P3,P4,P5], 测试=[P2]
第 3 轮：训练=[P1,P2,P4,P5], 测试=[P3]
第 4 轮：训练=[P1,P2,P3,P5], 测试=[P4]
第 5 轮：训练=[P1,P2,P3,P4], 测试=[P5]
```

**结果汇总**：
- 计算 5 轮测试 PCC 的均值和标准差
- 如果标准差 < 0.05，说明模型对所有患者都稳定
- 如果某一轮显著低于其他轮，说明该患者是"异常值"，需要检查数据质量

**预期结果解读**：
- 理想情况：5 轮 PCC 均值 ~0.55，标准差 < 0.03
- 现实预期：均值 ~0.50，标准差 0.05-0.08（域偏移影响）
- 警告信号：某一轮 PCC < 0.40，说明该患者的数据存在质量问题或域差异过大

### 6.5 新数据加入后的模型选择建议

| 模型 | 3 患者表现 | 5 患者预期 | 推荐优先级 |
|------|-----------|-----------|-----------|
| HisToGene | PCC 0.55 | PCC 0.58~0.62 | **最高**（大数据更能发挥大模型优势） |
| EGNv2 | PCC 0.42 | PCC 0.45~0.50 | **次高**（数据量增加对轻量模型也有帮助） |
| EGN-v1 | PCC 0.25 | PCC 0.28~0.32 | **低**（除非做重大架构改进） |

**关键决策点**：
- 如果 5 患者联合训练后 HisToGene Gap 仍 > 0.15，必须实施数据增强和更强的正则化
- 如果 EGNv2 + UNI 升级后能达到 PCC > 0.50，值得与 HisToGene 做模型集成
- 如果跨患者泛化 Gap > 0.15，需要引入域适应（Domain Adaptation）技术

## 第七部分：推荐测试的其他模型

除了当前三个模型，以下 5 个模型也值得在后续实验中尝试。每个模型都包含详细的适配性分析和推荐理由。

### 7.1 ST-Net（Spatial Transcriptomics Network）

**论文出处**：He et al., "Spatial transcriptomics analysis of gene expression prediction using deep learning", *Bioinformatics* 2020

**核心架构**：
- 特征提取器：VGG-16 或 ResNet-50（类似 EGNv2）
- 空间建模：2D 卷积层直接在全切片上操作，而非 patch 级别
- 创新点：将组织切片视为整体图像，用全卷积网络（FCN）直接预测空间基因表达图

**通俗解释**：
ST-Net 不像我们的模型那样"一块一块"地预测，而是把整张切片输入，输出一张"基因表达热力图"。就像卫星图像分割——输入一张地图，输出每个区域的植被覆盖度。

**在空间转录组领域的表现**：
- 在 10x Visium 数据上 PCC ~0.45-0.55
- 优势在于能直接建模全切片尺度的空间模式

**与当前数据的适配性**：
- **适配度**：中
- **调整需求**：需要将 patch 级数据重新拼接为全切片图像；输出需适配 30 通路
- **数据量要求**：较高（需要完整切片，而非独立 patch）

**预期性能和部署难度**：
- 预期 PCC：0.50~0.55（联合训练）
- 部署难度：中（需要重新组织数据格式）
- 计算资源：高（全切片输入显存占用大）

**推荐理由**：
ST-Net 的全卷积范式与当前 patch 级方法互补。如果 patch 级模型遇到瓶颈，全切片方法可能通过全局上下文理解获得更好的空间连续性预测。

---

### 7.2 BLEEP（Bi-modal Language-Encoded Expression Prediction）

**论文出处**：Ding et al., "BLEEP: A benchmark for language-encoded expression prediction", *Nature Methods* 2024

**核心架构**：
- 特征提取器：病理图像编码器（如 UNI）+ 文本编码器
- 创新点：将基因通路名称作为**文本提示**，用视觉-语言对比学习对齐图像特征和基因表达
- 训练方式：对比学习（Contrastive Learning），而非直接回归

**通俗解释**：
BLEEP 就像一个"看图说话"游戏。模型看到病理图像后，不是直接输出数字，而是回答"这张图像对应的炎症反应评分是高还是低？"通过将基因通路名称（如"Inflammatory_Response"）编码为文本向量，模型学会将图像与通路的文本描述关联起来。

**在空间转录组领域的表现**：
- 在跨数据集基准测试上 PCC ~0.50-0.58
- 在零样本（zero-shot）泛化上表现优异

**与当前数据的适配性**：
- **适配度**：高
- **调整需求**：需要将 30 条通路名称作为文本输入；需修改输出头为回归形式
- **数据量要求**：中等

**预期性能和部署难度**：
- 预期 PCC：0.52~0.60（联合训练）
- 部署难度：中（需要视觉-语言预训练模型）
- 计算资源：中

**推荐理由**：
BLEEP 的视觉-语言对齐机制可能特别适合**免疫/炎症通路**的预测——这些通路的名称本身就包含丰富的语义信息（如"Interferon_Alpha"直接暗示抗病毒反应）。当前 HisToGene 在 ifng 和 Inflammatory_Response 上已有不错表现，BLEEP 可能进一步提升这些语义明确的通路。

---

### 7.3 SpaGCN（Spatial Graph Convolutional Network）

**论文出处**：Hu et al., "SpaGCN: Integrating gene expression, spatial location and histology to identify spatial domains and spatially variable genes", *Nature Methods* 2021

**核心架构**：
- 图神经网络：GCN 的变体，专门设计用于空间转录组
- 图构建：同时考虑基因表达相似性和空间距离
- 创新点：**自适应图卷积**，根据空间距离动态调整邻居权重

**通俗解释**：
SpaGCN 是专门为空间转录组设计的图神经网络。与普通 GNN 不同，它不仅知道"谁是邻居"，还知道"距离多远"——近邻的影响大，远邻的影响小。这非常符合病理组织的生物学直觉：相邻细胞的相互作用更强。

**在空间转录组领域的表现**：
- 在空间域识别（Spatial Domain Identification）任务上 SOTA
- 在基因表达预测上 PCC ~0.48-0.55

**与当前数据的适配性**：
- **适配度**：高
- **调整需求**：需要将 patch 特征和空间坐标构建为 SpaGCN 的图格式；适配 30 通路输出
- **数据量要求**：中等

**预期性能和部署难度**：
- 预期 PCC：0.48~0.55（联合训练）
- 部署难度：中（需要理解 SpaGCN 的图构建逻辑）
- 计算资源：中

**推荐理由**：
SpaGCN 是空间转录组领域的专用工具，其自适应图卷积机制可能比 EGNv2 的固定半径图更精确地建模病理组织的空间关系。如果 EGNv2 的图卷积改进效果不佳，SpaGCN 是一个自然的升级方向。

---

### 7.4 HisToGene + UNI（病理基础模型增强版）

**论文出处**：基于 HisToGene 框架 + UNI 特征提取器（Mahmood Lab, *Nature Medicine* 2024）

**核心架构**：
- 保留 HisToGene 的坐标嵌入 + Transformer + MLP 回归头设计
- 将 Patch Embedding 替换为 UNI 预训练特征提取器
- UNI 输出 1024 维特征，与当前 HisToGene 的 dim=1024 完美兼容

**通俗解释**：
这相当于给 HisToGene 换了一个"更聪明的眼睛"。当前 HisToGene 的 ViT 是从头训练的，相当于一个刚学看病理切片的实习生；UNI 是在 10 万张病理切片上训练的专家，能识别细胞类型、组织结构和病变模式。

**在空间转录组领域的表现**：
- UNI 在多项病理图像任务上达到 SOTA
- 作为特征提取器，预期可将下游任务性能提升 10-20%

**与当前数据的适配性**：
- **适配度**：极高
- **调整需求**：最小——只需替换特征提取器，保留 HisToGene 的其他组件
- **数据量要求**：不变

**预期性能和部署难度**：
- 预期 PCC：0.60~0.68（联合训练）
- 部署难度：中（需下载 UNI 权重，约 300MB）
- 计算资源：中（UNI 推理速度比 ResNet-50 慢 2-3 倍）

**推荐理由**：
这是**风险最低、收益最确定**的改进方向。HisToGene 已证明是最佳架构，UNI 已证明是最佳病理特征提取器，两者结合的预期效果非常明确。如果只能做一个实验，这应该是首选。

---

### 7.5 STimage（Spatial Transcriptomics Image-based prediction）

**论文出处**：Park et al., "STimage: Deep learning-based spatial transcriptomics image analysis", *Genome Biology* 2022

**核心架构**：
- 特征提取器：ResNet-50 + 自监督预训练（DINO）
- 空间建模：Transformer 编码器（类似 HisToGene，但使用预训练特征）
- 创新点：结合对比学习和 masked image modeling 的自监督预训练

**通俗解释**：
STimage 先通过"看图猜缺失部分"的游戏（masked image modeling）学会理解病理图像，然后用这些学到的知识来做基因表达预测。这种自监督预训练不需要基因表达标签，可以充分利用大量无标签病理图像。

**在空间转录组领域的表现**：
- 在 HER2ST 数据集上 PCC ~0.50-0.56
- 自监督预训练使其在小数据集上表现更稳定

**与当前数据的适配性**：
- **适配度**：高
- **调整需求**：需要将病理图像组织为 STimage 的输入格式；适配 30 通路输出
- **数据量要求**：低（自监督预训练不需要标签）

**预期性能和部署难度**：
- 预期 PCC：0.52~0.58（联合训练）
- 部署难度：中
- 计算资源：中

**推荐理由**：
STimage 的自监督预训练思路特别适合当前场景——我们有大量病理图像（约 18K patches），但基因表达标签可能相对昂贵。如果未来需要扩展到更多患者，STimage 的预训练策略可以充分利用无标签数据。

### 7.6 模型推荐优先级汇总

| 优先级 | 模型 | 预期联合 PCC | 实施难度 | 与当前工作关联 | 推荐理由 |
|--------|------|------------|---------|--------------|---------|
| **1** | **HisToGene + UNI** | 0.60~0.68 | 中 | 直接升级 HisToGene | 风险最低，收益最确定，最佳架构+最佳特征 |
| **2** | **BLEEP** | 0.52~0.60 | 中 | 引入视觉-语言新范式 | 对免疫/炎症通路可能有独特优势 |
| **3** | **SpaGCN** | 0.48~0.55 | 中 | 替换 EGNv2 的 GNN 组件 | 空间转录组专用 GNN，图建模更精确 |
| **4** | **ST-Net** | 0.50~0.55 | 中 | 全新范式（全切片级） | 全卷积可能更好建模全局空间模式 |
| **5** | **STimage** | 0.52~0.58 | 中 | 引入自监督预训练 | 适合未来扩展更多无标签数据 |

## 第八部分：总体实验路线图

## 附录 A：名词解释表

| 术语 | 英文全称 | 解释 |
|------|---------|------|
| **PCC** | Pearson Correlation Coefficient | 皮尔逊相关系数，衡量预测值与真实值的线性相关程度 |
| **R2** | R-squared / Coefficient of Determination | 决定系数，衡量模型解释方差的比例 |
| **MSE** | Mean Squared Error | 均方误差，对大误差敏感 |
| **MAE** | Mean Absolute Error | 平均绝对误差，反映典型误差水平 |
| **Gap** | Overfitting Gap | 训练 PCC 与验证 PCC 的差值，衡量过拟合程度 |
| **ViT** | Vision Transformer | 视觉 Transformer，将图像分块后用自注意力建模 |
| **ResNet** | Residual Network | 残差网络，通过跳跃连接解决深层网络梯度消失问题 |
| **GCN** | Graph Convolutional Network | 图卷积网络，基于谱图理论的图神经网络 |
| **GraphSAGE** | Graph SAmple and aggreGatE | 归纳式图神经网络，支持小批量训练和 unseen 节点 |
| **GAT** | Graph Attention Network | 图注意力网络，用注意力机制学习邻居重要性 |
| **Exemplar** | Exemplar / Prototype | 代表样本，从训练集中选取的典型样本 |
| **KNN** | K-Nearest Neighbors | K 近邻算法，找到距离最近的 K 个邻居 |
| **ssGSEA** | Single-sample Gene Set Enrichment Analysis | 单样本基因集富集分析，计算通路活性评分 |
| **Z-score** | Standard Score | 标准化评分，均值为 0，标准差为 1 |
| **UNI** | Universal Pathology Embedding | 通用病理嵌入，大规模病理基础模型 |
| **CONCH** | CONtrastive Captioning with Histology | 病理视觉-语言预训练模型 |
| **HuberLoss** | Huber Loss | 对异常值鲁棒的损失函数，结合 MSE 和 MAE 优点 |
| **LOOCV** | Leave-One-Out Cross Validation | 留一法交叉验证 |
| **Domain Shift** | Domain Shift | 域偏移，不同数据集间的分布差异 |
