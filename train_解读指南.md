# train.py 深度解读指南

> 本文档面向初学者，详细解读基于 UNI2-h 特征的病理图像回归模型训练流程。

---

## 1. 文件概述

**一句话说明**：`train.py` 是一个用于训练**病理图像多目标回归模型**的脚本，基于预训练的 UNI2-h  backbone 提取图像特征，训练一个轻量级 MLP 回归头预测基因集分数。

**在项目中的角色**：这是模型训练阶段的核心脚本，位于数据预处理（split.py 划分数据集）之后、模型推理（infer.py 预测新数据）之前，负责特征提取、模型训练和最优模型保存。

---

## 2. 背景知识

### 2.1 迁移学习与冻结 backbone

**迁移学习（Transfer Learning）**：利用在大规模数据集（如 ImageNet、病理图像库）上预训练的模型，来解决特定领域的小样本问题。

**冻结 backbone**：
- UNI2-h 已在海量病理图像上预训练，具有强大的特征提取能力
- 冻结 backbone（不更新其参数）可以：
  - 大幅减少训练时间和显存占用
  - 防止在小数据集上过拟合
  - 利用预训练模型的通用病理特征

```
输入图像 → [冻结的 UNI2-h backbone] → 1536维特征向量 → [可训练的 MLP 回归头] → 预测值
              ↑_________________________↑
                        冻结（不训练）
                                   ↑_________________↑
                                        训练（更新参数）
```

### 2.2 损失函数：MSELoss

**MSE（Mean Squared Error，均方误差）**：

$$
MSE = \frac{1}{n} \sum_{i=1}^{n} (y_{true} - y_{pred})^2
$$

- 计算预测值与真实值的平方差均值
- 对大误差惩罚更重（平方放大）
- 适合回归任务，目标是最小化预测误差

### 2.3 优化器：AdamW

**AdamW** 是 Adam 优化器的改进版本：

| 特性 | 说明 |
|------|------|
| 自适应学习率 | 为每个参数自动调整学习率 |
| 权重衰减（Weight Decay） | 正确实现 L2 正则化，防止过拟合 |
| 动量估计 | 利用一阶和二阶矩估计加速收敛 |

**参数说明**：
- `lr=1e-3`：初始学习率
- `weight_decay=1e-4`：权重衰减系数，控制正则化强度

### 2.4 学习率调度：ReduceLROnPlateau

**功能**：当验证集 loss 不再下降时，自动降低学习率。

**参数说明**：
- `mode="min"`：监控指标越小越好（loss）
- `factor=0.5`：学习率乘以 0.5（减半）
- `patience=5`：连续 5 个 epoch 无改善才降低学习率

**作用**：帮助模型跳出局部最优，精细调整参数。

### 2.5 早停（Early Stopping）

**功能**：当验证集性能连续多轮无改善时，提前终止训练。

**目的**：
- 防止过拟合（模型在训练集上表现越来越好，验证集上变差）
- 节省计算资源
- 自动保存最优模型

**触发条件**：
- `patience=10`：连续 10 个 epoch 验证 loss 无改善
- `min_delta=0.0`：新 loss 必须比当前最优值低至少 0.0 才算改善

### 4.5 训练循环（含早停和 LR 调度逻辑）

**代码位置**：行 132-191

**功能说明**：
核心训练循环，每个 epoch 执行训练、验证、学习率调整、早停判断和模型保存。

**执行流程详解**：

```
初始化:
  - best_val_loss = ∞
  - patience_counter = 0
  - history = []

For each epoch:
  1. 训练阶段
     └─ train_one_epoch() → 返回 train_metrics
  
  2. 验证阶段
     └─ evaluate() → 返回 val_metrics
  
  3. 学习率调度
     └─ scheduler.step(val_metrics["loss"])
  
  4. 记录历史
     └─ history.append({...})
  
  5. 检查是否最优
     ├─ IF val_loss < best_val_loss - min_delta:
     │    - 更新 best_val_loss
     │    - 保存模型状态 (best_state)
     │    - patience_counter = 0
     │    - 打印 "New best val loss"
     └─ ELSE:
          - patience_counter += 1
          - 打印 "No improvement"
          - IF patience_counter >= early_stop_patience:
               - 触发早停，break
```

**关键代码行**：

| 行号 | 代码 | 作用 |
|------|------|------|
| 137 | `for epoch in range(args.num_epochs)` | 训练循环 |
| 139 | `train_metrics = train_one_epoch(...)` | 训练一个 epoch |
| 140 | `val_metrics = evaluate(...)` | 验证评估 |
| 142 | `scheduler.step(val_metrics["loss"])` | 更新学习率 |
| 154-165 | `history.append({...})` | 记录训练历史 |
| 168 | `if val_metrics["loss"] < best_val_loss - args.min_delta` | 判断是否改善 |
| 171-182 | `best_state = {...}` | 保存最优模型状态 |
| 183 | `patience_counter = 0` | 重置耐心计数器 |
| 186-190 | 早停判断逻辑 | 无改善则累加计数器 |

## 5. 训练超参数说明

| 参数名 | 默认值 | 含义 | 调优建议 |
|--------|--------|------|----------|
| `batch_size` | 256 | 每批处理的样本数 | 根据显存调整，常见 64/128/256/512 |
| `num_epochs` | 100 | 最大训练轮数 | 配合早停使用，可设较大值（如 200） |
| `learning_rate` | 1e-3 | 初始学习率 | 过大（>1e-2）可能不稳定，过小（<1e-4）收敛慢 |
| `hidden_dim` | 256 | MLP 隐藏层维度 | 数据量大可增大（512/1024），小则减小（128） |
| `dropout` | 0.2 | Dropout 丢弃概率 | 过拟合时增大（0.3-0.5），欠拟合时减小（0.1） |
| `weight_decay` | 1e-4 | L2 正则化系数 | 过拟合时增大（1e-3），欠拟合时减小（1e-5） |
| `early_stop_patience` | 10 | 早停耐心值 | 数据量大时可增大（15-20），小则减小（5-10） |
| `min_delta` | 0.0 | 改善阈值 | 设为 0.001 可忽略微小波动 |
| `num_workers` | 0 | 数据加载子进程数 | Windows 建议 0，Linux 可设为 4-8 |

### 超参数调优指南

**学习率选择**：
- **太大**（如 1e-2）：loss 震荡不下降，可能发散
- **合适**（如 1e-3）：loss 稳定下降
- **太小**（如 1e-5）：收敛极慢，可能陷入局部最优

**Batch Size 调整**：
- **较大**（如 512）：训练更快，梯度更稳定，但需更大显存
- **较小**（如 64）：内存友好，但梯度噪声大，训练 slower

## 6. 训练监控指标

### 6.1 指标含义

| 指标 | 全称 | 含义 | 理想值 |
|------|------|------|--------|
| **loss** | MSE Loss | 均方误差，预测值与真实值的差距 | 越小越好，接近 0 |
| **mae** | Mean Absolute Error | 平均绝对误差，预测偏差的平均大小 | 越小越好 |
| **r2** | R² Score | 决定系数，模型解释数据变异的能力 | 越接近 1 越好 |
| **pcc** | Pearson Correlation Coefficient | 皮尔逊相关系数，预测与真实值的线性相关性 | 越接近 1 越好 |

### 6.2 如何判断训练状态

**正常训练**：
```
Train | loss=0.5234 mae=0.5234 r2=0.2345 pcc=0.5234
Val   | loss=0.5345 mae=0.5345 r2=0.2234 pcc=0.5123
```
- 训练 loss 和验证 loss 同步下降
- r2 和 pcc 逐渐上升
- 训练和验证指标差距不大

**过拟合迹象**：
```
Train | loss=0.1234 mae=0.1234 r2=0.8234 pcc=0.9234
Val   | loss=0.8234 mae=0.8234 r2=0.1234 pcc=0.3234
```
- 训练 loss 持续下降，验证 loss 上升或停滞
- 训练 r2/pcc 很高，验证 r2/pcc 很低
- **对策**：增大 dropout、增大 weight_decay、早停

**欠拟合迹象**：
```
Train | loss=0.8234 mae=0.8234 r2=0.1234 pcc=0.3234
Val   | loss=0.8345 mae=0.8345 r2=0.1134 pcc=0.3123
```
- 训练和验证 loss 都很高
- r2 和 pcc 都很低
- **对策**：增大 hidden_dim、减小 dropout、增加训练轮数

```
│  Input:  [256, 1536]                                        │
│  Layer1: [256, 1536] → [256, 256] (Linear + GELU + Dropout) │
│  Output: [256, 256] → [256, 8]   (Linear)                   │
└─────────────────────────────────────────────────────────────┘
    │
    ▼ 损失计算与优化
┌─────────────────────────────────────────────────────────────┐
│  MSELoss: 计算 predictions vs targets 的均方误差             │
│  AdamW: 更新 MLP 参数（backbone 参数冻结不更新）              │
└─────────────────────────────────────────────────────────────┘
```

## 8. 初学者注意事项

### 8.1 常见错误

| 错误 | 原因 | 解决方法 |
|------|------|----------|
| `CUDA out of memory` | batch_size 过大 | 减小 batch_size（如 128 → 64） |
| `FileNotFoundError` | 路径错误或文件缺失 | 检查 `train_patches_dir` 等路径 |
| `Missing feature cache` | 未提取特征或缓存被删除 | 运行训练或添加 `--rebuild_cache` |
| 训练 loss 不下降 | 学习率过小或数据问题 | 增大 lr，检查数据归一化 |
| 验证 loss 远高于训练 loss | 过拟合 | 增大 dropout、weight_decay |

## 9. 扩展思考

### 9.1 可能的改进方向

**1. 模型结构改进**
- 当前：单层隐藏层的 MLP
- 扩展：增加隐藏层数、使用残差连接、尝试 Transformer 结构

**2. 损失函数改进**
- 当前：MSELoss（对所有目标一视同仁）
- 扩展：加权 MSE（不同目标赋予不同权重）、Huber Loss（对异常值更鲁棒）

**3. 数据增强**
- 当前：无数据增强（特征已预提取）
- 扩展：特征层面的增强（如添加噪声、Mixup）

**4. 多任务学习策略**
- 当前：所有目标共享同一个 MLP
- 扩展：为每个目标单独训练一个回归头、使用任务特定的注意力机制

**5. 超参数自动搜索**
- 当前：手动设置超参数
- 扩展：使用 Optuna、Ray Tune 进行自动化超参数搜索

### 9.2 与其他组件的集成

```
WSI 全切片图像
    │
    ▼ 切分工具
patch 图像集合
    │
    ▼ split.py（空间无重叠划分）
训练集 / 验证集
    │
    ▼ train.py（本文档）
提取 UNI2-h 特征 + 训练回归模型
    │
    ▼ 保存 best_model.pth
训练好的回归模型
    │
    ▼ infer.py（推理预测）
新数据的基因集分数预测
```

## 10. 总结

`train.py` 是一个简洁高效的病理图像回归模型训练脚本，其核心贡献是：

1. **迁移学习范式**：冻结强大的 UNI2-h backbone，仅训练轻量级回归头
2. **特征缓存机制**：避免重复特征提取，大幅加速训练
3. **完整的训练流程**：包含学习率调度、早停、模型保存等最佳实践
4. **多目标回归**：同时预测多个基因集分数

对于初学者，理解这个脚本有助于掌握：
- PyTorch 模型训练的基本流程（Dataset → DataLoader → Model → Train Loop）
- 迁移学习和冻结 backbone 的实践方法
- 学习率调度和早停等训练技巧
- 回归任务的评估指标（MSE、MAE、R²、PCC）
