## 2. Z-score 标准化的前提条件（教学篇）

### 2.1 什么是 Z-score？

**Z-score（标准分数）** 是一种将数据转换为统一尺度的方法，公式如下：

```
Z = (X - μ) / σ
```

其中：
- `X`：原始数据值
- `μ`：数据的均值（mean）
- `σ`：数据的标准差（standard deviation）

**直观理解**：
- Z-score 表示一个数据点距离均值有多少个标准差
- Z = 0 表示该值等于均值
- Z = 1 表示该值比均值高 1 个标准差
- Z = -2 表示该值比均值低 2 个标准差

### 2.2 Z-score 的理论假设

Z-score 方法**理论上假设数据应接近正态分布**（钟形曲线）。这是因为：

1. 在正态分布中，约 68% 的数据落在 Z ∈ [-1, 1] 范围内
2. 约 95% 的数据落在 Z ∈ [-2, 2] 范围内
3. 这种比例关系在非正态分布中可能不成立

### 2.3 判断数据是否适合 Z-score 的标准

| 指标 | 理想值 | 可接受范围 | 警示阈值 |
|------|--------|------------|----------|
| **偏度 (Skewness)** | 0 | \|偏度\| < 0.5 | \|偏度\| > 1 |
| **峰度 (Kurtosis)** | 0 | \|峰度\| < 3 | \|峰度\| > 3 |
| **正态性检验 p 值** | > 0.05 | > 0.01 | < 0.001 |
| **异常值比例** | < 1% | < 5% | > 10% |
| **均值-中位数差异** | ≈ 0 | 相对较小 | 差异显著 |

**详细解释**：

- **偏度**：衡量分布的不对称性
  - 偏度 = 0：完全对称（正态分布）
  - 偏度 > 0：右偏（长尾在右侧）
  - 偏度 < 0：左偏（长尾在左侧）
  - \|偏度\| < 0.5：近似对称 ✅
  - 0.5 ≤ \|偏度\| < 1：中等偏态 ⚠️
  - \|偏度\| ≥ 1：显著偏态 ❌

- **峰度**：衡量分布尾部的厚度（相对于正态分布）
  - 峰度 = 0：与正态分布相同
  - 峰度 > 0：尖峰厚尾（极端值更多）
  - 峰度 < 0：平峰薄尾（极端值更少）
  - \|峰度\| > 3：重尾分布，异常值风险高

## 5. 综合结论

### 5.1 数据整体是否符合 Z-score 条件？

**明确回答：从严格的统计意义上，本项目数据不完全符合 Z-score 的理想条件。**

具体而言：
- 8 个基因集中，**只有 1 个（mhc）** 接近正态分布
- **7 个基因集** 存在不同程度的偏态问题
- **ifng 列** 的异常值比例高达 19%，远超可接受范围

### 5.2 为什么 Z-score 在本项目中仍然有效？

尽管数据不完全符合正态假设，Z-score 在本项目中**仍然是有价值的**，原因如下：

#### 1. 量纲统一的必要性（主要目的）

在本项目中，8 个基因集的评分具有**完全不同的量纲和数值范围**：

| 基因集 | 数值范围 | 标准差 |
|--------|----------|--------|
| tls | -2,792 ~ 5,027 | 639 |
| mhc | 1,147 ~ 9,066 | 2,868 |
| icp | -4,664 ~ 1,710 | 543 |

如果不进行标准化，数值范围大的基因集（如 mhc）会在模型中占据主导地位，而数值范围小的基因集（如 icp）会被"淹没"。

**Z-score 的核心价值在于消除量纲差异**，使不同基因集具有可比性。

#### 2. 深度学习的鲁棒性

在深度学习场景中：
- 神经网络对输入分布的假设相对宽松
- Batch Normalization 等层会进一步处理数据分布
- Z-score 的主要作用是**加速收敛**和**稳定训练**

#### 3. 实际应用中的惯例

在生物信息学和基因表达分析领域：
- Z-score 标准化是**标准做法**
- 即使数据不完全正态，Z-score 仍被广泛使用
- 其效果已在大量研究中得到验证

### 5.3 Z-score 的局限性

使用 Z-score 时需要注意以下局限性：

| 局限性 | 说明 | 本项目影响 |
|--------|------|------------|
| **不改变分布形状** | Z-score 只是线性变换，不会将偏态数据变为正态 | 偏态的列标准化后仍然偏态 |
| **不处理异常值** | 极端值会被进一步放大（Z 值可能很大） | ifng 的 19% 异常值仍然存在 |
| **受极端值影响** | 均值和标准差对异常值敏感 | 异常值会"拉偏"标准化结果 |
| **假设对称性** | 对于严重偏态数据，Z 值的解释性下降 | 大部分列的 Z 值需要谨慎解读 |

## 6. 初学者知识延伸

### 6.1 Z-score 不要求严格正态也能用的场景

在以下场景中，即使数据不完全正态，Z-score 仍然有效：

#### ✅ 场景 1：量纲统一是主要目的
- 当你需要比较不同量纲的变量时
- 例如：同时分析身高（cm）和体重（kg）
- **本项目属于此类**

#### ✅ 场景 2：深度学习预处理
- 神经网络对输入分布相对不敏感
- Z-score 主要帮助梯度下降收敛
- BatchNorm 层会进一步归一化

#### ✅ 场景 3：后续使用非参数方法
- 如果后续分析使用非参数检验（如 Mann-Whitney U 检验）
- Z-score 只是作为数据缩放手段

#### ✅ 场景 4：数据近似对称（\|偏度\| < 2）
- 轻度偏态对 Z-score 影响不大
- 大部分数据的 Z 值仍有解释意义

### 6.2 什么时候必须考虑替代方案？

在以下情况，建议考虑 Z-score 的替代方法：

| 情况 | 阈值 | 建议方案 |
|------|------|----------|
| **异常值过多** | > 10% | Robust Scaler（使用中位数和 IQR） |
| **严重偏态** | \|偏度\| > 2 | 先进行 Log 变换，再 Z-score |
| **存在零或负值** | 数据中有 0 | 使用 Yeo-Johnson 变换 |
| **需要严格正态** | 后续需要正态假设 | Box-Cox 变换 |
| **极端值重要** | 异常值包含有用信息 | 分位数归一化（Quantile Transformer） |

### 6.3 可能的改进方向

针对本项目数据的特点，可以考虑以下改进：

#### 方案 1：Log 变换 + Z-score
```python
# 对正数数据进行 Log 变换后再标准化
import numpy as np
from sklearn.preprocessing import StandardScaler

# 假设 data 是原始数据
log_data = np.log1p(data - data.min() + 1)  # 处理负值
scaler = StandardScaler()
zscore_data = scaler.fit_transform(log_data)
```

**适用性**：
- 对右偏数据效果较好
- 可以压缩长尾，减少异常值影响
- 需要数据为正值（或进行平移）

#### 方案 2：Robust Scaler
```python
from sklearn.preprocessing import RobustScaler

scaler = RobustScaler()  # 使用中位数和 IQR
robust_data = scaler.fit_transform(data)
```

**适用性**：
- 对异常值不敏感
- 适合 ifng 这种异常值比例高的列
- 不假设正态分布

#### 方案 3：分位数归一化
```python
from sklearn.preprocessing import QuantileTransformer

transformer = QuantileTransformer(output_distribution='normal')
normal_data = transformer.fit_transform(data)
```

**适用性**：
- 强制将数据转换为正态分布
- 保留数据的相对顺序
- 计算成本较高

### 6.4 本项目的建议

综合考虑，对于本项目：

1. **当前阶段**：可以继续使用 Z-score，但需了解其局限性
2. **后续优化**：
   - 对 **ifng** 列考虑 Robust Scaler 或先进行 Log 变换
   - 对 **tls**、**icp**、**toxic** 等高偏度列考虑数据变换
   - 对比不同标准化方法对模型性能的影响
3. **监控指标**：在模型训练过程中监控这些列的梯度分布，确保训练稳定
