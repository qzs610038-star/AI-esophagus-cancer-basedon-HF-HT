审查日期：2026-05-30

## 0. 总体结论

PFMval 当前已经从“验证 H&E 能否预测空间转录组通路”推进到“围绕跨患者泛化和 Phase 3 预后转化做工程化迭代”的阶段。项目主线基本正确：先用 3 例 ST 患者建立 H&E -> 30 通路 ssGSEA 的虚拟通路模型，再把该模型迁移到约 80 例预后队列中，构建 H&E -> 虚拟通路 -> 预后 AUC 的临床转化链条。

最重要的阶段性成果是：项目已经证明 UNI2-h 病理基础模型特征是跨患者泛化的核心支撑，TV Loss 是目前最稳健、最低成本的有效改进，OmiCLIP、全局 AttnPool、GAT 深化、通路加权、单纯加深下游编码器等方向已经暴露出较明确的边界。

当前最大的决策风险不是“模型不够多”，而是“计划方向太多，Phase 3 闭环和服务器自动化仍未真正打通”。如果继续同时推进 RL、LoRA、CONCH、染色归一化、多教师、多层特征、对比预训练、通路语义等方向，容易让项目从清晰主线漂移成探索堆栈。

建议后续 2 周聚焦三件事：

1. 打通 Phase 3 预后最小闭环，哪怕先用现有最优模型离线预测 320 张 HE，完成患者级聚合 + 5-fold AUC。
2. 解决服务器远程启动/结果回收，让 RTX 4080 成为稳定训练节点，而不是继续依赖手动 RDP。
3. 把下一轮模型改进限定为 1 个主攻方向 + 1 个备选方向，不再开放式扩散。

## 1. 我对本项目目标的理解

### 1.1 核心科学问题

项目假设是：食管癌 H&E 病理图像中包含与空间转录组通路活性相关的形态学信息，可以用深度学习从常规 H&E patch 预测 30 条 ssGSEA 通路 Z-score。

### 1.2 当前任务分层

| 层级 | 目标 | 输入 | 输出 | 核心指标 | 状态 |
|---|---|---|---|---|---|
| Phase 2 | 虚拟通路预测 | 224x224 H&E patch | 30 条 ssGSEA Z-score | PCC、MAE、R2 | 已形成多模型基线和最优方案 |
| Phase 3 | 预后预测 | 患者多张 HE 切片 | 好/差预后二分类 | AUC | 方案明确，工程闭环未见完整落地 |
| 中长期 | 9 ST 患者扩展与论文级验证 | 更多 ST 患者 + HE | 更稳健虚拟通路模型 | LOPO mean±std | 等待数据与算力管线 |

### 1.3 临床转化逻辑

Phase 2 本身不是终点。真正可转化路径是：

```text
H&E patch
-> Phase 2 模型预测虚拟通路分数
-> 切片级 / 患者级聚合
-> 预后分类器
-> AUC 与通路解释
```

因此，Phase 2 的模型改进应服从 Phase 3：不应只追求单 fold PCC 小数点提升，而要关心这些虚拟通路分数是否能提升预后 AUC、是否可解释、是否跨患者稳定。

证据：`02_组会汇报/组会汇报_0418_HisToGene训练.md`

| 决策 | 审查 |
|---|---|

### 4.2 4月下旬：跨患者泛化与 UNI 特征确立

证据：`02_组会汇报/组会汇报_0424_模型训练结果.md`

| 决策 | 审查 |

### 4.5 5月中旬：AugMix/MixUp 与 P1 失败

证据：`01_指南与解读/分析报告/AugMix实验评估报告_20260515.md`、`02_组会汇报/组会汇报_0516_P1实验与下一步.md`

| 方向 | 结果 | 审查 |
|---|---|---|
| H&E 颜色增强 | 无效 | 结论可信。UNI 已有颜色不变性，静态 HED 增强信息增益低。 |
| MixUp | 有效，alpha=0.2 最优 | 可以保留，是低成本泛化正则。 |
| H&E + MixUp 联合 | 不如 MixUp 单独 | 合理，颜色扰动噪声可能稀释 MixUp。 |
| 通路加权损失 | 失败 | 当前 3 例下应暂停。9 例后可基于稳定 per-pathway mean±std 重试。 |
| 编码器加深 | 失败 | 当前 3 例下应暂停。9 例后可小规模重试。 |

关键审查：

- AugMix 报告主动指出超参数混杂，这是非常好的科研工程习惯。
- 但“下游天花板约 0.42，必须动上游”在后续 TV Loss + 融合后被部分修正。更准确说法应是：在冻结 UNI + 常规下游架构微调下，Fold1 单模型短期上限约 0.42；空间先验与融合仍可提供小幅增益。

### 4.6 5月下旬：OmiCLIP、Virchow2、TV Loss、融合

证据：`模型性能排名_Model_Performance_Ranking.md`、`02_组会汇报/组会汇报_0523_W21周报.md`、`experiments/results`

| 方向 | 实施状态 | 结果 | 审查 |
|---|---|---|---|
| OmiCLIP | 已实施 | 单患者约 0.554，跨患者三折约 0.197 | 明确负结果。不得作为跨患者主线。 |
| Virchow2 | 已实施 | Fold1 约 0.3537，弱于 UNI | 独立使用不强，作为互补特征边际有效。 |
| TV Loss | 已实施 | L2 w=0.01 三折均值 0.3943 | 当前最稳健有效改进。应保留为默认配置。 |
| UNI + Virchow2 晚期融合 | 已实施 | Fold1 grid search 0.4242 | 增益很小，适合作为结果锦上添花，不宜夸大。 |
| 全局 AttnPool | 已实施 | 跨患者下降 | 应拒绝，不再做默认方向。 |

关键纠偏：

- `fusion_report.json` 显示 TV 模型 29/30 通路优于 Virchow2，Virchow2 只在极少通路上互补。融合从 0.4212 到 0.4242，增益约 +0.003，可能小于运行波动。报告可以保留最优数值，但论文/组会中应强调“主要贡献是 TV Loss，融合为边际增益”。
- AttnPool 失败根因分析很扎实：不是患者特异性，而是注意力近乎均匀且无信息增益。后续如果尝试 per-pathway attention，必须明确它不是复活旧 AttnPool，而是一个参数量极低的新假设。

## 6. 未实施或不完善部分

### 6.1 Phase 3 闭环缺失

这是当前最重要缺口。

最低可行闭环应包括：

| 步骤 | 产物 |
|---|---|
| 收集 80 患者 HE 切片清单 | `phase3_manifest.csv` |
| 记录患者级预后标签 | `phase3_labels.csv` |
| 用当前最佳 Phase 2 模型推理 | `phase3_patch_predictions.csv` 或 `.npy` |
| 切片/患者聚合 | mean/max/var 聚合矩阵 |
| 5-fold AUC baseline | `phase3_baseline_auc_report.md` |

建议：先不引入 RL，不做通路选择，先用全 30 通路跑一个 Logistic Regression baseline。哪怕 AUC 不高，也能给项目一个真实的临床端反馈。

### 6.2 服务器自动化缺口

当前训练规模已经超过纯本地舒适区。服务器不通会影响：

- 9 折 LOPO。
- 在线 LoRA/解冻。
- 大规模扫参。
- 夜间无人值守训练。

建议优先级：

1. Tailscale 或 ZeroTier，优先解决“可达性”。
2. Windows OpenSSH 22330 入站端口，优先解决“标准化远程命令”。
3. HTTP 命令服务仅作为短期备选，安全性和维护性弱于 SSH/VPN。

### 6.3 文档中的计划扩散

当前文档中同时存在许多下一步方向：

- 染色归一化。
- UNI LoRA/渐进解冻。
- Virchow2 LoRA。
- per-pathway attention。
- CONCH / PathDINO / Prov-GigaPath。
- 多层 UNI 特征。
- GenePT/通路语义 Cross-Attention。
- RL B/D。
- BO 通路选择。
- 对比预训练。
- STFlow / STPath。

这些都不是坏方向，但不能并行作为近期重点。项目需要一个明确的“暂停队列”。

### 6.4 结果表述需统一

部分文档中存在以下小问题：

| 问题 | 建议 |

## 7. 不合理或需纠偏的决策点

### 7.1 不应把 RL 训练调度放在 Phase 3 闭环之前

理由：

- 方案 B 预期收益小。
- 离线 DQN 的历史 action 多样性不足。
- RL 成功也只是提升 Phase 2 PCC，而当前临床目标需要 Phase 3 AUC。

建议：方案 B 降级为“方法展示/训练管线实验”，先暂停主线资源。

### 7.2 不应再用单患者指标推动 backbone 选择

OmiCLIP 的案例已经证明单患者表现可以很好，但跨患者崩溃。后续新 backbone 如 CONCH、PathDINO、GigaPath，第一关必须是跨患者 Fold1 快速验证，不是 HYZ 单患者 val。

### 7.3 不应继续大幅扩展下游结构

GAT、全局 AttnPool、编码器加深、通路加权已经说明：在当前 3 例数据下，下游结构复杂化容易只增加参数，不增加信息。

建议：除非有强归纳偏置且参数增量小，否则不再做“更大下游模型”。

### 7.4 对 per-pathway attention 的定位要准确

它不是“AttnPool 复活”，而是针对全局 AttnPool 失败根因的新低参数假设。只有在额外参数很小、先跑 Fold1、对照 mean pooling 的前提下才值得做。

### 7.5 服务器问题不能继续拖延

这是工程瓶颈，不是附属问题。若 9 例数据到齐后服务器仍无法一键训练，项目迭代速度会被严重限制。

### 8.2 未来 2-4 周：模型改进建议

建议只选一个主攻方向：

| 方向 | 推荐程度 | 原因 |
|---|---|---|
| Phase 3 baseline + BO 通路选择 | 最高 | 直接服务临床 AUC，计算便宜，解释性强 |
| UNI2-H LoRA/渐进解冻 | 高，但依赖服务器 | 针对 frozen 特征瓶颈，但成本高 |
| 染色归一化 | 中高 | 针对跨患者域偏移，工程可控，但需重新提特征 |
| CONCH 新教师验证 | 中 | OmiCLIP 负结果提示 CoCa 不等于不可用，病理域 CoCa 可试，但先 Fold1 |
| per-pathway attention | 中低 | 参数小，可做，但不能压过 Phase 3 和服务器 |
| RL DQN 调度 | 低 | 先暂停，除非需要展示 RL 方法集成 |

### 8.3 9 例数据到齐后的标准流程

```text
1. 数据审计
   patch 数、标签列、坐标范围、缺失样本、标签分布

2. 特征提取
   先提 UNI tokens 干净缓存，不急着提增强/归一化多套缓存

3. 9 折 LOPO 基线
   先跑当前最优配置：UNI token + MixUp alpha=0.2 + TV L2 w=0.01

4. 统计报告
   overall PCC mean±std
   per-pathway PCC mean±std
   per-patient PCC 排序

5. 再决定旧负结果是否复查
   通路加权、编码器加深、图建模、MixUp alpha 重扫
```

### 8.4 决策门槛建议

为避免小数点追逐，建议设定硬门槛：

| 场景 | 有效提升门槛 |
|---|---|
| Fold1 快速筛选 | PCC 提升 >= 0.01 才进入三折 |
| 三折验证 | mean PCC 提升 >= 0.01 且 3 折都不明显退化 |
| 9 折 LOPO | paired test p < 0.05 或至少 7/9 折同向提升 |
| Phase 3 AUC | AUC 提升 >= 0.03 才值得作为临床方向 |
| 融合/后处理 | 增益 < 0.005 只作为附录，不作为主贡献 |
