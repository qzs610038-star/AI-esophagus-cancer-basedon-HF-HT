## 1. 研究背景（一句话版）

**我们在做什么**：从食管癌的病理切片图像（医生显微镜下看到的组织染色图），用AI预测该区域30条分子信号通路的活跃程度。

通俗类比：好比只看一个人的面相照片，预测他30项血液生化指标——图像能提供线索，但不可能100%准确。我们的目标是让AI的"看图猜指标"能力尽可能接近真实检测值。

**核心指标**：PCC（皮尔逊相关系数，衡量预测与真实的吻合度，0%=完全随机猜测，100%=完美预测）。当前最优水平：跨患者平均PCC=0.38（38%吻合度），目标：0.45-0.50。

## 3. 文献综述与差距分析结论

### 3.1 核心结论

**结论：学术界已从"纯看图预测"进入"图像+基因信息联合预测"时代，我们目前最大的短板不是模型架构不够好，而是完全没用上基因序列信息。**

### 3.2 文献覆盖范围

系统回顾37篇项目文献 + 追踪8篇2025-2026最新突破工作，覆盖五大方向：

| 方向 | 代表文献 | 对本项目启示 |
|------|---------|------------|
| 深度学习预测 | HisToGene, EGN, Hist2ST等12篇 | 架构优化的增量已趋近天花板 |
| 病理基础模型 | UNI, Virchow2, Prov-GigaPath等6篇 | 更大的"AI眼睛"可带来小幅提升 |
| **多模态融合** | FmH2ST, OmiCLIP, PEaRL等5篇 | **最大增量空间（+15-25%）** |
| 空间建模+GNN | GNN for NSCLC, STORM等3篇 | GAT理论有效但实验未兑现 |
| 数据质量评估 | 博弈论ST评估等5篇 | 数据预处理不可忽视 |

### 3.3 学术前沿演进（四代技术范式）

```
2020-2022: 纯看图预测（ST-Net, HisToGene, EGN）     ← 我们目前在这里
    ↓
2023-2024: 对比学习（BLEEP, mclSTExp）
    ↓
2024-2025: 超大基础模型（UNI2-h, Virchow2）          ← 我们的特征提取器在这里
    ↓
2025-2026: 图像×基因 多模态融合（HistoPrism, PEaRL） ← Phase2目标
```

### 3.4 项目与前沿的差距矩阵

| 差距维度 | 我们的现状 | 最新文献做法 | 差距 | 预期收益 |
|---------|-----------|------------|:----:|:-------:|
| **多模态融合** | 只用图像 | 图像×基因序列联合 | ★★★★★ | +0.05~0.10 |
| 训练范式 | 纯监督回归 | 对比预训练+回归微调 | ★★★★☆ | +0.03~0.06 |
| 域适应 | 无 | 染色归一化+对抗训练 | ★★★★☆ | +0.03~0.06 |
| 特征提取器 | UNI2-h（当前最优之一） | Virchow2(632M参数) | ★★★☆☆ | +0.02~0.05 |
| 空间建模 | Transformer/GAT | 层级空间编码器 | ★★★☆☆ | +0.02~0.04 |

**一句话总结**：补上"多模态融合"这块最大短板，比任何架构优化的收益都大。

**结论：9个改进方向按"投入产出比"分为三级，P0-2多模态融合是唯一预期收益超过10%的方案，应作为核心突破口。**

```
优先级    改进方向                           预期PCC增量    时间
────────────────────────────────────────────────────────────
P0-1 🔴  升级"AI眼睛"(UNI→Virchow2)       +0.02~0.05    3-4天
P0-2 🔴  多模态融合：引入基因序列信息(★核心) +0.05~0.10    5-7天
P0-3 🔴  GAT空间邻域建模                    ❌ 已验证无效  已完成
────────────────────────────────────────────────────────────
```

### 4.2 P0-2多模态方案详解（最大预期收益）

**为什么这是破局关键？**

通俗解释：目前AI只是"看图猜答案"，而我们其实手上有一份"答案的组成说明书"（每条通路由哪些基因组成）。多模态融合就是**让AI一边看图、一边参考说明书**——"这张图里有没有和这些基因相关的线索？"

**技术方案概览**：

```
病理图像 ──→ UNI2-h"AI眼睛" ──→ 图像特征
                                    │
30条通路的基因列表 ──→ GenePT ──→ 通路嵌入(通路的"身份证")
                                    │
              ┌─────────────────────┘
              ▼
    交叉注意力(图像问通路："我看到的跟你相关吗？")
              │
              ▼
         输出：30条通路评分预测
```

| 关键组件 | 方案 | 通俗解释 |
|---------|------|---------|
| 基因语言模型 | GenePT（1536维输出） | 把基因名翻译成AI能理解的数字编码 |
| 融合机制 | 2层交叉注意力（8头） | 图像和通路之间"对话"两轮 |
| 训练损失 | MSE + 0.1×InfoNCE | 既要预测准，也要让配对的图像-通路关系更紧密 |
| 预期收益 | PCC +0.05~0.10（+10-25%） | 有望从0.38突破到0.43-0.48 |

## 5. P0-3 GAT空间建模实验（W19完成）

### 5.1 核心结论

**结论：GAT空间建模对当前方案提升几乎为零（PCC 0.4068 vs 基线 0.4095，差异仅-0.27%），实验证明——当AI的"眼睛"已经足够好时，"多问邻居意见"基本没有帮助。**

### 5.2 为什么要做这个实验（医生可理解版）

**类比**：想象一位病理医生在看一个组织区域的切片——

- **没有GAT**：他只看自己负责的那块区域，独立判断
- **有GAT（图注意力网络）**：他不仅看自己的区域，还主动"询问"周围6个邻居区域："你们那边情况如何？有没有看到肿瘤/炎症的迹象？"然后根据邻居的回答调整自己的判断

**实验目的**：验证这种"邻里会诊"机制能否提高预测准确度。

### 5.3 实验结果

| 指标 | GAT模型 | 基线（方案B） | 差异 | 通俗解读 |
|------|:-------:|:----------:|:----:|---------|
| 最佳PCC | 0.4068 | 0.4095 | **-0.27%** | 加了邻居信息反而微降，在噪声范围内 |
| 收敛速度 | 50个epoch | ~3个epoch | 慢17倍 | 花了17倍时间才勉强追平基线 |
| 过拟合程度 | Gap=0.194 | Gap=0.171 | 略加剧 | "背答案"倾向轻微增加 |
| 参数量 | 13.46M | — | — | 新增4个代码文件 |

**收敛过程**（PCC随训练轮次的变化）：

| 阶段 | Epoch | PCC变化 | 发生了什么 |
|------|:-----:|:-------:|-----------|
| Stage1（冻结训练） | 1→10 | 0.16→0.36 | GAT层快速学会基本工作 |
| Stage2前半（联合训练） | 11→30 | 0.36→0.40 | 缓慢爬升，开始接近基线 |
| Stage2后半（精调） | 31→50 | 0.40→0.41 | 几乎停滞，已到天花板 |

### 5.4 为什么GAT没有带来预期提升

| 原因 | 通俗类比 | 技术解释 |
|------|---------|---------|
| **AI眼力已经很好** | 一位经验丰富的医生，视力本就极佳，多问隔壁同事意见改善空间不大 | UNI2-h（13亿参数预训练的特征提取器）提取的特征本身已含丰富上下文 |
| **跨患者邻域关系不稳定** | 在A小区学到的"邻里关系"，到B小区完全不适用 | 不同患者的空间拓扑差异大，图注意力权重难以跨患者迁移 |
| **训练数据不够支撑新参数** | 教材变厚了但学习时间没增加 | 参数增至13.46M但训练样本仍~15000个 |

**一句话教训**：在特征提取器已经很强时，"换更好的架构"远不如"引入新信息维度"有效。好比给一位视力满分的医生配放大镜（GAT），不如给他一份血液化验报告（基因序列多模态）。

### 5.5 Loss偏高的原因解读

> 有人可能问：训练Loss=0.66、验证Loss=0.86看起来挺高的，模型是不是没学好？

**答案：这不是模型有问题，而是任务本身的固有难度。**

| 问题 | 解释 |
|------|------|
| Loss=0.66意味什么？ | 30条通路MSE均值。经z-score标准化（把不同患者的指标统一到同一把尺子上）后，每条通路平均偏差仅≈**0.15个标准差**。通俗说：模型说"活跃度高1分"，实际是"高0.85分"，偏了0.15——属正常水平 |
| 为什么不能降到0？ | 只看组织染色图就想精确预测所有分子活动=只看面相精确预测血液指标——不可能100%准确 |
| 与基线对比如何？ | 两者Loss水平相当，说明这是任务难度上限而非GAT特有问题 |

## 6. 综合结论与决策

### 6.1 两周核心结论汇总

| # | 结论 | 依据 | 意义 |
|:-:|------|------|------|
| 1 | **多模态融合是最大突破口** | 文献一致表明+15-25%提升；项目基因信息完全未利用 | Phase2核心方向确立 |

| 决策 | 内容 | 理由 |
|------|------|------|
| **暂停GAT深化** | P0-3实验结论：增益<0.01%，资源撤出 | 投入产出比过低 |
| **全力推进P0-2** | 下周集中实现多模态融合 | 唯一预期收益>10%的方案 |
| **保留GAT为后备** | 待多模态引入新信息后，邻域建模可能重新有效 | 不排除组合使用可能 |
| **Virchow2评估并行** | P0-1同步进行，不阻塞P0-2 | 低成本验证，锦上添花 |

## 附录：技术术语速查表

*供非AI背景的组会参与者快速查阅*

| 术语 | 通俗解释 |
|------|---------|
| **PCC** | 皮尔逊相关系数。衡量预测与真实的吻合度。0=瞎猜，1.0=完美预测。我们目前0.38 |
| **UNI2-h** | AI的"眼睛"——一个13亿参数的预训练模型，负责从病理图像中提取有意义的特征 |
| **Transformer** | AI看图时的"注意力分配"机制，决定重点关注图像中的哪些区域 |
| **GAT（图注意力网络）** | 让每个组织区域主动"询问"周围邻居："你们那边情况如何？"的机制 |
| **过拟合** | 学生只背了标准答案没学到解题方法，换一套卷子就考不好 |
| **跨患者泛化** | 在一位患者身上总结的经验，能否直接用于另一位新患者 |
| **z-score标准化** | 把不同患者的指标统一到同一把尺子上比较（消除个体基准差异） |
| **多模态融合** | 同时使用多种信息源（图像+基因序列）进行预测，像医生同时看片子和化验单 |
| **交叉注意力** | 图像特征和通路特征之间"对话"的机制："我看到的跟你相关吗？" |
| **GenePT** | 基因语言模型，把基因名/通路信息翻译成AI能理解的数字编码 |
| **InfoNCE损失** | 一种训练约束——让配对的"图像-通路"关系更紧密，不配对的推远 |
| **域偏移** | 不同患者的组织切片"风格"不同（如染色深浅），导致模型水土不服 |
| **Epoch** | 模型把所有训练数据完整学习一遍=1个epoch。50 epoch=学习了50遍 |
| **方案B** | 当前最优的HisToGene-UNI Token+强正则化方案，三折平均PCC=0.3812 |
