## 关键术语：严格定义与通俗理解对照

> 本节为全文技术术语提供**双重解释**——严格定义（面向 ML 专业人员）与通俗类比（面向医学背景听众），两者对齐，确保不同背景的听众均能准确理解。

### 1. PCC（皮尔逊相关系数，Pearson Correlation Coefficient）

| | 内容 |
|---|---|
| **严格定义** | 给定两组配对观测值 {(xᵢ, yᵢ)}ⁿᵢ₌₁，PCC = Cov(X,Y) / (σₓ · σᵧ) = Σ(xᵢ−x̄)(yᵢ−ȳ) / √[Σ(xᵢ−x̄)² · Σ(yᵢ−ȳ)²]。值域 [−1, 1]，+1 = 完全线性正相关，0 = 无线性相关，−1 = 完全负相关。本任务中 X=30条通路ssGSEA真实评分，Y=模型预测评分，n=测试集patch数×30。 |
| **通俗理解** | 预测值与真实值的"吻合程度"。1.0 = 完美预测（AI 说的和实际完全一样），0.5 = 大致趋势对但细节不准，0 = 毫无关联（瞎猜水平），负数 = 预测方向反了。 |

### 2. HuberLoss

| | 内容 |
|---|---|
| **严格定义** | 分段损失函数，δ 为切换阈值（本任务 δ=1.0）：<br> 当 \|y−ŷ\| ≤ δ 时：L = ½(y−ŷ)²（MSE分支，对小误差精确拟合）<br> 当 \|y−ŷ\| > δ 时：L = δ·\|y−ŷ\| − ½δ²（MAE分支，对大误差/异常值线性惩罚，避免MSE的平方放大效应）<br>在 δ 处两分支值和一阶导数连续（C¹光滑）。 |
| **通俗理解** | 模型预测偏差的"惩罚分数"。小错按平方罚（要求精确），大错按绝对值罚（防止个别极端样本绑架整个训练方向）。相当于老师批卷：小错误扣分严格，但对"完全离谱的答案"不额外加重惩罚，避免学生为了这一个题荒废其他题。 |

### 3. UNI2-h（病理基础模型 / Pathology Foundation Model, PFM）

| | 内容 |
|---|---|
| **严格定义** | 基于 DINOv2（自蒸馏 Vision Transformer）在 ~10 万张 WSIs（Whole Slide Images，全切片病理图像）上自监督预训练的 ViT-h/14 模型（307M 参数）。输入 224×224 H&E patch → 输出 CLS token [1, 1536] 或 register tokens [1, 65, 1536]（含 1 个 CLS + 64 个 register）。本任务冻结 backbone 权重，离线提取特征缓存为 .pt 文件，训练时仅优化下游回归头。 |
| **通俗理解** | AI 的"眼睛"——已在海量病理切片上预习过，能从染色组织中看出有意义的信息。所谓"冻结"就是眼睛的参数不改变，只训练后面负责"理解"的大脑。 |

### 4. Virchow2（病理基础模型）

| | 内容 |
|---|---|
| **严格定义** | 基于 ViT-H/14 架构的病理基础模型（632M 参数），在 310 万张 WSIs 上自监督预训练（Nature Medicine 2024）。使用 DINOv2 + iBOT 联合自监督框架，输出特征维度 1280（< UNI 的 1536）。本任务同样冻结权重 + 离线缓存方案。 |
| **通俗理解** | 另一个更强大的 AI 眼睛——看的切片数量是 UNI 的 30 倍（310万 vs 10万），理论上"阅历"更丰富，但维度略低（1280 vs 1536），相当于"看的种类多但单张精度略粗"。 |

### 5. OmiCLIP / CoCa（多模态病理-基因基础模型）

| | 内容 |
|---|---|
| **严格定义** | 基于 CoCa（Contrastive Captioners, Google/DeepMind 2022）架构的 ViT-L/14 模型（307M 参数）。**CoCa 核心设计**：单一 ViT 编码器 → 两个输出头共享——(a) 对比学习头通过 Attentional Pooler 将图像特征池化为单一嵌入向量，与文本嵌入计算 CLIP-style contrastive loss；(b) 生成头（captioning head）用交叉注意力解码器从图像特征生成文本描述（LM loss）。两个头联合训练，使视觉编码器同时学习"判别性表示"（对比头）和"结构化语义"（生成头）。**OmiCLIP 变体**：将文本模态替换为基因表达数据（病理图像-基因表达配对），256 个 learnable queries + 1 个 CLS 构成 255 个 register tokens [B, 255, 768]。期望：对比学习使图像特征空间与基因表达空间对齐，token 级别特征应编码"基因通路相关的形态学模式"。本任务冻结 backbone 权重，离线提取 token 缓存。与 UNI/Virchow2 的关键区别：CoCa 先在 LAION-2B（自然图像+文本）预训练 → 再在病理-基因配对数据上 fine-tune，而非纯病理自监督。 |
| **通俗理解** | 第三个 AI 眼睛。与 UNI/Virchow2 只看病理切片不同，OmiCLIP 在训练时同时"看图"和"看化验单"——专门学习了"什么样的组织形态对应什么样的基因表达"。打个比方：UNI 是只看片子的影像科医生，OmiCLIP 是同时看了片子和化验单的分子病理医生。理论上应该更懂基因通路，但实际上跨患者迁移极差（见 §2b）。 |

**CoCa 架构关键组件**：

```
输入图像 (224×224)
      │
      ▼
ViT-L/14 Encoder (307M)
  ├── Patch Embedding: 256 patch tokens [256, 1024]
  │
  ▼
Attentional Pooler (learnable queries)
  ├── 1 × CLS query        ──→ CLS token    [1, 768]
  ├── 254 × Learnable queries ──→ Register tokens [254, 768]
  └── 输出: concat → [B, 255, 768]
      │
      ├──→ 对比头 (Contrastive Head):
      │      CLS token → Linear Proj → L2 Norm → Image Embedding [B, 768]
      │      与基因表达 Embedding 做 CLIP loss (对比学习)
      │      目标: 匹配的(图像,基因)对靠近，不匹配的远离
      │
      └──→ 生成头 (Captioning Head) — OmiCLIP 中替换为基因表达解码:
             Register tokens → Cross-Attention Decoder → 基因表达值
             目标: 从图像特征直接"翻译"出基因表达谱
```

> **关键理解**：Attentional Pooler 不是简单 mean pooling——254 个 learnable queries 是可训练向量，通过交叉注意力从 ViT patch tokens 中"主动查询"信息。每个 query 学到关注图像中不同的组织形态学模式。这 255 个 token 理论上应比 UNI 的 65 个 DINOv2 register tokens 包含更丰富的基因相关空间信息。**

**选择 OmiCLIP 的初始动因**：

| 动因 | 说明 | 实际结果 |
|------|------|:---:|
| **任务对齐** | OmiCLIP 是少数直接在"病理图像→基因表达"任务上训练的 PFM，与我们 30 通路 ssGSEA 预测任务高度一致 | 单患者有效（PCC=0.55），验证了任务对齐性 |
| **多模态先验** | CoCa 的对比学习使图像特征空间与基因表达空间对齐——理论上，相似的 H&E 形态应映射到相近的基因表达特征 | 单患者有效，但跨患者时形态-基因映射不迁移 |
| **Token 丰富度** | 255 个 register tokens（vs UNI 的 65 个）——期望更细粒度的空间信息编码，每个 token 捕捉不同的组织微环境模式 | 反而成为冗余源（3.9× token 数，小数据下信息/噪声比极低） |
| **SOTA 文献支撑** | NeurIPS 2025 发表，CoCa 架构在当时被认为是"视觉-语言"预训练的最优框架之一 | 文献中的成功场景（自然图像→文本）与我们的边缘场景（食管癌 H&E→ssGSEA）差异大 |
| **UNI 对比参照** | 引入一个"不同预训练范式"的 backbone，验证"预训练策略是否比预训练数据量更重要" | 结论：对跨患者泛化，预训练数据域（病理 vs 自然图像）> 预训练任务设计 |

> **经验教训**：选择 backbone 时，"任务对齐性"（单患者评估）不等于"泛化对齐性"（跨患者评估）。OmiCLIP 的 CoCa 预训练在自然图像上 → 其视觉底层表示仍偏向自然图像纹理/物体轮廓，在食管癌 H&E 组织形态上的泛化能力被高估。未来引入新 backbone 时，必须以"跨患者 Fold1 快速验证（~15 min）"而非"单患者拟合"作为选型第一关。

### 6. Manifold MixUp（特征空间混合增强）

| | 内容 |
|---|---|
| **严格定义** | 给定 batch 内两组 token 特征 (xₐ, yₐ), (x_b, y_b)，从 Beta(α, α) 采样 λ，构造混合：x̃ = λ·xₐ + (1−λ)·x_b，ỹ = λ·yₐ + (1−λ)·y_b。与传统 MixUp（像素空间混合）不同，Manifold MixUp 在 backbone 提取的高维语义特征空间进行线性插值。α→0 时 Beta 分布集中在 {0,1}（保守混合），α=1 时 = Uniform(0,1)（激进混合），α→∞ 时集中在 0.5。本任务最优 α=0.2。 |
| **通俗理解** | 不是在图像上把两张病理切片叠在一起（那会产生无意义的"糊图"），而是在 AI 的"脑海"（特征空间）里把两个患者的信息按比例融合。好比医生不是把两张片子物理重叠，而是在脑中综合两个病例的特征。α 参数控制"混合的激进程度"——越小越保守（大部分时候保留原样）。 |

### 7. 空间 Total Variation Loss（TV Loss，全变分损失）

| | 内容 |
|---|---|
| **严格定义** | 给定 batch 内 B 个 patch，以空间坐标构建 KNN 图（k 近邻，默认 k=6），边集 E。L1 模式：L_TV = (1/|E|) · Σ_{(i,j)∈E} ‖ŷᵢ − ŷⱼ‖₁，L2 模式：L_TV = (1/|E|) · Σ_{(i,j)∈E} ‖ŷᵢ − ŷⱼ‖₂²。总损失 = HuberLoss + λ_tv · L_TV。该正则项基于生物学先验——物理相邻的组织区域其基因表达谱应为连续渐变。推理零开销（仅在训练时计算）。 |
| **通俗理解** | 组织切片上相邻区域基因表达不会"断崖式突变"——这是生物学常识。TV Loss 把这个常识变成数学约束：模型预测相邻 patch 差异太大就扣分。相当于告诉 AI："记住，组织是连续渐变的，别让相邻区域的预测值差太多。" |

### 8. 晚期融合 / 集成学习（Late Fusion / Ensemble）

| | 内容 |
|---|---|
| **严格定义** | 给定 K 个已训练模型对同一测试样本的预测 {ŷ⁽ᵏ⁾}ᴷₖ₌₁，构造融合预测：ŷᶠᵘˢⁱᵒⁿ = Σₖ wₖ ⊙ ŷ⁽ᵏ⁾，其中 wₖ ∈ [0,1]³⁰ 为 per-pathway 权重向量，满足 Σₖ wₖ = 1（逐通路归一化）。本任务实现 5 种权重策略：(a) Uniform wₖ=0.5，(b) PCC-Weighted 按验证集 per-pathway PCC 比例分配，(c) Global Grid Search 全局统一标量 w，(d) Per-Pathway Grid Search 每条通路独立搜索（步长 0.05），(e) Oracle 每条通路选 PCC 更高模型（二值，理论上界）。 |
| **通俗理解** | 两个医生各自看片子给出诊断。晚期融合决定"如何综合两位医生的意见"——不是简单取平均，而是根据每位医生擅长的领域分配权重：代谢问题听 UNI 医生的，免疫问题听 Virchow2 医生的。 |

### 9. 注意力池化（Attention Pooling）

| | 内容 |
|---|---|
| **严格定义** | 将 LightweightTokenEncoder 中的等权平均池化 h = (1/N)·Σᵢ xᵢ 替换为可学习加权：aᵢ = softmax(MLP(xᵢ))，h = Σᵢ aᵢ·xᵢ。MLP: Linear(512→128) → ReLU → Linear(128→1)。参数量增加 ~66K（hidden_dim=512 时）。跨患者场景因注意力模式患者特异性导致过拟合加剧（PCC −4.0%），已拒绝用于跨患者。 |
| **通俗理解** | 当前模型看 65 个 patch 子区域时"一视同仁"（等权平均）。注意力池化让模型学着"该重点看哪里"——查免疫通路时自动聚焦淋巴细胞浸润区域，查代谢时聚焦细胞密度高的区域。 |

### 10. 过拟合 Gap

| | 内容 |
|---|---|
| **严格定义** | Gap = Train_PCC − Val_PCC，衡量模型对训练集拟合程度与泛化能力的差距。Gap 大 → 模型记忆了训练数据噪声（"背答案"），对未见数据预测能力弱。Gap 小 + Val_PCC 高 → 理想状态。Gap 小 + Val_PCC 低 → 欠拟合（"还没学会"）。 |
| **通俗理解** | 学生在模拟题上得分很高（Train PCC↑），但真考时分数大跌（Val PCC↓）→ 说明他只背了答案没学方法。Gap 就是"模拟分 − 真考分"的差值，越大说明越会"背答案"。 |

## 2. 前置工作：Virchow2 跨患者训练与验证

### 2.1 这是什么

> **通俗类比**：上周我们决定换个更强大的"AI眼睛"（Virchow2，632M参数，310万张切片预训练），本周完成了实际训练验证。

### 2.2 实验结果

| 模型 | 特征维度 | 参数量 | Fold1 Val PCC | 最佳Epoch | 训练时间 |
|------|:------:|:------:|:------------:|:--------:|:------:|
| HisToGene-UNI-Tokens + AugMix | 1536 | 9.8M | **0.4142** | 4 | ~15 min |
| HisToGene-Virchow2-Tokens | 1280 | 7.2M | 0.3537 | 11 | ~12 min |

> PCC（皮尔逊相关系数）：预测值与真实值的吻合程度，1.0=完美预测，0=毫无关联。

### 2.3 关键发现

**发现 1：Virchow2 独立使用时不及 UNI2-h，差距 −14.6%**

独立使用时，Virchow2（PCC=0.3537）明显弱于 UNI2-h AugMix（PCC=0.4142）。27/30 条通路上 UNI2-h 更优，Virchow2 仅在 3 条通路上有边际优势。

**发现 2：互补通路恰好是上周的"困难通路"**

| 通路 | UNI2-h PCC | Virchow2 PCC | 更优模型 | 上周状态 |
|------|:----------:|:------------:|:------:|:------:|
| Interferon_Alpha（干扰素α） | −0.004 | **−0.002** | Virchow2 | 上周最困难通路 |
| ifng（干扰素γ） | 0.124 | **0.128** | Virchow2 | 免疫通路，长期困难 |
| TNF_Signaling（TNF信号） | 0.220 | **0.262** | Virchow2 | 免疫通路 |

> **给医生的解释**：Virchow2 虽然"总分"不如 UNI，但在三条免疫相关通路上反而更强。这提示两个 AI 眼睛"看问题的角度不同"——UNI 擅长看代谢和结构，Virchow2 在免疫信号上更敏感。这正是"融合"策略的理论基础。

**发现 3：Virchow2 训练更稳定但峰值更低**

Virchow2 的过拟合 Gap 仅 ~0.09（vs UNI AugMix 的 ~0.23），收敛更平滑。但 HuberLoss 收敛值 (~0.89) 高于 UNI (~0.32)，提示特征质量或任务匹配度不如 UNI。

### 2.4 训练脚本改进

本周对 `train_histogene_virchow2_tokens.py` 做了以下增强：
- 新增 `--dropout` 参数化（默认 0.5）
- 新增梯度裁剪（`gradient_clip=1.0`）
- 评估集改为全量（train+val），避免 val-only 偏差

## 2b. 前置工作：OmiCLIP 跨患者验证（回顾）

> OmiCLIP 训练于上周完成（5.19-5.21），本周纳入性能排名进行横向对比分析。此处仅报告与 UNI/Virchow2 对比的关键发现。

### 2b.1 这是什么

**严格定义**：OmiCLIP 基于 CoCa（Contrastive Captioners）架构，ViT-L/14 backbone（307M 参数），在病理图像-基因表达配对数据上对比学习预训练。vision encoder 输出 register tokens [B, 255, 768]。本任务中冻结 backbone + 离线缓存，下游使用与其他 backbone 相同的 LightweightTokenEncoder + CoordEmbed + MLP 架构（~6.1M 参数）。

> **通俗类比**：OmiCLIP 是一个"既看过切片又看过化验单"的 AI 眼睛——在预训练阶段同时学了图像和基因数据，理论上比只看切片的 UNI 更理解基因通路。但实际上跨患者迁移严重失败（见下文）。

### 2b.2 与 UNI/Virchow2 的三维对比

| 维度 | UNI2-h | Virchow2 | OmiCLIP |
|------|:------:|:--------:|:------:|
| 预训练数据 | ~10万 WSIs（纯图像） | 310万 WSIs（纯图像） | 病理图像 + 基因表达（多模态） |
| 特征维度 | 1536 | 1280 | 768 |
| Token 数 | 65 | — | 255 |
| 单患者平均 PCC | 0.5265~0.5773 | — | **0.5541** |
| 跨患者 3折平均 PCC | **0.3700** | 0.3537 (仅Fold1) | **0.1970** |
| 泛化衰减 | **−29.7%~−35.9%** | — | **−64.4%** |
| 最佳 Epoch | 3-5 | 11 | **1**（立即过拟合） |

> 数据来源：单患者 OmiCLIP 结果（HYZ=0.5275, JFX=0.6082, LMZ=0.5267），跨患者 3 折（Fold1=0.2544, Fold2=0.1081, Fold3=0.2284）。详见 `模型性能排名_Model_Performance_Ranking.md` §OmiCLIP 系列。

### 2b.3 预期 vs 现实：选择动因与实际表现的落差

> 选择 OmiCLIP 的初始假设是"在病理+基因配对数据上预训练的 backbone 应天然更适配基因通路预测任务"。下表对照初始假设与实验结果。

| 初始假设 | 预期 | 实际结果 | 落差分析 |
|------|:---:|:---:|------|
| "CoCa 对比学习使图像特征与基因表达对齐" | 通路预测 PCC 应显著优于纯图像 UNI | 单患者持平 0.55，跨患者 0.19（远低于 UNI 的 0.37） | 对齐仅在训练分布内成立——CoCa fine-tune 用的病理-基因数据来自不同癌种/人群，与食管癌 HYZ/JFX/LMZ 的域偏移大 |
| "255 token 提供更丰富的空间粒度" | 更多 token → 更细粒度的组织微环境编码 | 反而导致严重过拟合（最佳 Epoch=1） | 255 token 中有信息量的可能仅少数，其余是冗余/噪声。小数据下 255 维 token 序列的方差远高于 65 维 |
| "生成头（captioning → 基因解码）提供的梯度有益" | 联合训练使视觉编码器学到"基因相关的视觉概念" | 自然图像预训练的视觉底层表示未改变 | 生成头仅在 fine-tune 阶段训练，不足以重塑 ViT 底层 filter——其仍偏向自然图像纹理而非组织形态学 |
| "多模态 backbone 比单模态更具可迁移性" | 跨患者泛化应更鲁棒 | 跨患者 −64.4%，最差 | 反直觉结论：**预训练数据域（病理 vs 自然图像）比预训练任务（多模态 vs 单模态）对跨患者泛化更重要** |

### 2b.4 关键发现

**发现 1：单患者与 UNI 持平，跨患者崩溃**

```
单患者: OmiCLIP=0.5541 ≈ UNI系列(0.52-0.58)  → 同等水平
跨患者: OmiCLIP=0.197  ≪ UNI系列(0.37)      → 衰减 −64.4%
```

> OmiCLIP 是唯一一个"单患者表现好但跨患者完全不可用"的 backbone。这一模式与 CoCa 预训练策略直接相关。

**发现 2：Fold2 (→JFX) PCC 仅 0.1081，几乎随机水平**

三折中 Fold2 最低（0.1081），比 Fold1（0.2544）再降 −57.5%。即使是表现最差的 UNI 模型在 Fold2 也有 0.3424。OmiCLIP 在 JFX0729 患者上的预测效果接近随机猜测。

**发现 3：最佳 Epoch = 1 → 立即过拟合**

所有 OmiCLIP 跨患者实验的最佳 epoch 均为 1（或极早期），之后 validation PCC 持续下降。而 UNI 系列最佳 epoch 通常为 3-14。这说明 CoCa 特征与食管癌 ssGSEA 任务的匹配度极差——模型在"看了一眼训练数据后立即开始背答案"。

### 2b.5 失败根因分析

| 原因层级 | 具体分析 |
|------|---------|
| **表层原因** | 255 token × 768 dim 的特征空间包含大量冗余信息（token 数是 UNI 的 3.9×），而训练样本仅 15K，信息/噪声比极低 |
| **中层原因** | CoCa 预训练使用自然图像+文本（LAION-2B），仅 fine-tune 阶段使用了病理-基因配对数据。其视觉表征仍然偏向自然图像语义（物体、场景），而非组织形态学 |
| **深层原因** | 对比学习目标（CLIP loss）学到的特征是"区分不同样本"的判别特征，而非"构建连续回归映射"的连续特征。在回归任务（ssGSEA 预测）中，这些判别特征缺乏平滑泛化能力 |
| **架构原因** | 当前使用 register tokens (255) 做 mean pooling，缺乏预训练阶段的 token 选择/降维策略。对比 UNI 的 65 tokens（经过专门设计的 register 选择），OmiCLIP 的 token 质量显著更低 |

> **给医生的解释**：OmiCLIP 就像一个"在国外学医"的学生——它的预训练（CoCa）主要用自然图片+文字，就像在国外用英语教材学医。虽然最后 fine-tune 阶段补了一些病理课（病理-基因配对数据），但基础思维框架还是"自然图片模式"——看到一张病理切片，它潜意识里还在用"这是猫还是狗"的方式理解。到了中国医院（食管癌），完全不适应。

### 2b.6 结论与处理

```
OmiCLIP 跨患者泛化 = 不满足基本要求 (PCC < 0.20)
→ 不纳入跨患者模型选择
→ 不参与融合方案
→ 仅在单患者场景可考虑使用（但无实际部署价值）
→ 未来若用 CoCa 架构从头在病理数据上预训练，可能改善
```

> 这个负结果也验证了 CLUADE.md 的结论："UNI backbone 是关键——所有使用 UNI2-h 特征的模型，跨患者 PCC 均 > 0.34；不使用 UNI 的模型跨患者 < 0.20"。OmiCLIP 加入了这个"不适用"列表。

## 3. Phase 1 + Phase 4：多教师晚期融合（UNI + Virchow2）

### 3.1 这是什么

> **通俗类比**：两个医生（UNI 和 Virchow2）各自看同一张片子给出诊断。融合策略就是"如何综合两位医生的意见"——不是简单取平均，而是根据每位医生擅长的领域分配权重。比如代谢通路听 UNI 的，免疫通路听 Virchow2 的。

### 3.2 融合策略与结果

**融合实验 1（= Phase 1）：UNI AugMix (0.4142) + Virchow2 (0.3537)**

| 融合策略 | PCC | vs 最佳单模型 |
|---------|:----:|:------------:|
| UNI2-h AugMix（单模型 A） | 0.4142 | — |
| Virchow2（单模型 B） | 0.3516 | −15.1% |
| Uniform（w=0.5 等权） | 0.4076 | −1.6% |
| PCC-Weighted（按通路PCC比例） | 0.4105 | −0.9% |
| Global Search（全局统一w=0.80） | 0.4164 | +0.5% |
| **Grid Search（逐通路最优w）** | **0.4167** | **+0.6%** |
| Oracle（理论上界） | 0.4148 | +0.2% |

**融合实验 2（= Phase 4）：UNI AugMix+TV (0.4212) + Virchow2 (0.3537)**

| 融合策略 | PCC | vs 最佳单模型 |
|---------|:----:|:------------:|
| UNI2-h AugMix+TV（单模型 A） | 0.4212 | — |
| Virchow2（单模型 B） | 0.3516 | −16.5% |
| PCC-Weighted | 0.4159 | −1.3% |
| Global Search（w=0.85） | 0.4228 | +0.4% |
| **Grid Search（逐通路最优w）** | **0.4242** | **+0.7%** |

### 3.3 关键发现

**发现 1（Phase 4 核心结论）：融合收益主要来自 TV Loss，Virchow2 仅是锦上添花**

```
UNI AugMix (0.4142) + Virchow2 (0.3537) → 融合 0.4167 (+0.0025, +0.6%)
UNI AugMix+TV (0.4212) + Virchow2 (0.3537) → 融合 0.4242 (+0.0030, +0.7%)
```

融合本身仅提升 ~+0.0025-0.0030。主要提升来自 TV Loss（+0.0070），融合只是锦上添花。**总提升 0.4142→0.4242 = +2.4%，其中 TV Loss 贡献 +1.7%，融合贡献 +0.7%。**

> **给医生的解释**：融合策略相当于"让两个医生会诊"——但 UNI 这个医生在 27/30 个问题上更强，Virchow2 只在 3 个问题上稍微好一点。所以综合意见的提升有限。真正的提升来自 TV Loss 这个"训练技巧"。

**发现 2：通路权重分布验证互补性假设**

逐通路最优权重 w（w=1.0 = 全听 UNI，w=0.0 = 全听 Virchow2）：

| 权重大于 0.8（UNI 主导） | 权重 0.4-0.6（均衡） | 权重小于 0.4（Virchow2 主导） |
|------------------------|-------------------|---------------------------|
| 20 条通路 | 7 条通路 | 3 条通路 |
| E2F_Targets, Apoptosis, DNA_Damage, Coagulation... | Hypoxia, ICP, IL6_JAK_STAT3, Complement... | **Interferon_Alpha (w=0)**, **TNF_Signaling (w=0.3)**, ifng (w=0.4) |

> Virchow2 在 Interferon_Alpha 通路上权重为 0——说明融合自动选择了"完全不听 UNI，只听 Virchow2"。

**发现 3：Oracle 理论上界仅 0.4148，远低于 Grid Search 0.4167**

Oracle（每条通路直接选 PCC 更高的模型）反而低于 Grid Search（逐通路加权平均），说明**即使是 UNI 表现较差的通路，完全切换为 Virchow2 也不如保留部分 UNI 信息**——两个模型的信息有互补但非替代关系。

## 4. Phase 2：空间 TV Loss

### 4.1 这是什么

> **通俗类比**：组织切片上相邻区域的基因表达应该是渐变的，不会从 A 区到 B 区突然"断崖式"变化。TV Loss 就是给模型加了一个"平滑约束"——如果相邻两个 patch 预测值差太多，就惩罚它。相当于告诉 AI："记住，组织是连续的，别乱跳。"

### 4.2 技术实现

- **新文件** `spatial_tv_loss.py`：纯 PyTorch 实现，batch 内 KNN 建图（k=6），L1/L2 两种模式
- 集成到 `train_histogene_uni_tokens.py` 和 `train_histogene_uni_tokens_augmix.py`
- 新增 CLI 参数：`--tv_weight`（默认 0 禁用）、`--tv_k`（默认 6）、`--tv_mode`（l1/l2/laplacian）
- 不修改模型架构，仅训练时附加 loss；推理零开销

### 4.3 实验结果

| 配置 | PCC | vs 基线 | 训练时间 |
|------|:----:|:------:|:------:|
| UNI AugMix（基线） | 0.4142 | — | ~15 min |
| UNI AugMix + TV L1（w=0.05, k=6） | **0.4212** | **+1.7%** | ~16 min |

> 数据来源：`ensemble_results/fusion_tv_virchow2/fusion_report.json`（model A PCC=0.4212）

TV Loss 的超参数仅测试了一组（w=0.05, k=6, l1），未来可进一步扫参优化。

## 5. AttnPool 注意力池化（Phase 3 — 拒绝）

### 5.1 这是什么

> **通俗类比**：当前模型看 65 个 patch 子区域时是"一视同仁"（等权平均）。注意力池化让模型自己学着"该重点看哪里"——比如看免疫通路时自动关注淋巴细胞浸润区域，看代谢时关注细胞密度高的区域。

### 5.2 实验结果

| 场景 | 配置 | PCC | vs 等权池化 | 过拟合 Gap |
|------|------|:----:|:----------:|:----------:|
| 单患者（HYZ15040） | AttnPool | — | +0.0036 | — |
| 跨患者 Fold1 | AttnPool | **0.4045** | **−0.0167 (−4.0%)** | 加剧 |

> 跨患者场景下 AttnPool 不仅未带来提升，反而使 PCC 从 0.4212（TV 基线）降至 0.4045，**净损失 −4.0%**。该方向经完全验证后明确拒绝用于跨患者场景。

AttnPool 额外增加 ~2.1M 参数（+27%），在小数据跨患者场景下过拟合更严重。已保留为可选特性（`--use_attn_pool` flag），但当前最优方案中不启用。

同步实现了 **TokenDrop** 正则化（训练时随机丢弃 token，`--token_drop_rate`），与 AttnPool 配合使用，但单独效果未系统评估。

### 7.4 9 患者数据后可开辟的全新方向

> 以下方向在 3 患者下因样本量或患者多样性不足无法进行，在 9 患者下成为可行。

#### 方向 A：患者层级 Stratified 交叉验证 + 域泛化

```
当前（3患者）: Fold1=JFX+LMZ→HYZ, Fold2=HYZ+LMZ→JFX, Fold3=HYZ+JFX→LMZ（3折）
9患者后:     Leave-One-Patient-Out (9折) 或 Group K-Fold (K=3, 每组3患者)

| 新能力 | 说明 |
|------|------|
| **患者层级统计检验** | 9 折结果做配对 t-test 或 Wilcoxon signed-rank test，首次量化"模型 A 是否统计显著优于模型 B" |
| **域泛化（Domain Generalization）** | 将 6 患者作为多源域训练 → 3 患者作为未见目标域测试，评估模型在"完全不同的患者群体"上的泛化。CORAL / DANN / MMD 等域对齐方法可首次应用 |
| **困难患者识别** | 9 折交叉验证可发现"哪些患者始终预测差"→ 反向分析这些患者的临床/病理特征，指导数据收集策略 |

#### 方向 B：临床变量 + 病理图像的浅层多模态融合

> 9 患者预期附带完整临床信息（TNM 分期、分化程度、淋巴结转移、吸烟史等）。当前 3 患者临床变量不足以训练任何模型。

| 融合层级 | 方法 | 预期 |
|------|------|------|
| **输入层** | 临床变量编码为向量 → 与 patch 特征 concat 或交叉注意力 | 简单但可能被图像特征主导 |
| **中间层** | 临床变量作为"患者级条件"调制 patch 特征的 LayerNorm（FiLM / AdaIN） | 让模型"根据分期调整对形态学的解读" |
| **输出层** | 临床变量 → 线性预测通路评分基线 + 图像分支预测残差 | 最可解释：分解"临床已知的"和"形态学额外的" |

> **为什么之前不做**：3 患者临床变量仅 3 个不同的"患者级标签"，训练会直接过拟合到具体患者而非学到临床变量→通路的映射。9 患者提供了 9 个独立数据点用于患者级临床变量建模。

#### 方向 C：通路共表达网络引导的 Structured Prediction

```

当前模型 30 个输出独立预测 → 逐通路 HuberLoss → 忽略了通路间结构化依赖。
```

| 方法 | 说明 | 9患者可行性 |
|------|------|:---:|
| **Graphical Lasso 先验** | 从 9 患者 ssGSEA 真实标签估算 30×30 通路协方差矩阵 Σ → 预测协方差逼近 Σ 作为辅助损失 | 3 患者 30 变量协方差矩阵严重欠定（n=3 < p=30），9 患者勉强可行（n=9, p=30） |
| **通路分组输出头** | 将 30 通路按 Hallmark 分类（代谢/免疫/增殖/信号）分组，每组共享一个中间表示层 | 9 患者每组 3-8 通路，参数量可控 |
| **图神经网络通路解码器** | 用已知通路调控网络作为图结构 → GNN 输出最终预测，让信息沿生物学通路关系传播 | 与样本量弱相关，主要依赖通路网络先验质量 |

#### 方向 D：患者内 Spatial Autocorrelation 的显式建模

> TV Loss 是"弱空间约束"（仅要求相邻 patch 预测平滑）。真正的空间转录组学有更强的空间自相关结构——基因表达在组织微环境尺度（~200-500μm）呈现特征性的空间聚集模式。

| 方法 | 与 TV Loss 的区别 |
|------|------|
| **Moran's I 正则化** | 统计空间自相关指数，将"预测值的空间自相关应与真实值一致"作为正则项 |
| **Graph Fourier Transform 滤波** | 在组织空间图的图谱域对预测值做高低频分解，约束频谱匹配 |
| **Patch 邻域上下文聚合** | 预测某 patch 时，显式 concat 其 k 个空间邻居的 patch 特征（需额外特征提取） |

> 9 患者提供了更多组织架构多样性（不同患者的肿瘤微环境空间组织模式不同），使空间正则化的超参数调优更稳健。

#### 方向 E：Backbone 全面 Benchmark + 多教师融合扩展至 3+ 教师

> 当前仅 UNI + Virchow2 两个教师。9 患者数据量允许引入更多 backbone 并系统对比。

**P-2：两阶段 vs 端到端的架构预研**

```

**P-6：可解释性基础设施——通路→预后归因可视化预案**

> 最终临床部署时，不能仅输出"该患者 2 年生存概率 65%"，必须解释"为什么是 65%——主要是哪些通路异常驱动了这个评估"。

| 组件 | 方法 | 本阶段可做 |
|------|------|:---:|
| **通路级归因** | SHAP / Integrated Gradients 计算每条通路对预后预测的贡献 | ✅ 仿真数据上验证代码 |
| **空间级归因** | 将通路贡献反向映射到原始 H&E 切片上的 patch 位置 → 生成"高风险区域热图" | ✅ 当前通路预测模型已有空间坐标 |
| **病理概念对齐** | 高风险通路 ↔ 已知病理概念（如 EMT↑ → 浸润前沿、Ki67↑ → 增殖活跃区） | ⚠️ 需病理医生协作标注 |

> **本阶段产出**：归因可视化 prototype（用当前 3 患者数据生成 demo 热图），用于向临床合作者展示"最终系统将如何解释其预测"。

## 8. 本周经验教训

1. **TV Loss 是最"划算"的改进**——不改架构、不增参数、不加数据，仅加一个生物学先验约束就+1.7%。验证了"引入领域知识作为正则化"方向正确。对比 AttnPool（+2.1M 参数却 −4.0%），说明约束型改进（加 prior）优于容量型改进（加参数），尤其在小数据跨患者场景
2. **四阶段完成，两极分化明显**：Phase 2 (TV Loss) +1.7% 和 Phase 3 (AttnPool) −4.0% 形成鲜明对比。成功的 Phase 基于"引入外部先验"，失败的 Phase 基于"增加模型自由度"——小数据下自由度增加 ≈ 过拟合入口
3. **融合价值取决于模型间的互补性**——UNI 和 Virchow2 仅 3/30 通路互补，两个同类型 ViT 病理模型的融合增益天然有限。若未来引入不同模态模型（如基因表达预测器），互补性可能更强
4. **Virchow2 frozen 特征不是 UNI 的直接替代品**——独立 PCC 仅 0.3537。当前结论是"frozen → 不行"，而非"Virchow2 本身不行"。LoRA 微调或是更公平的评估方式
5. **AttnPool 的教训**：单患者 +0.0036 的表面提升掩盖了跨患者 −4.0% 的严重退化。任何增加可学习参数的方法必须先验证"是否学到了患者不变的特征"，而非仅看单患者指标

## 附录 D：预判专业提问与回答

> 以下为汇报前预判的 ML 专业听众可能提问。每个问题附简短回答（口头回答 ≤ 3 句），实际组会中可直接引用。

### Q1：TV Loss 只跑了一组超参数（w=0.05），怎么知道这是最优的？

**预判理由**：仅报告单组 TV 参数结果，专业人员会质疑是否充分探索。

**回答要点**：
1. w=0.05 是经验值（参考 GNN 文献中图正则化的典型范围 0.01-0.1），先验证方向有效性再扫参
2. +1.7% 的提升已足够证明方向有效，下周计划系统扫参 {0.01, 0.05, 0.1} × {l1, l2}
3. TV Loss 单次训练仅 ~16 分钟，扫参成本低

---

### Q2：Virchow2 离线特征 PCC 只有 0.35，是不是说明这个 backbone 不适合这个任务？

**预判理由**：Virchow2 独立 PCC 远低于 UNI（0.35 vs 0.42），听众可能直接否定 Virchow2。

**回答要点**：
1. 离线 frozen 特征不代表 backbone 的全部能力——UNI frozen 也只有 0.38（无 AugMix 时），AugMix+TV 才推到 0.42
2. Virchow2 在 3 条免疫通路上优于 UNI，说明其学到的表征维度不同，在"免疫信号"这个子任务上有独特优势
3. 当前结论是"Virchow2 frozen 特征不如 UNI frozen"，而非"Virchow2 本身不行"——LoRA 微调或用 CLS token 替代 mean pooling 可能释放更多潜力

---

### Q3：融合提升仅 +0.0025，是否显著？还是随机波动？

**预判理由**：融合增益太小（+0.6%），专业人员会质疑是否在噪声范围内。

**回答要点**：
1. 上周两个相同配置的独立运行 PCC 差 0.0095，说明单次运行波动约 ±0.005。+0.0025 在此范围内，不能排除随机性
2. 但 Grid Search 逐通路权重分布有明确物理意义（免疫通路偏 Virchow2、代谢通路偏 UNI），验证了互补性假设
3. 融合的主要价值不在绝对 PCC 提升，而在验证了"多教师方向可行"——如果有 3-5 个互补教师模型，融合收益会更显著

---

### Q4：为什么 Virchow2 和 UNI 的互补性这么弱（仅 3/30 通路）？

**预判理由**：两个完全不同架构/预训练数据的 backbone，直觉上应该有更多互补通路。

**回答要点**：
1. 两者都是 ViT-based 病理基础模型，预训练任务相似（自监督），学到的表征可能有大量重叠
2. 互补的 3 条通路恰好是"形态学不可见"类型（免疫/炎症），提示 Virchow2 可能在"弱信号"表征上略有不同
3. 真正强互补需要不同模态的模型（如基因表达模型 + 图像模型），而非两个同类型病理模型

---

### Q5：AttnPool 为什么在单患者有效但跨患者失败？

**预判理由**：这是一个典型的"过拟合到训练分布"案例，专业人员会追问根因。

**回答要点**：
1. AttnPool 学到的 token 重要性权重是患者特异性的——在 HYZ 上学会"关注淋巴细胞区域"，但 JFX/LMZ 的淋巴细胞区域 token 位置不同
2. 单患者场景：训练/验证来自同一患者的数据分布，注意力模式可迁移；跨患者：训练患者（JFX+LMZ）≠ 测试患者（HYZ），注意力模式不可迁移
3. +2.1M 参数在小数据（15K）跨患者场景下是纯粹的正则化负担。解决方案：per-pathway attention 或冻结注意力权重可能改善

---

### Q6：下一步为什么先扫 TV Loss 参数而不是跑全三折？

**预判理由**：上周已在 Fold1 验证多个方向，听众可能期望看到三折结果。

**回答要点**：
1. TV Loss 的 w=0.05 是经验值，如果扫参发现 w=0.01 或 w=0.1 效果更好，三折应该基于最优配置跑，避免重复
2. 扫参成本低（5 组 × 16 分钟 = 1.3 小时），三折成本高（3 folds × 16 分钟 × N 组配置）。先扫参再跑最优配置的三折是更高效率的策略
3. 如果扫参发现 TV Loss 收益在 w=0.05 即峰值，下周可直接跑三折

---

### Q7：这个任务（30 通路回归）PCC=0.42 现在算什么水平？

**预判理由**：多次汇报缺少外部参照系，听众难以判断进度。

**回答要点**：
1. HisToGene 原论文（NeurIPS 2022）在 8 通路上报告 PCC~0.3-0.5（单患者），但 8 通路 vs 30 通路任务难度不同
2. 最新的纯图像方法（如 HistoPrism 2025）在 TCGA 跨癌种上 PCC~0.4-0.6，但数据量远大于我们（万级 WSI vs 3 患者 1.5 万 patch）
3. 我们 3 患者、30 通路、纯图像跨患者 PCC=0.424 在"极小样本 + 多通路"这个设定下属于合理偏上水平。突破 0.45 需要要么引入基因先验（多模态），要么微调 backbone

---

### Q8：AttnPool 为什么从 0.4212 直接跌到 0.4045（−4.0%）？不是"无效"，而是"有害"？

**预判理由**：+2.1M 参数的注意力池化按理应比等权池化更具表示能力，结果反而大幅下降，听众会质疑实现 bug 或训练不充分。

**回答要点**：
1. 非 bug——AttnPool 在单患者上 +0.0036 证明代码实现正确且训练充分。问题出在泛化阶段：AttnPool 学到的 token 重要性模式是训练患者（JFX/LMZ）特异性的，对测试患者（HYZ）不适用
2. 更糟的是，AttnPool 在训练时过度依赖"训练患者中高信息量的 token 位置"，测试时 HYZ 的组织形态分布不同 → 注意力分配错误 → 预测偏差远大于等权池化（等权至少不犯错）
3. 类比：AttnPool 相当于"让学生专攻模拟题的重点题型"，但如果真考题的重点完全不同，专攻反而比"全面复习"（等权池化）成绩更差。这个教训说明在小数据跨分布场景下，**简单方法（均值）可能比可学习方法（注意力）更鲁棒**

---

### Q9：OmiCLIP 单患者 PCC=0.55 很强，为什么跨患者直接崩到 0.19？CoCa 架构本身有问题吗？

**预判理由**：单患者 vs 跨患者的 −64.4% 衰减是所有 backbone 中最严重的，专业人员会追问"是不是训练配置没调好"而非架构问题。

**回答要点**：
1. 训练配置与 UNI 系列相同（LightweightTokenEncoder + HuberLoss + AdamW），且通过 token_drop_rate=0.3 + weight_decay=5e-3 的强正则化专门优化过。问题不在训练配置，在特征本身
2. 根因是 CoCa 的预训练/微调范式与纯病理自监督（DINOv2）的本质差异：CoCa 用自然图像+文本预训练 → 病理-基因配对 fine-tune，学到的特征是"区分不同样本"的判别特征；UNI/DINOv2 从头在病理数据上自监督，学到的是"组织形态学语义"的连续特征。回归任务需要后者
3. 255 token 也加剧了问题——token 量是 UNI(65) 的 3.9 倍，在小数据下严重过参数化。即使 Mean Pooling，255 个高噪声 token 的均值方差更大。架构本身优秀，但"自然图像 CoCa → 食管癌回归"的迁移路径不对

---

### Q10：OmiCLIP 如果加 LoRA 微调 backbone 会不会好？

**预判理由**：OmiCLIP 失败的根因被归为"特征迁移差"，听众自然会问微调是否能解决。

**回答要点**：
1. 理论上可能——LoRA 微调可以让 CoCa 的视觉编码器适应病理形态学，缩小"自然图像 → 组织切片"的域差距
2. 但与其他方向竞争算力：Virchow2（frozen 0.35 + LoRA 预期 0.45+）和 UNI（frozen 0.42 + LoRA 预期 0.50+）的 LoRA 优先度都高于 OmiCLIP
3. OmiCLIP 的独特价值在于其基因-图像多模态预训练——如果微调成功，可能在免疫/炎症等"形态学不直接可见"的通路上实现突破。但这是高风险高回报方向，当前阶段优先确定性更高的 UNI/Virchow2 改进

---

### Q11：Phase 4 是否等同于 Phase 1 + Phase 2 的简单叠加？为什么不加上 Phase 3 一起？

**预判理由**：Phase 4 只是将 P2 训练出的 TV 模型 + P1 的融合脚本组合，零额外训练，听众可能质疑这是否算独立 Phase。

**回答要点**：
1. 是的——Phase 4 确实是 P2（TV 模型输出）+ P1（融合方法）的组合。但这不是"偷懒"，而是刻意设计：四阶段执行规划明确要求"先独立验证 → 再叠加最优组合"，避免盲目联合训练
2. Phase 3（AttnPool）在跨患者上已明确拒绝（−4.0%），将其纳入联合训练会破坏 TV Loss 的收益，所以 Phase 4 仅叠加验证有效的方法（TV + 融合）
3. 如果 AttnPool 在跨患者上有效（比如 +0.003），Phase 4 会设计为 TV + AttnPool 联合训练 → 再融合 Virchow2。但由于 P3 被拒绝，P4 自然简化为 P1+P2 叠加。这恰恰体现了"先独立消融再组合"的策略正确性
