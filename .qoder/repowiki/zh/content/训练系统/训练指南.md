## 关键 CLI 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--patient` | HYZ15040 | 患者名称（HYZ15040 / JFX0729 / LMZ12939） |
| `--backbone` | uni_tokens | 特征骨干（uni_tokens / uni_tokens_aug / omiclip / virchow2） |
| `--num_epochs` | 150 | 最大训练轮数 |
| `--batch_size` | 256 | 批大小（UNI token 模式可用 256，ViT 端到端用 64） |
| `--lr` | 1e-3 | 初始学习率 |
| `--early_stop_patience` | 10 | 早停耐心值（连续 N 轮 val_loss 不降则停止） |
| `--dropout` | 0.2 | Dropout 比率 |
| `--hidden_dim` | 256 | 回归头隐藏层维度 |
| `--resume` | — | 断点续训检查点路径 |
| `--rebuild_cache` | False | 是否重建特征缓存 |
| `--amp` | True | 启用混合精度（仅 CUDA） |
| `--fold` | — | 指定折号（跨患者脚本，1/2/3） |

## 训练输出目录结构

```
{model_dir}/checkpoints/results_vis/{dataset}_{timestamp}/
├── model_params.txt            # 模型超参数与训练配置
├── training_history_{dataset}.csv  # 每轮训练/验证指标记录
├── predictions.csv             # 预测结果（true_xxx / pred_xxx 列格式）
├── training_curves.png         # 训练曲线（Loss/MAE/R²/PCC 四合一）
├── pcc_barplot.png             # 逐通路 PCC 柱状图（含均值线）
└── per_pathway_pcc.csv         # 30 条通路的逐一 PCC 值
```

### 文件说明

- **model_params.txt**：记录完整训练配置，包括模型架构参数、学习率、批大小、早停配置等
- **training_history.csv**：每轮 epoch 的 train_loss、train_mae、train_r2、train_pcc、val_loss、val_mae、val_r2、val_pcc、lr
- **predictions.csv**：格式为 `patch_id, true_tls, pred_tls, true_tgfb, pred_tgfb, ...`（30 条通路）
- **training_curves.png**：上下两行分别展示 Loss/MAE 和 R²/PCC 随 epoch 变化曲线
- **pcc_barplot.png**：横轴 30 条通路名，纵轴 PCC 值，柱子红绿渐变色，标注均值水平线
- **per_pathway_pcc.csv**：每行一条通路，包含 PCC、MSE、MAE、R²

## 训练控制

### 暂停训练

在项目根目录创建空文件 `PAUSE_TRAINING`：

```bash
touch PAUSE_TRAINING
```

系统在当前 epoch 结束后自动保存恢复检查点并退出。删除该文件后重新运行可继续训练。

### 断点续训

```bash
PYTHONIOENCODING=utf-8 "C:\Program Files\Python313\python.exe" train_histogene_uni_tokens.py \
    --resume histogene/checkpoints/HYZ15040_UNI_tokens/best_model.pth
```

检查点包含：模型权重、优化器状态、调度器状态、训练历史、scaler 状态。

### 训练通知

训练完成或异常中断时，通过 plyer 库发送 Windows Toast 桌面弹窗通知。若 plyer 未安装，自动降级为纯文本控制台输出。

## 评估指标

| 指标 | 全称 | 计算方式 | 用途 |
|------|------|----------|------|
| PCC | Pearson Correlation Coefficient | 逐通路计算后取均值 | **主要指标**，衡量线性相关性 |
| MAE | Mean Absolute Error | `mean(abs(pred - true))` | 衡量预测绝对误差 |
| R² | Coefficient of Determination | `1 - SS_res / SS_tot` | 衡量方差解释比例 |

注意事项：

- 最佳 epoch 以 **val_loss 最小** 为准（非 val_pcc 最大）
- 当标签或预测为常数时 R² 可能为 NaN，系统已做保护处理
- PCC=0 表示标签存在零方差
- `evaluate()` 必须同时输出 PCC、MAE、R² 三个指标

## 训练监控节约原则

为避免资源浪费，训练监控遵循以下原则：

1. **按需查看**：训练历史自动保存到 CSV，无需实时盯屏。训练结束后通过 `visualize_model_comparison.py` 或 `visualize_cross_patient.py` 批量生成可视化报告
2. **控制台输出**：每轮仅输出一行关键指标（epoch、loss、MAE、R²、PCC、lr），避免日志洪流
3. **历史 CSV**：每轮写入 training_history CSV，支持随时中断后查看已有训练曲线
4. **通知机制**：训练完成或异常时桌面弹窗提醒，无需人工轮询
5. **避免重复**：训练输出目录带时间戳，不覆盖历史结果，每次训练独立保存
6. **`generate_full_report` 只调用一次**：避免重复创建时间戳目录

## 训练常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| `UnicodeEncodeError` | Windows GBK 编码 | 命令前加 `PYTHONIOENCODING=utf-8` |
| 特征缓存缺失 | 未执行特征提取 | 先运行对应 `extract_*.py` 脚本 |
| 三层交集样本少 | patches ∩ CSV ∩ cache 交集中样本少 | 使用二层交集（cache ∩ labels），样本更多 |
| `dataset_name` 默认值错误 | 从 `--patient` 自动推导 | 不要硬编码 `dataset_name` |
| val_loss 不下降 | 学习率过大或过小 | 调整 `--lr`，默认 1e-3 适用于 UNI token |
| CUDA out of memory | batch_size 过大 | 降低 `--batch_size`（UNI token 可用 256，ViT 用 64） |
