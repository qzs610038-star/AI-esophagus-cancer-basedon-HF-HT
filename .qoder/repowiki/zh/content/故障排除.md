# 故障排除

## 一、环境与编码问题

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| 训练脚本报 `UnicodeEncodeError` | Windows 中文控制台默认 GBK，脚本中 Unicode 字符无法输出 | 所有训练命令加 `PYTHONIOENCODING=utf-8` 前缀 |
| `ImportError` 或 `ModuleNotFoundError` | 依赖未安装或 Python 环境错误 | HisToGene 系列用 `C:\Program Files\Python313\python.exe`；EGN-v2/GAT(需PyG) 用 `D:\conda_envs\pfmval_py310\python.exe` |
| CUDA 不可用或版本不匹配 | CUDA 路径变更或 PyTorch 与 CUDA 版本不对应 | 确认 CUDA 11.8 在 D 盘，PyTorch 2.6.0+cu118；C 盘原路径已是 junction 到 D 盘 |

## 二、数据路径问题

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| `FileNotFoundError` 读取 patch 或 CSV | 路径硬编码指向不存在的目录 | 使用 `config_utils.get_patient_paths()` 获取路径，禁止硬编码绝对路径 |
| 标签与图片不匹配，Dataset 报错 | CSV 中 patch_id 与文件名的 stem 不一致 | 检查 `tools/check_csv_patient.py` 验证一致性 |
| 特征缓存缺失 | 未先运行特征提取脚本 | 先执行 `extract_uni_tokens.py` 或对应 backbone 的特征提取脚本 |
| 训练集/验证集为空 | 三层交集(patches∩CSV∩cache)样本过少 | 优先使用二层交集(cache∩labels)，样本更多、效果更好 |

## 三、训练问题

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| CUDA out of memory (OOM) | batch_size 过大 | 降低 batch_size（建议从 128 降到 64 或 32） |
| 验证集指标全 NaN | 预测值存在常数列或极端异常值 | 检查预测输出是否退化；用 `data_distribution_analysis.py` 分析分布 |
| 训练不收敛或 loss 震荡 | 学习率不当 | 调整学习率（HisToGene 系列默认 1e-4，EGN 系列 1e-3） |
| `dataset_name` 推导错误 | 脚本中 `dataset_name` 硬编码了默认值 | `dataset_name` 应从 `--patient` 参数自动推导，不硬编码 |
| 训练中断后无法恢复 | checkpoint 未保存或路径错误 | 确认 `get_output_dir()` 返回的路径可写；训练脚本支持从 checkpoint 恢复 |

## 四、推理问题

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| predictions.csv 列名不匹配 | 格式应为 `true_xxx`/`pred_xxx` | 确认推理脚本使用了正确的列名格式 |
| per_pathway_pcc.csv 内容为空 | evaluate() 未正确输出所有指标 | 确认 `evaluate()` 同时输出 PCC、MAE、R² |
| 推理结果目录混乱 | `generate_full_report()` 被多次调用 | 该函数全局只调用一次，避免重复创建时间戳目录 |
| 中文路径写入失败 | PyTorch 2.0.1 无法处理中文路径写文件 | 使用 `io.BytesIO` + Python `open()` 绕过 |

## 五、OmiCLIP 特定问题

| 症状 | 原因 | 解决方案 |
|------|------|----------|
| OmiCLIP 权重加载报 `total_mem` 错误 | 属性名是 `total_memory` 而非 `total_mem` | 检查并修正代码中的属性名拼写 |
| HuggingFace 模型下载失败 | 国内网络限制 | 配置镜像：`export HF_ENDPOINT=https://hf-mirror.com` |
| `dataset_uni_tokens.py` 维度断言失败 | 该文件有 1536 维硬编码断言，不适用于 OmiCLIP | 自定义 Dataset 类，不依赖该文件的硬编码断言 |

## 七、PowerShell 注意事项

在 PowerShell 中运行多条命令时，**不支持 `&&`**，使用 `;` 分隔：

```powershell
# 正确
PYTHONIOENCODING=utf-8 python train.py --patient HYZ15040; echo "Done"

# 错误
PYTHONIOENCODING=utf-8 python train.py --patient HYZ15040 && echo "Done"
```

## 八、路径拼写提醒

原始数据目录 `patch_noov_spilt` 的拼写包含 typo（"spilt" 而非 "split"），**不可修正**。所有引用该目录的代码必须保持原始拼写，否则会导致路径找不到。
