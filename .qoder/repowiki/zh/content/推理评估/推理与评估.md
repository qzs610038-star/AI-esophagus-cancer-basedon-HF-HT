## predictions.csv 格式规范

推理阶段生成的预测结果文件，列名严格遵循以下格式：

| 列类型 | 格式 | 示例 |
|--------|------|------|
| 真实值 | `true_通路名` | `true_APOPTOSIS`, `true_PI3K_AKT_MTOR` |
| 预测值 | `pred_通路名` | `pred_APOPTOSIS`, `pred_PI3K_AKT_MTOR` |

- 每行对应一个 patch 样本
- 通路名与 ssGSEA 评分的 30 条基因通路一一对应
- 列数 = 30 条通路 x 2（true + pred）= 60 列（可能含额外的标识列）

## per_pathway_pcc.csv 格式规范

逐通路评估指标文件，按 PCC 降序排列：

| 列名 | 说明 |
|------|------|
| `pathway` | 通路名称 |
| `pcc` | 皮尔逊相关系数（主要指标） |
| `r2` | 决定系数 |
| `mae` | 平均绝对误差 |
| `rank` | 排名（1 = PCC 最高） |

## 评估指标定义

### PCC（Pearson Correlation Coefficient，主要指标）

- 衡量预测值与真实值的线性相关程度，范围 [-1, 1]
- 越接近 1 表示正线性关系越强
- 当前最佳结果：HisToGene-UNI Token 跨患者三折平均 PCC = 0.3812

### MAE（Mean Absolute Error）

- 预测值与真实值绝对差的均值，量纲与原值一致
- 值越小越好，对异常值相对稳健

### R²（Coefficient of Determination）

- 反映模型解释方差的比例，范围 (-inf, 1]
- 越接近 1 越好；负值表示模型不如简单均值预测

### evaluate() 函数规范

所有训练脚本的 `evaluate()` 函数必须**同时输出** PCC、MAE、R² 三项指标，不得遗漏任何一项。典型调用方式：

```python
val_pcc, val_mae, val_r2 = evaluate(model, val_loader, criterion, device)
```

模型选择依据：**val_loss 最小的 epoch**（非 val_pcc 最大）。

## 可视化输出文件说明

| 文件 | 内容 | 生成脚本 |
|------|------|----------|
| `training_curves.png` | 训练/验证 loss 随 epoch 变化曲线 | 各模型的 visualize 脚本 |
| `pcc_barplot.png` | 30 条通路 PCC 降序柱状图（含宏平均线） | visualize_all_models.py |
| `training_history_{dataset}.csv` | 逐 epoch 记录 train_loss / val_loss / val_pcc / val_mae / val_r2 | 训练脚本自动保存 |

可视化输出保存至：`{model_dir}/checkpoints/results_vis/{dataset}_{timestamp}/`，每个训练批次创建独立时间戳目录，不覆盖历史结果。

## generate_full_report 使用规则

`generate_full_report()` 定义在 `visualize_results.py`，**全局只调用一次**。每次调用会创建带时间戳的输出目录，多次调用会产生多个冗余目录。

调用位置：训练脚本末尾，训练完成后自动调用：

```python
from visualize_results import generate_full_report
generate_full_report(
    model_name="HisToGene-UNI",
    true_csv=...,
    pred_csv=...,
    output_dir=...
)
```

## 训练历史记录

`training_history_{dataset}.csv` 包含所有 epoch 的完整记录：

| 列名 | 说明 |
|------|------|
| `epoch` | 轮次编号 |
| `train_loss` | 训练集损失 |
| `val_loss` | 验证集损失 |
| `val_pcc` | 验证集平均 PCC |
| `val_mae` | 验证集平均 MAE |
| `val_r2` | 验证集平均 R² |

文件保存于 checkpoint 或 results 目录下，可用于事后分析和绘图中最优 epoch 的定位。
