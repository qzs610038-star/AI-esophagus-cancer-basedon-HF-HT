# 环境与配置

## Python 环境

项目使用三个独立 Python 环境，对应不同依赖需求：

| 环境 | Python 路径 | 用途 | 关键依赖 |
|------|-----------|------|---------|
| **HisToGene 系列** | `C:\Program Files\Python313\python.exe` | HisToGene-UNI Token、AugMix、OmiCLIP 训练、Virchow2 训练 | Python 3.13, PyTorch 2.6.0+cu118, timm, huggingface_hub |
| **EGN-v2 / GAT** | `D:\conda_envs\pfmval_py310\python.exe` | EGN-v2+UNI、HisToGene-UNI+GAT 训练（需 PyG） | Python 3.10, PyTorch 2.1.0+cu118, torch_geometric |
| **OmiCLIP 特征提取** | `D:\conda_envs\loki_env\python.exe` | OmiCLIP backbone 特征提取（需要 open_clip） | Python 3.9, open_clip 2.26.1 |

**注意**：特征提取与训练可在不同环境进行。例如 OmiCLIP 特征用 `loki_env` 提取后，训练仍使用 Python313 环境。

## Windows GBK 编码问题

Windows 中文控制台默认编码为 GBK，训练脚本中的 Unicode 字符（如进度条、特殊符号）会触发 `UnicodeEncodeError`。

**解决方案**：所有训练命令必须加 `PYTHONIOENCODING=utf-8` 前缀。

### PowerShell 命令模板

```powershell
# 设置编码（每个新会话只需执行一次）
$env:PYTHONIOENCODING = "utf-8"

# 特征提取
& "C:\Program Files\Python313\python.exe" extract_uni_tokens.py --patient HYZ15040

# 单患者训练（主力）
& "C:\Program Files\Python313\python.exe" train_histogene_uni_tokens_augmix.py --patient HYZ15040 --epochs 50

# 跨患者训练
& "C:\Program Files\Python313\python.exe" train_cross_patient_histogene_uni.py --fold 1 --epochs 100

# OmiCLIP 特征提取（使用 loki_env）
& "D:\conda_envs\loki_env\python.exe" extract_omiclip_features.py --patient HYZ15040

# EGN-v2 + UNI 训练（使用 pfmval_py310 环境）
& "D:\conda_envs\pfmval_py310\python.exe" train_egnv2_uni.py --patient HYZ15040 --epochs 100
```

### Bash 命令模板（Linux 服务器）

```bash
# 编码问题仅影响 Windows，Linux 无需设置 PYTHONIOENCODING
python train_histogene_uni_tokens_augmix.py --patient HYZ15040 --epochs 150

# 跨患者训练
python train_cross_patient_histogene_uni.py --fold 1 --epochs 150
```

## PowerShell vs Bash 注意事项

| 特性 | PowerShell | Bash (Linux/macOS/WSL) |
|------|-----------|------------------------|
| 编码设置 | `$env:PYTHONIOENCODING = "utf-8"` | 不需要 |
| 命令连接 | 用 `;` 分隔（不支持 `&&`） | 用 `&&` 或 `;` |
| Python 路径 | 必须用引号包裹 `& "C:\Program Files\..."` | 直接路径或 `python3` |
| 环境变量 | `$env:VAR = "val"` | `export VAR="val"` |

## config.yaml 配置说明

`config.yaml` 是项目唯一配置入口，服务器迁移只需修改此文件。

### 关键配置段

```yaml
# HuggingFace 配置
huggingface:
  token: ""              # HF Token（留空从 HF_TOKEN 环境变量读取）
  local_only: false      # 无网络时设为 true

# 训练设备
training:
  device: "auto"         # auto / cuda / cpu

# 路径根目录（服务器迁移时取消注释并填写实际路径）
paths:
  # patch_base: "/data/server/HE_patches"
  # ssgsea_base: "/data/server/ssGSEA_scores"
  caches:
    # uni_tokens: "/data/caches/uni_tokens"
    # omiclip: "/data/caches/omiclip"
  resources:
    # omiclip_checkpoint: "/data/models/omiclip/checkpoint.pt"

# 患者级路径覆盖（绝对路径覆盖，最高优先级）
patients:
  HYZ15040:
    # patches_dir: "/data/HE_images/HYZ15040"
    # labels_path: "/data/ssGSEA/HYZ15040_scores.csv"
```

### 配置验证

```bash
python config_utils.py
```

该命令输出所有配置项的解析结果与路径存在性检查，用于验证迁移配置是否正确。
