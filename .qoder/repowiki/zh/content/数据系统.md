# 数据系统

## 数据目录结构

项目涉及三种数据：原始 patch 图像、预提取特征缓存、基因通路标签。

### Patch 图像

```
data_new_3ST/patch_noov_spilt/
├── HYZ15040_noov_split/   (2,655 patches)
│   ├── train_patches/      (.png)
│   └── val_patches/        (.png)
├── JFX0729_noov_split/    (7,838 patches)
│   ├── train_patches/
│   └── val_patches/
└── LMZ12939_noov_split/   (7,513 patches)
    ├── train_patches/
    └── val_patches/
```

### 特征缓存（按 backbone 区分）

| Backbone | 缓存目录 | 文件格式 | 特征维度 |
|----------|---------|----------|---------|
| uni_tokens | `uni2h_cache_tokens/{patient}/train/` 和 `val/` | `.pt` | [N, 265, 1536] |
| uni_tokens_aug | `uni2h_cache_tokens_aug/{patient}/train/` 和 `val/` | `.pt` | [N, 265, 1536] |
| uni_cls | `uni2h_cache/{patient}/train/` 和 `val/` | `.pt` | [N, 1536] |
| omiclip | `omiclip_cache/{patient}/train/` 和 `val/` | `.pt` | [N, 255, 768] |
| virchow2 | `virchow2_cache_tokens/{patient}/train/` 和 `val/` | `.pt` | — |

### 标签文件

```
data_new_3ST/ssGSEA_zscore/
├── HYZ15040_ssGSEA_zscore.csv
├── JFX0729_ssGSEA_zscore.csv
└── LMZ12939_ssGSEA_zscore.csv
```

每行一个 patch，列包括 `patch_id` + 30 条通路的 Z-score 标准化 ssGSEA 评分。

## config_utils.py 关键 API 速查表

| 函数 | 用途 | 返回值示例 |
|------|------|-----------|
| `get_project_root()` | 获取项目根目录绝对路径 | `/data/PFMval_new` |
| `load_config(path?)` | 加载 config.yaml（优先级：参数 > 环境变量 > 自动搜索） | `dict` |
| `get_patient_paths(patient, backbone, config?)` | 获取患者完整路径字典 | `{train_patches, val_patches, labels_csv, token_cache_train, token_cache_val, ...}` |
| `get_all_patient_configs(patients?, backbone?, config?)` | 批量获取多患者路径 | `{HYZ15040: {...}, JFX0729: {...}, ...}` |
| `get_fold_config(fold, config?)` | 获取三折交叉验证 train/test 配置 | `{train: [JFX, LMZ], test: HYZ}` |
| `get_output_dir(subdir?, config?)` | 获取输出目录路径 | `/data/PFMval_new/checkpoints/xxx` |
| `get_omiclip_checkpoint_path(config?)` | OmiCLIP 权重文件路径 | `/data/PFMval_new/pretrained_omiclip/checkpoint.pt` |
| `resolve_path(path_str, project_root?)` | 相对路径转绝对路径 | 规范化绝对路径 |
| `get_device(config?)` | 获取 torch.device（auto/cuda/cpu） | `torch.device("cuda")` |
| `get_hf_config(config?)` | HuggingFace token 和 local_only 配置 | `{token, local_only}` |

## config.yaml 结构

```yaml
data:               # 受保护文件使用的默认数据路径
paths:              # 新脚本的路径根目录配置（服务器迁移时修改此项）
  patch_base:       # 覆盖默认 patch 根目录（可选）
  ssgsea_base:      # 覆盖默认标签根目录（可选）
  caches:           # 各 backbone 缓存目录覆盖
    uni_tokens:
    uni_tokens_aug:
    omiclip:
    virchow2:
  outputs:          # checkpoints_root / results_vis（可选）
patients:           # 每位患者的子路径配置，支持绝对路径覆盖
  HYZ15040:
    patches_subdir:
    labels_csv:
    patches_dir:    # 绝对路径覆盖（可选，服务器用）
    labels_path:    # 绝对路径覆盖（可选，服务器用）
cross_validation:   # 三折交叉验证配置
  folds:
    1: {train: [JFX0729, LMZ12939], test: HYZ15040}
    2: {train: [HYZ15040, LMZ12939], test: JFX0729}
    3: {train: [HYZ15040, JFX0729], test: LMZ12939}
```

## 三折交叉验证

| Fold | 训练集 | 测试集 |
|------|--------|--------|
| 1 | JFX0729 + LMZ12939 | HYZ15040 |
| 2 | HYZ15040 + LMZ12939 | JFX0729 |
| 3 | HYZ15040 + JFX0729 | LMZ12939 |

跨患者训练时，训练集的两个患者数据合并为一个 Dataset 使用。

## 数据流

```
                        ┌────── config.yaml ──────┐
                        │ paths / patients /       │
                        │ cross_validation         │
                        └───────────┬─────────────┘
                                    │ 读取配置
                                    ▼
                        ┌──────────────────────────┐
                        │    config_utils.py       │
                        │  get_patient_paths()     │
                        │  返回: {train_patches,   │
                        │   val_patches, labels_csv│
                        │   token_cache_train, ...}│
                        └───────────┬─────────────┘
                                    │ 绝对路径字典
                                    ▼
┌──────────────────────────────────────────────────────────────────┐
│  ① 特征提取 (extract_*.py)   输入: .png ──► 输出: .pt 缓存        │
│                                                                   │
│  ┌── 主分支: UNI2-h Token ────────────────────────────────────┐  │
│  │ patch .png → UNI2-h (冻结) → .pt [265, 1536]               │  │
│  │ extract_uni_tokens.py → uni2h_cache_tokens/{p}/{t,v}/*.pt   │  │
│  ├── 增强分支: UNI2-h + AugMix ───────────────────────────────┤  │
│  │ patch .png → AugMix随机增强 → UNI2-h → .pt [265, 1536]     │  │
│  │ extract_uni_tokens_augmented.py → uni2h_cache_tokens_aug/   │  │
│  │ 样本数 = 原图数 × aug_factor，提升泛化能力                   │  │
│  ├── OmiCLIP 分支 ────────────────────────────────────────────┤  │
│  │ patch .png → coca_ViT-L-14 (冻结) → .pt [255, 768]         │  │
│  │ extract_omiclip_features.py → omiclip_cache/                │  │
│  └── Virchow2 分支 ───────────────────────────────────────────┤  │
│     patch .png → Virchow2 (冻结) → .pt → virchow2_cache_tokens/ │  │
│                                                                   │
│  所有 extract 脚本通过 get_patient_paths(patient, backbone)       │
│  自动获取输入/输出路径，无需硬编码                                  │
└────────────────────────────┬─────────────────────────────────────┘
                             │ .pt 缓存文件 (每 patch 一个)
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│  ② 样本匹配 — 两层交集 vs 三层交集                                │
│                                                                   │
│  两层交集 (默认, cache ∩ labels):                                  │
│  ┌─────────────────┐         ┌──────────────────┐                │
│  │ 缓存 .pt 文件列表│         │ 标签 CSV (patch_id│                │
│  │ {coord}.pt × N  │         │  + 30通路 Z-score)│                │
│  └────────┬────────┘         └────────┬─────────┘                │
│           │      按坐标/ID 取交集      │                           │
│           └────────────┬──────────────┘                           │
│                        ▼                                          │
│                 有效样本索引列表                                    │
│                                                                   │
│  三层交集 (patches ∩ CSV ∩ cache):                                 │
│  额外要求 .png 图像文件必须存在 → 样本数更少但更严格                │
│  用于精确对比消融实验                                               │
└────────────────────────────┬─────────────────────────────────────┘
                             │ 有效样本索引列表
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│  ③ Dataset 构建 (dataset_*.py)   输入: .pt列表+CSV   输出: Dataset│
│                                                                   │
│  dataset_uni_tokens.py / dataset_uni_tokens_augmix.py:            │
│    __init__(): 遍历缓存目录, 与标签 CSV 做交集匹配, 构建索引列表    │
│    __getitem__(idx):                                               │
│      feature = torch.load(cache_path)    # [265, 1536]            │
│      label   = csv.loc[patch_id, 30通路] # [30]                   │
│      coord   = parse_coord(patch_id)     # [2] (X, Y)             │
│      return feature, coord, label                                  │
│    __len__(): return len(valid_indices)                            │
└────────────────────────────┬─────────────────────────────────────┘
                             │ Dataset 对象
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│  ④ DataLoader + 训练循环 (train_*.py)                             │
│                                                                   │
│  DataLoader(dataset, batch_size=N, shuffle=True)                  │
│    ┌──────────────────────────────────────────────────┐          │
│    │ batch: {features [B,265,1536], coords [B,2],     │          │
│    │         labels [B,30]}                            │          │
│    └────────────────────┬─────────────────────────────┘          │
│                         ▼                                          │
│    for epoch in range(max_epochs):                                 │
│      for batch in train_loader:                                    │
│        pred = model(batch.features, batch.coords)  # [B,30]       │
│        loss = MSELoss(pred, batch.labels)                          │
│        loss.backward() → grad_clip(1.0) → optimizer.step()        │
│                                                                   │
│    验证阶段: val_loss → ReduceLROnPlateau(patience=5)              │
│    早停: patience=10, 监控 val_loss (非 val_pcc)                   │
│    保存: best_model.pth (val_loss 最小的 epoch)                    │
│                                                                   │
│  跨患者模式: get_fold_config(fold) → 2患者 DataLoader 合并训练     │
│  单患者模式: 同患者 train/val 子目录分别构建 DataLoader             │
└────────────────────────────┬─────────────────────────────────────┘
                             │ best_model.pth
                             ▼
┌──────────────────────────────────────────────────────────────────┐
│  ⑤ 评估与输出                                                     │
│                                                                   │
│  加载 best_model.pth → 测试集推理                                  │
│  输出 predictions.csv (列名格式: true_xxx / pred_xxx)             │
│  → visualize_*.py 生成: PCC热图、散点图、通路排序图                │
│  → 保存至 results_vis/{dataset}_{timestamp}/                       │
└──────────────────────────────────────────────────────────────────┘
```

## 路径解析优先级

1. `config.yaml` → `patients.{name}.patches_dir` / `labels_path`（绝对路径覆盖，最高优先级）
2. `config.yaml` → `paths.patch_base` / `paths.ssgsea_base` + 患者 subdir（服务器迁移配置）
3. 项目根目录下的默认路径（本地零配置运行，fallback）

迁移到服务器只需取消注释 `config.yaml` 中 `paths:` 节的对应路径并填写实际值，无需修改任何 Python 代码。
