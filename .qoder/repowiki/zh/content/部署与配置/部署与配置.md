## 简介
本指南面向生产环境部署与配置，围绕 PFMval 项目提供从环境准备、数据与模型管理、分布式与推理配置、压缩与加速、容器化、监控日志、模型更新维护到安全与访问控制的全流程说明。文档基于仓库内现有脚本与实现进行归纳与扩展，特别强调新增的配置管理系统、通知系统、可视化模块，以及新增的EGN-v1/v2部署方案和OpenMidnight集成配置，帮助读者在真实环境中稳定、高效地运行训练与推理任务。

## 项目结构
项目采用"数据预处理 + 特征提取 + 训练/推理 + 配置管理 + 通知 + 可视化"的分层结构，核心模块如下：
- 数据预处理：split.py（空间无重叠划分）、zscore.py（Z-score标准化）
- 特征提取与缓存：uni2h_utils.py（加载UNI2-h、特征提取、Dataset、回归头、指标计算）
- 训练与推理：uni2h/train.py、uni2h/infer.py、histogene/train.py、histogene/infer.py
- EGN模型系列：egnv1/（ViT-Large + GCN + Exemplar）、egnv2/（ResNet-50 + GraphSAGE + Exemplar）
- OpenMidnight集成：openmidnight/（DINOv2基础模型 + MLP回归头）
- 配置管理：config.yaml（集中配置）、config_utils.py（配置工具）
- 通知系统：notify_utils.py（训练完成/中断通知）
- 可视化模块：visualize_results.py（通用可视化脚本）
- 可选适配方案：histogene/（基于ViT的端到端方案，包含model.py、dataset.py、train.py、infer.py）

## 核心组件
- 数据预处理管线
  - split.py：基于文件名坐标与距离阈值进行空间无重叠划分，输出 train_patches/ 与 val_patches/
  - zscore.py：对最后8列基因集评分执行Z-score标准化，统一量纲
- 特征提取与缓存
  - uni2h_utils.py：加载UNI2-h骨干（冻结），批量提取并缓存特征（.pt），提供Dataset、回归头、指标计算
- 训练与推理
  - uni2h/train.py：特征提取与缓存、构建Dataset、训练循环、早停、保存最佳模型
  - uni2h/infer.py：加载checkpoint、特征提取与缓存、推理、逐目标与宏平均指标
  - histogene/train.py：端到端训练，集成配置系统和通知系统
  - histogene/infer.py：端到端推理，保存逐通路指标与预测CSV
  - egnv1/train.py：EGN-v1完整训练流程，支持多患者联合训练
  - egnv2/train.py：EGN-v2完整训练流程，支持多患者联合训练
  - openmidnight/train_openmidnight.py：OpenMidnight特征提取 + MLP回归训练
- 配置管理系统
  - config.yaml：集中配置文件，包含数据路径、HuggingFace配置、训练配置
  - config_utils.py：配置工具，提供配置加载、路径解析、设备选择等功能
- 通知系统
  - notify_utils.py：训练完成/中断通知，支持系统通知、日志文件标记、暂停信号检测
- 可视化模块
  - visualize_results.py：通用可视化脚本，生成训练曲线、参数面板、指标表格、PCC柱状图等

### 数据预处理组件
- split.py
  - 功能：按空间距离约束划分训练/验证集，避免空间泄漏
  - 关键参数：距离阈值、验证集比例、随机种子
  - 输出：train_patches/ 与 val_patches/ 目录
- zscore.py
  - 功能：对最后N列（默认8列）执行Z-score标准化，统一量纲
  - 输出：*_ssGSEA_scores_zscore.csv

- uni2h_utils.py

### 训练与推理组件
- uni2h/train.py
  - 流程：加载骨干→提取并缓存特征→构建Dataset/Dataloader→创建回归头→训练循环→早停→保存最佳模型
  - 关键超参：batch_size、learning_rate、hidden_dim、dropout、patience
- uni2h/infer.py
  - 流程：加载checkpoint→重建模型→特征提取与缓存→推理→逐目标与宏平均指标→输出CSV

- histogene/infer.py
  - 流程：端到端推理，保存逐通路指标与预测CSV
- egnv1/train.py（新增）
  - 流程：ViT特征提取→图构建(KNN/spatial/hybrid)→代表库构建→GCN训练→多患者联合训练支持
  - 关键参数：ViT-Large backbone、GCN图卷积、Exemplar融合机制
- egnv2/train.py（新增）
  - 流程：ResNet特征提取→空间图构建→代表库构建→GraphSAGE训练→多患者联合训练支持
  - 关键参数：ResNet-50 backbone、GraphSAGE图卷积、LayerNorm归一化
- openmidnight/train_openmidnight.py（新增）
  - 流程：预提取特征→MLP回归头训练→学习率调度→最佳模型保存
  - 关键参数：1536维输入特征、512/256隐藏层、MSE损失、AdamW优化器

## EGN-v1部署方案

### EGN-v1模型简介
EGN-v1（Exemplar Guided Network）采用ViT-Large（86M参数）作为特征提取器，结合GCN图卷积网络和Exemplar学习机制。该模型在2023年WACV会议发表，是EGN系列的首个版本。

### 核心架构特点
- **特征提取器**：ViT-Large（patch32, depth=8, heads=16, dim=1024）
- **图卷积层**：GCN（Graph Convolutional Network）
- **图构建方式**：KNN图（基于特征相似性）
- **Exemplar机制**：Exemplar Bridging块，嵌入ViT中间层
- **隐藏维度**：1024（匹配ViT输出）

### 部署建议与参数配置

#### 硬件配置要求
- **GPU**：建议32GB显存（如RTX 4090/3090），ViT-Large需要较大显存
- **内存**：建议64GB以上，支持大batch训练
- **存储**：特征缓存约1-2GB，取决于数据规模

#### 关键参数建议
| 参数 | 建议值 | 说明 |
|------|--------|------|
| backbone | ViT-Large | 官方参数：dim=1024, depth=8, heads=16 |
| graph_layers | 2 | 保持与v2一致 |
| hidden_dim | 1024 | 匹配ViT输出维度 |
| k_exemplars | 10 | 保持与v2一致 |
| graph_type | knn/hybrid | KNN特征图或混合图 |
| batch_size | 16 | 相比v2大幅降低（ViT显存占用大） |
| dropout | 0.5 | 相比v2显著提高（抑制过拟合） |
| learning_rate | 1e-5 | 更保守（大模型需要稳定优化） |
| weight_decay | 1e-4 | 增加正则化抑制过拟合 |

#### 训练流程
1. **数据准备**：使用split.py进行空间无重叠划分
2. **特征提取**：ViT-Large特征提取（冻结前4层）
3. **图构建**：KNN图或空间图构建
4. **代表库构建**：Exemplar KNN检索
5. **模型训练**：GCN图卷积训练
6. **多患者联合训练**：支持HYZ15040/JFX0729/LMZ12939联合

#### 成本-收益分析
- **投入**：5.5-7天（环境搭建+代码适配+训练+评估）
- **预期收益**：EGN-v1 PCC 0.40-0.44，相比v2提升0.01-0.04
- **对比**：UNI升级投入3-4天，预期PCC提升0.05-0.10

  - patch文件名需符合"patch_x\d+_y\d+.png"格式，否则无法匹配坐标
- 特征缓存缺失
  - 确认cache_root路径一致，先运行特征提取再进行训练/推理
  - 检查config_utils.resolve_path()是否正确解析路径
- GPU显存不足
  - 降低batch_size（如128或64），确保特征加载在CPU端
  - 使用config_utils.get_device()自动检测设备，避免手动配置错误
  - EGN-v1需要更大的显存（建议32GB），EGN-v2需要16GB+
- 过拟合
  - 增大dropout（0.3~0.5）、减小hidden_dim（128）、降低学习率
  - EGN-v1需要更高的dropout（0.5 vs 0.3）
- 标签与图片不匹配
  - 检查CSV第一列patch_id与文件名一致性
- Z-score后出现NaN
  - 检查是否存在常数列或缺失值
- 推理指标全NaN
  - 检查预测值是否存在常数列或极端异常值
- 配置文件加载失败
  - 检查config.yaml格式是否正确，确认项目根目录包含config.yaml
  - 使用config_utils.load_config()的错误处理机制
- 通知系统异常
  - 检查plyer库安装情况，Windows平台Toast通知需要相应支持
  - 验证PAUSE_TRAINING信号文件是否存在
- 可视化生成失败
  - 检查matplotlib字体配置，确保中文字体可用
  - 验证输入的CSV文件格式和路径
- EGN-v1训练失败
  - 检查ViT-Large权重加载是否成功
  - 确认batch_size设置为16（避免显存溢出）
  - 验证GCN图构建是否正确
- EGN-v2训练失败
  - 检查ResNet特征提取是否正常
  - 确认GraphSAGE图卷积层配置正确
  - 验证空间图构建半径（默认300）设置合理
- OpenMidnight集成问题
  - 检查OpenMidnight权重文件路径是否正确
  - 验证特征提取与标签对齐逻辑
  - 确认MLP回归头维度匹配（1536→512→256→8）

## 结论
本指南基于仓库现有实现，结合新增的配置管理系统、通知系统、可视化模块，以及新增的EGN-v1/v2部署方案和OpenMidnight集成配置，给出了生产环境部署与配置的系统性方案。通过详细的模型比较分析和部署建议，读者可以根据自身硬件与业务场景，在UNI2-h迁移学习方案、HisToGene端到端方案、EGN-v1/v2图神经网络方案和OpenMidnight基础模型方案之间做出最优选择。

**核心建议**：
1. **短期部署**：优先选择EGN-v2 + UNI特征升级方案，投入3-4天，预期PCC提升0.05-0.10
2. **长期优化**：在EGN-v2基础上进行超参数微调和GNN增强，累计PCC可达0.50-0.56
3. **替代方案**：如需更强大的特征提取能力，可考虑OpenMidnight + MLP回归头方案
4. **硬件选择**：根据模型复杂度选择合适的GPU配置，EGN-v1需要32GB+显存，EGN-v2需要16GB+显存

## 附录

### A. 生产环境部署要求与配置
- **环境与依赖**
  - Python 3.10 + PyTorch 2.1.0 + CUDA 11.8
  - 依赖库：pandas、scikit-learn、Pillow、numpy==1.26.4、huggingface_hub、timm>=0.9.8、matplotlib、plyer
  - EGN模型额外依赖：torch_geometric、sklearn
  - OpenMidnight额外依赖：timm、torchvision、einops
- **硬件配置建议**
  - **GPU**：建议至少12GB显存（如RTX 4090/3090），显存紧张时降低batch_size
  - **内存**：建议≥32GB，用于数据加载与特征缓存
  - **存储**：特征缓存约数百MB至数GB，取决于数据规模；建议SSD以提升I/O
  - **EGN-v1**：建议64GB+显存，支持ViT-Large训练
  - **EGN-v2**：建议32GB+显存，支持ResNet-50训练
  - **OpenMidnight**：建议32GB+显存，支持ViT-G/14特征提取
- **分布式与推理**
  - 训练：单机多卡可通过DataParallel或DistributedDataParallel扩展（需自行改造）
  - 推理：特征缓存可跨节点共享，减少重复计算
  - EGN模型：GraphSAGE支持归纳学习，适合推理时的未见图结构
  - OpenMidnight：支持单GPU到多节点训练，适合大规模数据集
- **模型压缩与加速**
  - UNI2-h骨干冻结，仅训练回归头，参数量小、推理快
  - 可尝试INT8/动态量化（需评估精度）与ONNX导出（需验证兼容性）
  - EGN模型：GraphSAGE相比GCN计算开销更小
  - OpenMidnight：ViT-G/14特征提取效率高
- **容器化部署**
  - Docker镜像建议：nvidia/cuda:11.8.0-runtime-ubuntu20.04 + Python 3.10 + PyTorch 2.1.0
  - 挂载卷：数据目录、特征缓存目录、检查点目录、输出结果目录
  - 运行命令示例（不含具体路径）：docker run --gpus all -v /host/data:/data -v /host/cache:/cache -v /host/checkpoints:/checkpoints -v /host/res:/res pfmval:latest python uni2h/train.py ...
  - OpenMidnight容器：建议使用官方提供的虚拟环境
- **监控与日志**
  - 日志：训练/推理脚本输出stdout/stderr，建议重定向至文件并按日期轮转
  - 指标：MSE、MAE、R2、PCC，建议定期导出CSV并可视化
  - 资源：nvidia-smi、top、iostat监控GPU/CPU/磁盘
  - 通知：通过notify_utils.py提供训练完成/中断通知，支持系统Toast通知
  - OpenMidnight：支持Weights & Biases日志记录
- **模型更新与维护**
  - 版本控制：以Git管理代码与检查点；特征缓存以目录结构+哈希标识版本
  - 更新流程：新数据→重新运行split/zscore→重新提取特征→增量训练→评估→替换线上checkpoint
  - 配置管理：通过config.yaml集中管理所有配置，支持环境变量覆盖
  - EGN模型：支持多患者联合训练，便于模型扩展
  - OpenMidnight：支持断点续训，便于大规模训练
- **安全与访问控制**
  - HF Token：通过config_utils.py的优先级机制管理，支持配置文件和环境变量
  - 文件权限：仅授予必要用户读写权限；缓存与检查点目录严格控制
  - 网络：若使用私有镜像仓库，配置代理与证书
  - OpenMidnight：支持HF Token认证，便于访问预训练权重
- **可视化与报告**
  - 自动生成训练报告：通过visualize_results.py生成完整的可视化报告
  - 支持命令行和Python函数两种使用方式
  - 自动字体配置，支持中文字体显示
  - EGN模型：支持逐通路指标可视化和模型参数面板
  - OpenMidnight：支持训练历史可视化和性能指标分析
