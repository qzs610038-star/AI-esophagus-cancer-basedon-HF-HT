# 可视化与输出规范

> ⚡ 本文件仅在涉及训练结果可视化、报告生成时阅读。

---

## 一、时间戳隔离

- 每次生成的可视化结果必须保存到独立时间戳子目录
- 路径格式：`{model_dir}/checkpoints/results_vis/{dataset}_{timestamp}/`
- `generate_full_report` 只能全局调用一次，避免重复创建时间戳目录
- 新模型可视化需主动检索历史规范，不能凭猜测

## 二、数据一致性

- 最佳epoch基于 val_loss 最小（非 val_pcc 最大），两者可能对应不同epoch
- `model_params.txt` 可能记录中间checkpoint值，完整训练历史以CSV为准
- 汇总对比时务必使用同一标准（val_loss最小epoch）

## 三、逐通路PCC双输出

- 图片：`pcc_barplot.png`（直观展示）
- 表格：`per_pathway_pcc.csv`（可编辑，便于筛选高预测效果通路）
- 列格式：pathway / pcc / r² / mae / rank（按PCC降序排列）

## 四、标准输出文件清单

每次训练完成后必须生成以下文件：
1. `model_params.txt` — 模型参数设置+关键指标
2. `training_history_{dataset}.csv` — 逐epoch训练记录（含train_loss, val_loss, val_pcc, val_mae, val_r2）
3. `predictions.csv` — 逐样本预测（列名：`true_通路名`/`pred_通路名`）
4. `training_curves.png` — Loss/PCC/MAE训练曲线图
5. `pcc_barplot.png` — 逐通路PCC柱状图
6. `per_pathway_pcc.csv` — 逐通路指标表
