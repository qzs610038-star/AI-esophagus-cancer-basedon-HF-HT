# 组会汇报撰写指南

> 本文档汇总了历次组会汇报的写作经验与教训，供撰写新汇报时参照。
> 核心目标：**让有医学背景但无AI背景的听众（含医生）也能快速理解研究进展**。

---

## 一、核心写作原则

### 1.1 结论前置

- **每章/每节的第一句话就是结论**，读者不用往下翻就知道"做了什么、效果如何"
- 结论格式示例：
  > ✅ GAT模型在跨患者泛化场景下PCC提升12.3%，表明图结构信息有效增强空间上下文建模。

### 1.2 医生可读

- 有医生参与，所有技术原理**必须用类比/比喻解释**
- 首次出现的专业术语 → 后跟括号注释或脚注
- 示例：`PCC（皮尔逊相关系数，衡量预测值与真实值的吻合度，1.0=完美预测）`

### 1.3 避免术语堆砌

- 一段话中专业术语不超过3个，超出时拆分或附注
- 能用日常语言说清楚的，不用专业术语

### 1.4 数据可视化优先

- **表格 > 文字描述**：数值对比一律用表格呈现
- **图表 > 表格**：趋势变化用折线图/柱状图
- 长段纯文字分析必须配至少一个表格或图

### 1.5 数据真实性

- **所有数据必须来自实际训练结果CSV**，严禁编造或近似
- 引用结果时标注来源文件路径
- 模型对比排除已淘汰方案（如EGN-v1已不参与正式对比）

### 1.6 学术准确性

- 技术描述严格对齐原始论文与官方代码
- 不确定的参数/结构，先查论文再写
- 引用模型时标注出处（如 HisToGene, NeurIPS 2022）

## 二、文档结构模板

```markdown
# 组会汇报_[主题]_[日期]

## 摘要
> 2-3句概括本次汇报的核心发现。

## 一、研究背景（1页内）
- 问题定义
- 数据集概况（患者数、样本量、通路数）

## 二、方法概述
- 模型架构（配架构示意图）
- 关键创新点（用类比解释）
- 超参数设置（附设置依据）

## 三、实验结果
### 3.1 主要指标对比（表格）
### 3.2 逐通路分析（Top5提升/下降）
### 3.3 跨患者泛化（如适用）
### 3.4 消融实验（如适用）

## 四、讨论与结论
- 关键发现（编号列出）
- 局限性
- 下一步计划

## 附录
- 详细训练参数表
- 完整通路PCC表
```

### 结构注意事项

- 新增模型结果需完整覆盖：**指标表 + 衰减表 + 分析要点 + 结论**
- 使用 `>` 块引用标注补充说明或给初学者的注释
- 关键数据变更时**同步更新所有相关表格和分析段落**

## 三、技术概念类比库

> 撰写汇报时可直接复用以下类比，无需每次重新构思。

| 技术概念 | 医生友好解释 |
|---------|------------|
| PCC（皮尔逊相关系数） | 预测值与真实值的吻合程度，1.0=完美预测，0=毫无关联 |
| MSE损失函数 | 预测偏差的"惩罚分数"，偏得越多罚得越重 |
| 过拟合 | 学生只背了答案没学方法，换套卷子就考砸 |
| 欠拟合 | 学生连课本都没看完，什么都答不好 |
| 跨患者泛化 | 在A患者身上学到的经验能否用于B患者 |
| 特征提取器（UNI2-h） | AI的"眼睛"，从病理图像中看出有意义的信息 |
| Transformer注意力机制 | 模型看图时的"注意力分配"，决定重点看哪里 |
| GAT（图注意力网络） | 让每个组织区域主动询问周围邻居的情况，再综合判断 |
| GNN（图神经网络） | 把组织切片看作一张"地图"，让模型理解区域之间的空间关系 |
| z-score标准化 | 把不同患者的指标统一到同一把尺子上衡量 |
| Early Stopping | 学习到"刚好够"就停，防止过度学习（背答案） |
| Batch Size | 每次让模型同时看多少个样本学习，类似每堂课教几个病例 |
| Learning Rate | 模型的"学习步幅"，太大容易跳过最优解，太小学得太慢 |
| Epoch | 把所有训练数据完整学一遍算一个epoch，类似把教材从头到尾看一遍 |
| Dropout | 训练时随机"遮住一只眼"，迫使模型不依赖单一特征 |
| 多模态融合 | 同时利用"看图"和"读基因"两种信息做判断，类似医生结合影像和化验 |
| 对比学习 | 让模型学会区分"相似"和"不同"的样本对，类似找不同游戏 |
| 域适应 | 让模型适应不同"考场环境"的能力 |
| 梯度累积 | 显存不够时的"分批攒经验"策略，攒够一批再更新 |
| ViT（Vision Transformer） | 把图像切成小块后逐块分析，类似医生用放大镜逐区域看切片 |
| KNN图构建 | 根据物理距离找到每个点的"最近邻居"，建立空间联系 |
| 残差连接 | 给模型留一条"直达通道"，防止信息在深层网络中丢失 |
| 位置编码 | 告诉模型每个图块"在切片上的哪个位置"，保留空间信息 |
| 通路评分（ssGSEA） | 把上百个基因的表达量浓缩为一个"通路活性得分" |
| 消融实验 | 逐个去掉模型组件观察效果变化，类似排查哪根线路出了问题 |
| 预训练模型 | 已在海量数据上"预习"过的模型，迁移到新任务时起步更高 |
| 微调（Fine-tuning） | 在预训练基础上针对特定任务"专项训练" |
| 冻结参数 | 锁住已学好的部分不再改动，只训练新增模块 |
| 注意力热图 | 可视化模型"在看哪里"，帮助解释预测依据 |
| 收敛 | 模型性能趋于稳定不再提升，类似学习到了瓶颈 |

## 四、数据呈现规范

### 4.1 指标对比表标准格式

```markdown
| 模型 | 数据集 | Mean PCC ↑ | MSE ↓ | 最佳Epoch | 备注 |
|------|--------|-----------|-------|----------|------|
| HisToGene-UNI | HYZ15040 | 0.312 | 0.847 | 89 | 单患者训练 |
```

- ↑ 表示越高越好，↓ 表示越低越好
- 保留3位小数
- 最佳epoch基于 **val_loss最小**（非val_pcc最大）

### 4.2 衰减/提升对比表

```markdown
| 模型 | 单患者PCC | 跨患者PCC | 衰减幅度 |
|------|----------|----------|---------|
| HisToGene-UNI | 0.312 | 0.213 | -31.7% |
```

### 4.3 图表要求

- 折线图必须标注坐标轴含义和单位
- 柱状图需附数值标签
- 配色考虑打印友好（避免纯红绿对比）

### 4.4 数据来源标注

- 每个表格下方用小字标注数据来源：
  > 数据来源：`checkpoints/HisToGene_UNI_HYZ15040/training_history.csv`

## 五、通路分析写作规范

### 5.1 通路命名

- **禁止使用通路索引号**（如 Pathway_0），必须替换为全名
- 格式：`英文全名（中文含义）`
- 示例：`HALLMARK_EPITHELIAL_MESENCHYMAL_TRANSITION（上皮-间质转化通路）`

### 5.2 Top5提升/下降通路分析模板

```markdown
#### 预测效果最好的Top5通路

| 排名 | 通路名称 | PCC | 生物学解释 |
|-----|---------|-----|-----------|
| 1 | HALLMARK_GLYCOLYSIS（糖酵解通路） | 0.65 | 糖酵解活性在H&E中表现为细胞密度和嗜酸性改变，形态学特征明显 |

#### 预测困难的Bottom5通路

| 排名 | 通路名称 | PCC | 困难原因 |
|-----|---------|-----|---------|
| 1 | HALLMARK_TGF_BETA_SIGNALING（TGF-β信号通路） | 0.05 | 依赖微环境上下文信息，无法从单patch形态学直接推断 |
```

### 5.3 预测困难归因类型

常见的"形态学不可见"解释模板：

1. **微环境依赖型**：该通路活性由细胞间信号调控，单patch无法捕获跨区域交互
2. **分子层面型**：通路活性由蛋白修饰/表观遗传决定，H&E染色无法反映
3. **异质性型**：同一形态学表现下通路活性差异大，一对多映射困难
4. **低动态范围型**：该通路在所有样本中活性接近，预测与随机无异

## 六、参数设置解释模板

每个关键超参数需从**四个维度**说明设置依据：

```markdown
### [参数名] = [值]

| 维度 | 说明 |
|------|------|
| 显存约束 | 当前GPU显存为X GB，该设置下显存占用约Y GB |
| 表征能力 | 该值能捕获Z级别的特征粒度 |
| 鲁棒性 | 避免过拟合/欠拟合的权衡 |
| 收敛性 | 在N个epoch内可稳定收敛 |
```

### 常见参数解释示例

| 参数 | 值 | 设置依据 |
|------|---|---------|
| img_size | 224 | UNI2-h预训练分辨率，保持一致避免性能损失 |
| patch_size | 112μm | 与10x物镜下组织区域对应，平衡细节与上下文 |
| depth | 4 | 4层Transformer足以建模通路级特征，更深易过拟合小数据 |
| heads | 8 | 多头注意力捕获不同粒度的空间模式 |
| lr | 1e-4 | Adam优化器下的经验最优，配合cosine衰减 |
| batch_size | 1 | 受限于单患者样本量和图结构，每次处理一张完整图 |

## 七、常见问题与教训

### 7.1 数据相关

| 问题 | 教训 |
|------|------|
| 引用了错误的训练结果 | 必须确认CSV文件路径和时间戳，不能凭记忆 |
| 单患者vs跨患者数据搞混 | 表格中明确标注训练模式（单/联合/跨患者） |
| 通路数量不一致 | 确认ssGSEA的hallmark通路固定为50条 |

### 7.2 写作相关

| 问题 | 教训 |
|------|------|
| 全文充满术语无解释 | 首次出现必须附注，后续可用简称 |
| 结论埋在分析末尾 | 严格执行结论前置，每节第一句即结论 |
| 表格数据与文字不一致 | 修改数据时全文搜索同步更新 |
| 模型命名混乱 | 统一格式：`模型名-特征类型`（如HisToGene-UNI） |

### 7.3 技术准确性

| 问题 | 教训 |
|------|------|
| 模型架构描述与论文不符 | 写架构时必须对照原始论文，不凭印象 |
| 对比了已淘汰模型 | EGN-v1已不参与正式对比，仅作历史参考 |
| 混淆val_loss最小与val_pcc最大 | 统一标准：最佳epoch = val_loss最小的epoch |

## 八、检查清单

完成汇报文档后，逐项自查：

### 结构完整性
- [ ] 摘要（2-3句核心发现）
- [ ] 每节开头有结论性总结句
- [ ] 新增模型包含完整四要素（指标表+衰减表+分析+结论）

### 可读性
- [ ] 所有术语首次出现时有通俗解释
- [ ] 技术原理配有类比/比喻
- [ ] 无超过5行的纯文字段落（需配表/图）

### 数据准确性
- [ ] 所有数值来自实际CSV，非手动编造
- [ ] 数据来源路径已标注
- [ ] 表格数据与文字描述一致
- [ ] 最佳epoch标准统一（val_loss最小）

### 通路分析
- [ ] 通路名使用全名+中文，非索引号
- [ ] Top5提升通路有生物学解释
- [ ] Bottom5困难通路有形态学不可见归因

### 参数说明
- [ ] 关键超参数有设置依据
- [ ] 说明覆盖显存/表征/鲁棒性/收敛性

### 格式规范
- [ ] 表格有↑↓标注指标方向
- [ ] 块引用用于补充说明
- [ ] 模型命名统一（`模型-特征`格式）
- [ ] 无对已淘汰模型（EGN-v1）的正式对比

### 专业听众Q&A预判
- [ ] 每个实验结果预判 ≥ 3 个专业人员可能提问
- [ ] 每个预判问题附简短回答（控制在 3 句内）
- [ ] 问题覆盖：方法学合理性 / 实验设计严谨性 / 结果解释 / 对比文献

## 九、专业听众 Q&A 预判指南

> 组会听众包含 ML 专业研究生和老师。汇报需同时满足"医生听得懂"和"专业听众挑不出漏洞"两个目标。本节指导如何预判专业提问并准备回答。

### 9.1 专业听众的关注点（与医生不同）

| 医生关心 | ML 专业人员关心 |
|---------|--------------|
| 这个模型能帮我看病吗？ | 实验设计是否公平？是否有混杂变量？ |
| 预测得准不准？ | 为什么选择这个 baseline？是否跟 SOTA 比过？ |
| 比现有方法好多少？ | 统计显著性？是否做了多次重复实验？ |
| 下一步做什么？ | 失败实验的根因分析是否到位？ |

### 9.2 Q&A 预判格式模板

在汇报文档末尾追加"预判 Q&A"附录，格式如下：

```markdown
## 附录 X：预判专业提问与回答

> 以下为汇报前预判的专业听众可能提问，提前准备回答。实际汇报时如被问到可直接引用。

### Q1：[问题标题]

**预判理由**：[为什么预测会被问到这个问题——基于哪段汇报内容、哪个实验选择]

**回答要点**：
1. [核心论点 1]
2. [核心论点 2]
3. [如果被追问 → 备选回答]

### Q2：...
```

### 9.3 高频预判问题类型

以下为组会中专业人员经常提问的类型，撰写汇报时需逐一预判：

#### 类型 A：实验设计合理性

| 典型问题 | 准备方向 |
|---------|---------|
| "为什么只跑 Fold 1 不跑全三折？" | 解释快速验证策略：单折 15 分钟 vs 三折 45 分钟，仅在单折确认有效后才全跑 |
| "为什么不用 k-fold 交叉验证？" | 跨患者场景下固定 fold 划分（按患者分）比随机 k-fold 更有临床意义 |
| "你这个 α 扫参范围为什么是 0.2/0.4/0.6？" | 解释 Beta(0.2,0.2) 双峰分布在低 α 区间的理论依据 |
| "超参是怎么选的？有没有做 grid search？" | 说明手动调参 vs grid search 的算力约束 |

#### 类型 B：结果可靠性

| 典型问题 | 准备方向 |
|---------|---------|
| "0.4228 vs 0.4133 这个差异显著吗？" | 同参数两次运行的随机波动 ~0.0095（2.2%），证明 0.42 附近为稳定天花板 |
| "有没有做统计检验？" | 解释当前用单次实验 Best PCC 做排序的局限性，建议补充多次运行均值±标准差 |
| "这个 PCC 是高还是低？跟别人比呢？" | 引用文献中类似任务的 PCC 范围（如 HisToGene 原论文 ~0.3-0.5），提供参照系 |

#### 类型 C：方法学选择

| 典型问题 | 准备方向 |
|---------|---------|
| "为什么用 HuberLoss 不用 MSE？" | HuberLoss 对异常值鲁棒，P1-1 变体 A 用 MSE 后 PCC 下降 + loss 尺度失控证明此选择正确 |
| "为什么选 Virchow2 不选 Prov-GigaPath？" | 比较三个 PFM：UNI(307M)/Virchow2(632M)/Prov-GigaPath(~1B)，Virchow2 是性价比最优的升级选择 |
| "LoRA rank 选多少？为什么？" | 病理任务通常 rank=4-8 足够，rank 越高增加参数越多但收益递减 |
| "Transformer 层数为什么只试到 2？" | 试 3 层的边际成本 vs 预期收益；首先验证 1→2 是否有信号，无效则 2→3 大概率也无益 |

#### 类型 D：负结果归因

| 典型问题 | 准备方向 |
|---------|---------|
| "P1-1 失败了，是不是权重设计得不好？" | 不仅权重问题——即使 3.7× 区分度 + HuberLoss（变体 B）也低于基线。根因是容量瓶颈 |
| "你怎么确定是特征瓶颈而不是其他原因？" | 列举证据链：①三个架构收敛于 ~0.38 ②损失加权无效 ③架构加深无效 ④全距仅 0.0095 |
| "有没有可能是数据量太小？" | 承认此局限性——15K 样本对 30 通路回归是中等偏小，但也是可获得的最大数据集 |

#### 类型 E：文献对比

| 典型问题 | 准备方向 |
|---------|---------|
| "这个方向最近有什么相关论文？" | 准备 1-2 篇最近顶会/顶刊论文（如 OmiCLIP NeurIPS 2025、HistoPrism Nature 2025）的核心结论和我们的差距 |
| "跟 HisToGene 原论文比怎么样？" | 原论文单患者 PCC~0.3-0.5，我们跨患者 PCC~0.42 在合理范围内。但我们有 30 通路而原论文仅 8 通路，任务更困难 |

### 9.4 Q&A 准备清单

撰写完汇报后，逐项确认：

- [ ] 每个"阴性结果"实验预判了 ≥ 2 个质疑方法学的问题
- [ ] 每个"新方案"（如 Virchow2）预判了 ≥ 2 个合理性问题
- [ ] 准备了文献对比参照系（我们的 PCC vs 文献中类似任务的 PCC）
- [ ] 准备了统计可靠性讨论（单次实验局限性 + 多次运行波动范围）
- [ ] 对算力受限的决策（不做 LoRA 做 Virchow2）有充分工程论证
- [ ] 所有预判回答控制在 3 句以内（长回答在实际组会中会被打断）
- [ ] Q&A 语言适合口头回答（短句、结论前置、有数据支撑）
